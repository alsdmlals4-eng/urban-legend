# DESIGN_INTENT

## 기획 의도

`도시괴담 기록국`은 괴담을 힘으로 쓰러뜨리는 게임이 아니라, 괴담이 작동하는 규칙을 조사하고 기록해 봉인·회수하는 게임이다.

초기 MVP의 목적은 모든 시스템을 크게 완성하는 것이 아니라, 다음 감각을 짧은 루프로 검증하는 것이다.

```text
이상한 사건을 읽고, 조사하고, 선택하고, 기록으로 남긴다.
```

## 플레이어에게 주고 싶은 감정

- 낯선 도시괴담 사건에 투입되는 긴장감
- 단서가 쌓이며 사건 규칙이 보이는 이해감
- 선택한 조사 방식에 따라 위험과 보상이 달라지는 책임감
- 요원이 단순 보정 수단이 아니라 함께 수사하는 동료라는 감각
- 결과 화면에서 내가 만든 사건 기록을 확인하는 정리감

## 핵심 선택 구조

선택은 다음 네 종류로 나눈다.

| 선택 종류 | 역할 | 예시 |
|---|---|---|
| 대화 선택 | 플래그, 단서, 다음 노드, 조사 조건을 연다 | 전광판 촬영, 역무원실 접근 요청 |
| 조사 포인트 선택 | 단서/힌트/플래그를 얻는다 | 피해자 휴대폰, 검은 승차권, 플랫폼 스피커 |
| 조사 방법 선택 | 파괴/관찰/분석으로 위험도와 이해도, 요원 신뢰도를 바꾼다 | 잠긴 문을 부수기, 열쇠 찾기, 패턴 분석 |
| 회수 선택 | 괴이 안정도와 회수 조건을 관리한다 | 기록 스캔, 임시 봉인지, 요원 지원 |

## 요원 설계 원칙

요원 성향은 세 개만 사용한다.

- `analytical` / 분석형: 규칙, 패턴, 기록, 조건 해석
- `empathetic` / 공감형: 피해자 보호, 기억 흔적, 위험 완화
- `breakthrough` / 돌파형: 강행 진입, 위험 감수, 빠른 현장 확보

요원 신뢰도는 연애 호감도가 아니다.

```text
요원 신뢰도 = 수사 파트너로서의 신뢰
```

신뢰도 이벤트는 정답을 대신 알려주지 않고, 다음 조사나 회수 판단의 방향만 작게 보조한다.

## 벤치마킹 반영 원칙

참고 방향:

- `Return of the Obra Dinn`: 관찰과 기록의 결합
- `The Case of the Golden Idol`: 사건 보고서와 페어플레이 추리
- `Her Story`: 파편 기록 복원
- `Disco Elysium / Kim Kitsuragi`: 동료 반응과 수사 파트너 존재감
- `PARANORMASIGHT`: 도시괴담과 분기 재확인

반영 방식:

- 단서 없는 반전은 금지한다.
- 요원 반응은 플레이어 선택을 기억하게 하는 보조 장치로 쓴다.
- 전체 흐름은 HTML 대시보드로 시각화하되, Codex 구현 범위와 분리한다.

## 금지 방향

- 괴담을 단순 몬스터로 취급하지 않는다.
- 전투력/데미지/처치 중심으로 확장하지 않는다.
- 요원 신뢰도를 연애 호감도처럼 만들지 않는다.
- 미니게임이 본편 추리를 방해할 만큼 길어지지 않게 한다.
- 모든 NPC와 요원 개인 루트를 한 번에 만들지 않는다.
- 기록물/장비를 단순 파밍 보상으로 축소하지 않는다.

## Two-Case Demo Direction

저승역과 비 오는 골목의 빨간 우산은 서로 다른 조사 상황을 보여 주는 기준 샘플이다. 세 번째 사건을 추가하기 전에, 플레이어가 두 사건에서 선택과 결과를 읽기 쉽게 만들고 같은 핵심 루프가 안정적으로 반복되는지 검증한다.

사건 보고서는 다음 질문에 답해야 한다.

1. 어떤 사건을 해결했는가?
2. 어떤 단서를 얻었는가?
3. 어떤 미니게임 결과가 있었는가?
4. 어떤 요원이 반응했고 신뢰도가 어떻게 변했는가?
5. 어떤 기록물·장비·연구 보상이 남았는가?
6. 다음 사건으로 무엇이 이어지는가?

MVP-021~030은 새 이야기보다 UI, 시스템 피드백, 저장 안정성, 모바일 가독성, 두 사건 통합 QA를 우선한다.
