# 장면 중심 현장·회수 UI

## 목적

조사와 회수 화면을 정보 패널 중심의 도구 화면에서 배경·요원·괴이가 먼저 보이는 현대 오컬트 게임 화면으로 전환한다. 게임 규칙과 저장 데이터는 유지하며 표현 계층만 바꾼다.

## 현장 화면

- 메인 조사는 화면 상단의 큰 배경과 하단의 상황 설명·선택 도크를 사용한다. 로그는 핵심 안내가 필요할 때 시스템 메시지를 대신하며, 편성 요원 반응은 작은 얼굴 초상과 함께 최대 1~2줄만 표시한다.
- 전신 스탠딩과 대화 중심 연출은 향후 일상 에피소드용으로 남기며 메인 조사에는 상시 표시하지 않는다.
- 조사 지점 선택 시 하단이 지점 34%, 조사 방식 66%의 두 열로 전환된다. 두 열만 독립적으로 스크롤하며 화면 전체는 스크롤하지 않는다.
- 팀 상태는 좌상단 칩, 단서 수집과 회수 진입은 우상단 HUD, 상세 단서·힌트·학습 기록은 닫을 수 있는 기록 서랍에 둔다.
- `FIELD_DIALOGUE`, `FIELD_CHOICES`, `POINT_PICKER`, `METHOD_PICKER`, `RESULT`는 저장 필드가 아니라 현재 대화와 선택 상태에서 계산하는 UI 모드다.

## 회수·안정화 화면

- 얇은 상단 HUD, 중앙 대치 스테이지, 하단 액션 도크의 세 계층으로 구성한다.
- 기존 `recovery_patterns.responses`를 `ActionChoiceCard`로 표시한다. 판정, 정답, 안정도, 피해와 회수 조건은 기존 `GameState`가 계속 소유한다.
- 대표 요원 교체는 기존 선택 요원 배열을 순환하는 보조 동작이며 새로운 전투 시스템이나 저장 필드를 만들지 않는다.
- 핵심 단서는 HUD에 요약하고 상세 효과·지원·개발 이동은 우측 근거 서랍에서 확인한다.

## 공통 컴포넌트와 자산

- `ActionChoiceCard`는 행동 ID를 신호로만 전달하고 상태를 수정하지 않는다.
- `TeamStatusChip`은 체력·정신력·행동 가능·대표 여부만 표시한다.
- 요원과 괴이 원본 이미지는 보존한다. `assets/ASSET_MANIFEST.json`의 파생 컷아웃만 장면 합성에 사용하며 누락 시 기존 합성 이미지로 폴백한다.
- F2 편집기는 `cinematic_*` 안정 노드만 편집하고 동적 카드는 개별 등록하지 않는다.

## 호환성과 승격 후보

- `dialogue_scene`의 구형 저장 리디렉션, 에피소드 JSON, `GameState` API와 저장 스키마 `mvp-035`를 유지한다.
- Base 승격 후보: Godot UI에서 배경 스테이지와 상태 도크를 분리하고, 게임 상태를 소유하지 않는 재사용 선택 카드로 동적 행동을 표현하는 원칙.
