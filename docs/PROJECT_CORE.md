# Urban Legend 프로젝트 코어

> 상태: `IDENTIFIED` — 기존 승인 문서·현재 구현·불변 조건을 읽기 전용으로 분류한 결과다. 이 문서는 아직 `CORE_CONFIRMED` 또는 `CORE_RECORDED`가 아니며, 그 상태는 이 계약 전문에 대한 사용자 명시 승인 뒤에만 가능하다.

## 프로젝트 정체성 한 문장

플레이어는 **괴이 기록국** 요원 팀을 운용해 괴이의 규칙을 조사·추론하고, 현재 출현을 **안정화 상태**로 만든 뒤 **잔향**과 **위험 사례**를 다음 생존 절차인 **괴이 매뉴얼**로 기록한다.

## PROJECT_CORE

1. **규칙 조사와 공정한 추론**: 관찰 가능한 단서로 괴이 규칙을 판정하며 요원·기록관 아카·장비·자동행동이 정답을 대신하지 않는다.
2. **핵심 루프**: 사건·상황 파악 → 단서 조사 → 규칙 가설·선택 → 검증 → 안정화 → 잔향 회수 → 보고서·DB·괴이 매뉴얼 갱신 → 다음 일정.
3. **실패의 정보 가치**: 실패는 단순 진행 차단이 아니라 `위험 사례`와 배제 근거로 남아 다음 판단을 개선한다.
4. **비전투 정체성**: 괴이는 처치 대상이 아니며 HP·공격·처치 중심 전투로 핵심 경험을 대체하지 않는다.
5. **기록 보상**: 사건의 최종 성취는 적 제거가 아니라 재현 가능한 대응 절차와 괴이 매뉴얼 축적이다.

## CORE_SUPPORT

- 권나래 고정 주인공과 최대 두 명의 서포트 편성.
- 기록관 아카의 관제·안내와 관계·대사·표정·컷인.
- HQ 일정, DB·연구·시장·세력 의뢰, 보고서와 회수 연출.
- 정보 위계·접근성·사운드·아트는 코어를 설명하고 강화하지만 판정 상태를 소유하지 않는다.

이 요소들은 현재 경험을 강하게 지지하지만, 동일한 플레이어 약속을 보존하는 대체 구현 가능성을 별도로 검토한다.

## MVP_SUPPORT와 CONTENT_VARIANT

- 현재 구현 사건 3개, 최대 10일·오전/오후 구조, 세력 3계열, 현재 요원·외부 접점 수.
- 개별 사건·미니게임·일상 에피소드·대사·보상 수치·콘텐츠 분량.

현재 MVP를 실행·관찰하는 데 필요하지만 개별 수량·사례가 프로젝트 정체성 전체를 고정하지는 않는다.

## TECHNICAL_FOUNDATION

- Godot 4.7 stable·GDScript·PC 16:9·마우스/키보드.
- 저장 Schema `mvp-039`와 `mvp-038` 이관, 기존 ID·저장 키·`selected_agent_ids` 호환.
- `scripts/core/game_state.gd`, `data/episodes/**`, `project.godot`, `knowledge/base-pack/**` 보호 경로.
- Scene·데이터·저장 상태의 단일 소유자와 기존 세 사건 왕복 호환.

기술 기반은 여러 기능이 의존하지만 구현 방식 전체가 제품 정체성과 동일한 것은 아니다. 대체 시 저장·ID·행동 계약을 보존해야 한다.

## PRESENTATION_SHELL / CHANGEABLE

- UI 배치·테마·카드 형태·컷인 방식·세부 색·폰트·연출 밀도.
- 특정 사건명·대사·배경·개별 캐릭터 자산과 콘텐츠 수량.
- 생성 도구·파이프라인 내부 구현.

변경 가능하지만 정보 공정성, 현대 오컬트 톤, 한국어 가독성, 입력 접근성, 상태 비소유 원칙을 침해하면 재설계가 필요하다.

## REQUIRES_REAPPROVAL

- 조사·추론 대신 전투·자동 정답·숨은 수치 중심으로 핵심 행동 변경.
- 안정화·잔향·위험 사례·괴이 매뉴얼의 의미 변경.
- 저장 Schema·기존 ID·캠페인·경제·엔딩 의미 변경.
- 권나래 고정 주인공 또는 플레이어 역할의 본질 변경.
- 프로젝트 코어 항목의 제거·대체·축소.

## 제거·대체 테스트

- 이 요소를 제거해도 규칙 조사→판단→안정화→기록의 약속이 남는가?
- 다른 구현으로 대체해도 정보 공정성과 플레이어 선택이 유지되는가?
- 콘텐츠를 축소해도 실패가 학습 근거가 되고 다음 사건을 시도할 이유가 남는가?
- 변경하면 전투 게임·자동 진행·비주얼 노벨 등 다른 제품 정체성으로 바뀌는가?
