## 작업 원칙

- 괴이는 처치하지 않는다. 조사·안정화·잔향 회수·매뉴얼 기록이 핵심이다.
- 요원·아카·장비·계약은 정답을 대신 고르지 않는다.
- 저장·진행·경제·엔딩 의미 변경은 고위험이다. 보호 경로와 저장 왕복 테스트가 필요하다.
- 사용자 소유 `.import`, 생성 `.uid`, 수동 캡처는 승인 없이 수정·삭제·커밋하지 않는다.
- 모든 작업은 관련 활성 기획서를 갱신한다. 세부 절차는 [Documentation Map](DOCUMENTATION_MAP.md)을 따른다.
