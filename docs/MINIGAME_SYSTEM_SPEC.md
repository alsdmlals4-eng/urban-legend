# 괴이 조사 미니게임 공통 설계 v0.1

> 상태: **사용자 승인 설계 / 구현 미반영**
>
> 기준일: 2026-07-22
>
> 현행 통합 기준: `MVP-043 + CORE-VALIDATION-001` / `Ver 4.2` / 저장 스키마 `mvp-039`
>
> 적용 범위: 시나리오별 대표 미니게임, 마지막 단서, 위험도·괴이 고착 변수, 조작 횟수 보상, 저장·복구

## 1. 역할

각 시나리오는 대표 미니게임 1개를 가진다. 미니게임은 별도 오락이 아니라 조사에서 세운 가설을 현장에서 검증하는 마지막 조사 행동이다.

```text
괴이 조사
→ 조사 페이즈
→ 사건 고유 미니게임
→ 마지막 단서 확보
→ 회수 페이즈
→ 안정화 상태
→ 잔향 회수
→ 괴이 매뉴얼 갱신
```

### 저승역 MVP-043 + CORE-VALIDATION-001 표시·저장 계약

- `tutorial`: `조작 학습 1/2 · 결과 미저장`, 3×3 내부 학습 단계다.
- `final`: `현장 검증 2/2 · 결과 저장`, 4×4에서만 완료 신호를 한 번 보낸다.
- 최종 화면은 비교 근거/노선 보드/현장 반응 3열이며, 기록 오버레이가 열려 있는 동안 보드 입력을 차단한다.
- 보드 진행은 저장하지 않고 진입 체크포인트와 최종 결과 상세만 기존 저장 구조에 남긴다.

미니게임 성공은 회수 페이즈를 대체하지 않는다. 성공과 실패 모두 관측 결과를 남기며, 플레이어는 이를 회수 대응의 근거로 사용한다.

## 2. 공통 진행 계약

1. 조사 페이즈에서 미니게임에 필요한 근거를 확보한다.
2. 미니게임은 조사 페이즈의 마지막에 열린다.
3. 미니게임 결과가 사건의 마지막 단서가 된다.
4. 정상 완료는 회수 페이즈 진입 조건이다.
5. 잘못된 규칙 적용은 위험 사례로 남긴다.
6. 조작 횟수 초과는 실패가 아니며 핵심 진행을 막지 않는다.
7. 정밀 해결 보상은 핵심 단서·핵심 서사·회수 정답을 독점하지 않는다.

## 3. 조작 횟수 규칙

### 횟수에 포함

- 타일 90도 회전 1회
- 분기 방향 전환 1회
- 고착 해제 장치 조작 1회
- 노선 또는 구조를 실제로 바꾸는 사건 고유 행동 1회

### 횟수에서 제외

- 타일 선택만 하기
- 단서 다시 보기
- 요원 무전 다시 듣기
- 확대·축소와 화면 이동
- 경로 미리보기
- 이동 또는 결과 확정 버튼

되돌리기는 이전 횟수를 취소하지 않는다. 퍼즐을 처음부터 다시 시작하면 보드와 현재 횟수는 초기화한다.

## 4. 최적 횟수와 정밀 기준

각 퍼즐은 변수 없는 기본 보드의 실제 최단 조작 횟수 `N`을 가진다.

```text
기본 최적 횟수: N
정밀 해결 기준: N + 2
```

| 결과 | 조건 | 핵심 진행 | 보너스 |
|---|---|---|---|
| 최적 해결 | `N` 이하 | 정상 | 최적 기록·특별 반응 |
| 정밀 해결 | `N+2` 이하 | 정상 | 정밀 해결 보상 |
| 일반 해결 | `N+2` 초과 | 정상 | 추가 보상 없음 |
| 위험 사례 | 잘못된 규칙·목적지·경로 확정 | 관측 후 재조정 또는 불리한 진행 | 위험 사례 등록 |

`N`은 완료 후 공개한다. 플레이 중에는 `정밀 해결 기준 N+2`와 현재 조작 횟수만 보여준다.

## 5. 위험도·괴이 고착 변수

### 위험도

위험도는 정보 교란과 즉각적인 현장 변화를 만든다.

예시:

- 가짜 목적지 또는 가짜 표식 추가
- 미리보기 일부 노이즈 처리
- 서로 충돌하는 무전·표시 발생
- 예고된 경로 흔들림

### 괴이 고착도

기존 내부 `anomaly_stability`는 플레이어 노출에서 **괴이 고착도**로 표현한다. 값이 높을수록 괴이가 공간과 규칙에 강하게 박혀 있다.

예시:

- 특정 타일 또는 장치 고착
- 우회 해제 조작 필요
- 정상 경로 일부 차단
- 구조를 다시 고정해야 하는 저항 발생

### 변수 단계

| 상태 | 변수 구성 |
|---|---|
| 위험도·고착도 모두 낮음 | 변수 없음 |
| 둘 중 하나가 높음 | 약한 변수 1개 |
| 둘 다 높음 | 약한 변수 2개 또는 구조 변수 1개 |
| 둘 다 매우 높음 | 강한 변수 1개 + 약한 변수 1개 |

첫 구현에서는 한 변수 조합이 강제로 요구하는 추가 조작을 최대 2회로 제한한다. 변수 적용 후의 실제 최단 횟수가 `N+2`를 넘는 보드는 출현시키지 않는다.

변수는 퍼즐 시작 전에 이름과 예상 영향을 예고한다. 정답 타일이나 안전 목적지는 공개하지 않는다.

## 6. 조사 결과와 변수 연결

변수는 무작위 벌점이 아니라 조사 결과의 귀결이어야 한다.

```text
위험한 조사·오대응
→ 위험도 상승
→ 정보 교란 변수 강화

핵심 기록 미확보·대응 지연
→ 괴이 고착도 상승
→ 구조 저항 변수 강화

충분한 근거·안전한 조사
→ 변수 억제 또는 대응 정보 확보
```

단서는 변수를 삭제하거나, 정체를 예고하거나, 대응 장치를 표시할 수 있다. 단서가 미니게임을 자동 해결해서는 안 된다.

## 7. 정밀 해결 보상

### 기본 완료 보장

- 마지막 단서
- 회수 페이즈 진입
- 괴이 매뉴얼 필수 항목
- 사건 핵심 서사

### `N+2` 이내 보상

- 사건별 정밀 해결 기록
- 잔향 기록 품질 또는 보고서 완성도 상승
- 피해자 후유증 완화 가능성
- 회수 페이즈의 부담 소폭 감소
- 전용 요원 반응

### `N` 이하 추가 표현

최적 해결은 별도 큰 성장 보상을 주지 않는다. 기록 표식, 결과 문구, 요원 특별 반응으로 숙련을 인정한다.

### 금지 보상

- 회수 정답 자동 공개
- 필수 단서 독점
- 핵심 이야기 독점
- 다음 사건 정답 힌트
- 장기 밸런스를 크게 흔드는 영구 능력치

## 8. 저장·종료·복구 규칙

### 확정 원칙

- 미니게임 도중에는 수동 저장과 자동 저장을 하지 않는다.
- 미니게임 진입 직전에 복구용 체크포인트를 저장한다.
- 정상 완료 후 다음 페이즈 진입 시 다시 저장할 수 있다.

### 예상치 못한 종료

게임 강제 종료, 충돌, 전원 종료, 프로세스 중단 또는 미니게임 중 불러오기가 발생하면 다음 실행에서 미니게임의 첫 시작 지점으로 돌아간다.

복구 시 초기화되는 항목:

- 타일·장치 방향
- 현재 조작 횟수
- 현 시도의 이동 위치
- 현 시도에서만 발생한 임시 연출 상태

복구 시 유지되는 항목:

- 미니게임 진입 전까지 확보한 단서
- 진입 전 위험도와 괴이 고착도
- 적용할 보드 ID와 변수 조합
- 사건 진행 상태와 요원 편성

같은 체크포인트를 불러오면 같은 보드와 같은 변수 조합으로 시작한다. 종료·불러오기로 유리한 변수나 보드를 다시 뽑을 수 없어야 한다.

### 정상적인 중단 시 안내

플레이어가 미니게임 중 메인 메뉴 또는 게임 종료를 선택하면 다음을 알린다.

> 미니게임 진행 상황은 저장되지 않습니다. 다시 시작하면 이 미니게임의 처음부터 진행합니다.

확인 후 종료할 수 있으며, 다음 불러오기에서는 미니게임 시작 연출의 짧은 재진입 버전부터 시작한다.

### 구현 축소 원칙

초기 PoC는 타일별 진행 상태를 저장하지 않는다. 복구에 필요한 최소 정보는 미니게임 진입 체크포인트, 고정된 보드 ID, 변수 조합 또는 결정론적 시드다. 기존 저장 구조로 표현할 수 있는지 먼저 확인하며, 새 저장 필드는 영향 분석과 승인 없이 추가하지 않는다.

## 9. 저승역 적용

### 고정 편성

- 오현: 방송실·공식 운행 기록 확인, 무전 전달
- 권나래: 피해자 호송, 현재 위치와 현장 표지 보고
- 강이준: 현실 경계와 선택 노선 고정

### 노선 복원 미니게임

플레이어는 오현과 권나래의 보고를 대조해 안전 목적지를 판별하고, 뒤틀린 노선 타일을 회전·조정하여 피해자와 호송 요원의 탈출 경로를 만든다.

- 변수 없는 기본 보드의 최단 횟수: `N`
- 정밀 해결 기준: `N+2`
- 위험도 변수: 목적지 혼선, 예고된 경로 흔들림
- 괴이 고착 변수: 노선 고착, 해제 장치 필요
- 마지막 단서: **안전 노선 검증 기록**
- 물리 증거: 잔향이 각인된 검은 승차권

잘못된 목적지 또는 순환 노선 확정은 위험 사례를 만든다. 단순 횟수 초과는 정상 완료다.

## 10. 데이터 계약 초안

구현 전 실제 데이터 로더와 저장 구조를 확인한다. 개념상 필요한 값은 다음과 같다.

```json
{
  "minigame_role": "final_clue_validation",
  "base_optimal_moves": 8,
  "precision_move_allowance": 2,
  "board_id": "afterlife_route_board_01",
  "variant_seed": 0,
  "active_variants": ["destination_noise", "locked_track"],
  "last_clue_id": "clue_safe_route_verification",
  "precision_reward_id": "record_afterlife_precision_route",
  "restart_policy": "restart_from_minigame_entry"
}
```

필드명과 저장 위치는 구현 단계에서 기존 스키마와 비교해 확정한다.

## 11. 검증 기준

- 변수 없는 보드의 실제 최단 횟수 `N`이 탐색 또는 전수 검증으로 확인된다.
- 모든 허용 변수 조합의 실제 최단 횟수가 `N+2` 이하다.
- 단서 열람·무전 재확인은 조작 횟수를 소비하지 않는다.
- `N+2` 초과 완료도 마지막 단서와 회수 진입을 보장한다.
- 잘못된 규칙 적용과 단순 비효율이 서로 다른 결과로 기록된다.
- 미니게임 도중 저장 UI가 비활성화되거나 저장 불가가 명확하다.
- 강제 종료와 불러오기 후 같은 보드·같은 변수로 미니게임 처음부터 시작한다.
- 종료·불러오기를 반복해 변수나 보상을 재추첨할 수 없다.
- 마지막 단서가 회수 페이즈의 판단 근거로 실제 사용된다.
