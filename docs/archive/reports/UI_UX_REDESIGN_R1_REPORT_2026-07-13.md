# 회수·대화 UI/UX 리디자인 1차 구현 기록

- 날짜: 2026-07-13
- 표시 버전: `Ver 3.1` / `MVP-031`
- 기준: Issue #31, 사용자 참고 이미지, DeepSeek R0 조사 `ses_0a77e31aaffeNr7tjGJbzfQBKf`
- 구현 보조: DeepSeek `builder-flash` 세션 `ses_0a76ec18affe4naZwZTA1GYyTg`; 초안 시간 초과 후 Codex가 실제 diff를 폐기하고 안전한 최소 변경으로 재구현

## 변경 전·후

| 화면 | 변경 전 | 변경 후 |
|---|---|---|
| 회수·안정화 | 전체 ScrollContainer와 다수 설명 패널이 배경을 가림 | 중앙 괴이·좌측 대표 요원·우측 핵심 단서·하단 행동 오버레이로 재배치 |
| 대화 | 무대와 요원 정보가 여러 패널로 분리됨 | 배경과 2~3명 스탠딩을 우선하고 하단에 VN 대사창·선택·보조 조작을 배치 |
| 상세 정보 | 단서·지원 정보가 항상 펼쳐짐 | 회수 상세와 기록국 지원 정보를 필요할 때만 여는 보조 영역으로 이동 |
| 저장·규칙 | 기존 회수 규칙과 런타임 편집기 키 사용 | 게임 값·저장·콘텐츠 키·런타임 편집기 scene id를 변경하지 않음 |

## 벤치마크 적용

- `PARANORMASIGHT`: 배경과 인물 우선의 현대 오컬트 대화 위계
- `The Case of the Golden Idol`: 핵심 단서와 상세 추론 정보의 단계적 노출
- `Persona 5 Royal`: 즉시 구분되는 명령 선택 리듬만 재해석
- 타 게임의 고유 에셋, 문구, 전투 수치, UI 외형은 복제하지 않았다.

## 검증과 남은 위험

- Godot 4.7 headless 프로젝트 파싱과 회수·대화·조사 씬 단독 실행을 검사한다.
- 1280×720 실제 창에서 회수와 대화 화면의 배경 가시성, 첫 시선, 행동 접근성을 확인했다.
- 직접 씬 실행은 선택 요원이 없는 상태라 2명/3명 스탠딩과 대표 요원 교체의 실제 화면은 전체 플레이 수동 QA가 필요하다.
- 1920×1080, 전체화면, 저승역·빨간 우산 전체 흐름, 저장·이어하기는 후속 수동 QA 항목이다.

## Base 전·후 제안

이번 작업에서는 Base 공용 저장소를 변경하지 않았다. 승격 후보는 다음과 같다.

| 전 | 후 후보 |
|---|---|
| UI 작업 완료 기준이 실행 성공 중심 | 화면 캡처에서 배경 가시성·첫 시선·행동 도달성을 함께 판정 |
| 런타임 UI 저장 호환을 일반 위험으로만 기록 | scene id·content key 유지 또는 마이그레이션 여부를 작업 계약 필수 항목으로 기록 |
