# 이미지 자산 제작 워크플로

## 역할과 단계

DeepSeek는 기존 자산의 경로, 크기, 사용 장면, 스타일 불변 조건을 `IMAGE_BRIEF.md`로 정리한다. Codex는 이를 짧은 생성 프롬프트로 정규화하고 이미지 모델을 호출한다.

1. 대표 콘셉트 2~3개만 생성한다.
2. 사용자가 대표 콘셉트 하나를 선택한다.
3. 선택 이미지를 참조해 표정·상태·크기 변형을 만든다.
4. 한 차례의 표적 수정 뒤 Codex가 기술·시각 QA를 수행한다.

투명 컷아웃은 기본적으로 크로마키 생성 후 로컬 제거를 사용한다. 복잡한 반투명 소재만 사용자 승인 뒤 별도 투명 출력 경로를 사용한다. 기존 자산은 명시적인 교체 승인 없이는 덮어쓰지 않는다.

## `ASSET_MANIFEST.json`

manifest에는 단계(`concept`/`final`), 선택 콘셉트, 최종 프롬프트, 용도, 파일명, 크기, 알파 여부, 참조 이미지, 교체 승인, QA 결과를 기록한다. Codex는 다음을 확인한다.

- PNG 경로가 작업 묶음 밖으로 나가지 않는가
- 해상도, 알파, 여백, 파일명이 규칙에 맞는가
- 최종 변형이 선택 콘셉트를 참조하는가
- 기존 파일 덮어쓰기가 승인되었는가
- Godot import가 성공하고 16:9 화면에서 UI와 플레이 요소를 가리지 않는가
- 기존 프로젝트 스타일과 일치하는가

이번 워크플로 변경에서는 계약과 fixture 검증만 구현하며 실제 게임 자산은 해당 게임 작업이 시작될 때 생성한다.
