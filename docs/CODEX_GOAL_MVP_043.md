## 1. 한 문장 목표

저승역 사건을 반사 신경형 주파수 판정에서 **단서로 안전 목적지를 판별하고, 3인 고정 작전으로 뒤틀린 노선을 최소 조작 횟수 안에 복원하는 조사형 퍼즐**로 전환한다.

최종 사건 흐름은 다음과 같다.

```text
괴이 조사 시작
→ 조사 페이즈
→ 사건 고유 미니게임
→ 마지막 단서 확보
→ 회수 페이즈
→ 안정화 상태
→ 잔향 회수
→ 괴이 매뉴얼 갱신
```

미니게임은 별도 오락이 아니라 조사에서 세운 가설을 행동으로 검증하는 마지막 조사 단계다.

## 3. 프로젝트 핵심 불변 조건

### 공식 용어

- 기관: `괴이 기록국`
- 사건 완료: `안정화 상태`
- 실패 학습: `위험 사례`
- 회수 대상: `잔향`
- 최종 기록 보상: `괴이 매뉴얼 작성·갱신`

### 세계관 원칙

괴이는 인간의 공포, 후회, 집착, 죄책감, 기억과 소문에서 발생한다. 인간의 감정과 기억이 남아 있는 한 완전히 죽지 않는다.

괴이 기록국의 목적은 괴이를 처치하는 것이 아니다.

```text
현상을 조사한다.
→ 반복 규칙을 확인한다.
→ 현재 출현을 안정화한다.
→ 잔향과 증거를 회수한다.
→ 다음 피해를 막을 대응 절차를 기록한다.
```

요원은 플레이어에게 근거와 관찰을 제공하지만 최종 정답이나 대응을 대신 선택하지 않는다.

## 4. 저승역 승인 시나리오

### 정서적 발생 원인

- 막차를 놓친 뒤 마지막 기회를 잃었다는 후회
- 반드시 원래 자리로 돌아가야 한다는 귀환 강박

### 괴이 규칙

- 비정상 안내방송의 목적지 구간은 실제로 비어 있다.
- 방송을 들은 사람은 그 공백을 자신이 돌아가고 싶은 장소로 채워 듣는다.
- 같은 방송을 들었어도 사람마다 기억하는 목적지가 다르다.
- 안내방송이 끝나기 전에 이동하면 시간은 흐르지만 공간 위치가 출발점으로 초기화된다.
- 개인이 인식한 목적지를 공식 경로로 적용하면 폐쇄 순환 또는 같은 승강장 복귀가 발생한다.

### 안정화 원리

- 방송이 끝날 때까지 기다린다.
- 목적지 구간이 공백임을 확인한다.
- 공식 운행 기록과 공식 식별음을 기준으로 안전 경로를 판별한다.
- 공백에 공식 절차를 적용해 개인별 목적지 투영을 붕괴시킨다.
- 현실 노선과 출구를 고정해 현재 출현을 `안정화 상태`로 만든다.

### 회수 대상

- `저승역 반복 안내 잔향`

### 마지막 단서

- `안전 노선 검증 기록`

기본 기록 의미:

> 공식 운행 기록과 일치하는 노선은 공간 초기화 없이 현실 승강장으로 연결되었다. 개인이 인식한 목적지는 실제 방송 원본에 존재하지 않았다.

## 5. 저승역 3인 고정 작전

저승역은 편성 선택 없이 아래 세 요원을 고정 투입한다.

| 요원 | 작전 위치 | 역할 | 정답 공개 제한 |
|---|---|---|---|
| 오현 | 방송실·공식 기록 단말기 | 목적지 공백, 공식 운행 기록, 식별 번호와 안내 종료 시점 무전 | 안전 목적지를 직접 선택하지 않는다. |
| 권나래 | 피해자와 노선 내부 | 피해자 호송, 현재 위치, 현장 표지, 피해자 상태 보고 | 피해자의 기억을 객관 사실로 확정하지 않는다. |
| 강이준 | 현실 승강장 경계 | 현실 출구 유지, 선택 노선 고정, 고착 구간 해제 보조 | 플레이어 대신 타일을 조작하지 않는다. |

기본 무전 리듬:

```text
오현의 객관 기록
→ 권나래의 현장 관찰
→ 피해자의 개인적 목적지 주장
→ 플레이어의 노선 조작
→ 강이준의 경계 고정
→ 현장의 즉시 반응
```

현재 PoC는 세 역할을 문구로만 표현한다. 실제 편성을 세 명으로 강제하는 로직, 컷인, 초상화, 음성 연출은 아직 구현되지 않았다.

## 6. 미니게임 공통 계약

### 시나리오 구조

- 시나리오당 대표 미니게임은 1개다.
- 미니게임은 조사 페이즈 끝에 배치한다.
- 결과가 마지막 단서가 된다.
- 미니게임 완료 뒤 별도의 회수 페이즈로 진행한다.
- 미니게임 성공이 사건 안정화를 대체하지 않는다.

### 조작 횟수

횟수에 포함:

- 타일 90도 회전
- 분기 방향 전환
- 고착 해제 장치 조작
- 노선 구조를 실제로 바꾸는 사건 고유 행동

횟수에서 제외:

- 단서 확인
- 무전 다시 보기
- 타일 선택만 하기
- 확대·축소
- 경로 미리보기
- 이동 또는 경로 확정 버튼

되돌리기는 기존 횟수를 취소하지 않는다. `처음부터 다시`는 현재 시도의 보드와 조작 횟수만 초기화한다.

### 최적·정밀 기준

변수 없는 기본 보드의 실제 최단 조작 횟수를 `N`으로 둔다.

```text
기본 최적 횟수: N
정밀 해결 기준: N + 2
```

| 결과 | 조건 | 진행 |
|---|---|---|
| 최적 해결 | `N` 이하 | 정상 진행, 최적 기록과 특별 반응 |
| 정밀 해결 | `N+2` 이하 | 정상 진행, 정밀 해결 보상 |
| 일반 해결 | `N+2` 초과 | 정상 진행, 추가 보상 없음 |
| 위험 사례 | 잘못된 목적지·규칙·경로 확정 | 현장 결과를 기록하고 재조정 |

횟수 초과는 실패가 아니다. 핵심 이야기, 마지막 단서, 회수 진입은 일반 해결에도 보장한다.

### 정밀 해결 보상 방향

허용:

- 정밀 해결 기록
- 잔향 기록 품질 또는 보고서 완성도 상승
- 피해자 후유증 완화
- 회수 페이즈 부담 소폭 감소
- 전용 요원 반응

금지:

- 회수 정답 자동 공개
- 필수 단서 독점
- 핵심 이야기 독점
- 다음 사건 정답 힌트
- 장기 밸런스를 크게 흔드는 영구 능력치

현재 PoC는 `최적/정밀/일반` 등급 문구만 저장한다. 실제 보상 지급은 아직 구현되지 않았다.

## 7. 위험도·괴이 고착도 변수

### 위험도

정보 교란과 즉시 변화를 만든다.

- 가짜 목적지 또는 가짜 표식
- 미리보기 노이즈
- 충돌하는 무전·표시
- 예고된 경로 흔들림

### 괴이 고착도

기존 내부 `anomaly_stability`를 플레이어에게는 `괴이 고착도`로 표현한다. 값이 높을수록 괴이가 공간과 규칙에 강하게 박혀 있다.

- 타일 또는 장치 고착
- 우회 해제 조작
- 정상 경로 차단
- 구조 재고정 저항

### 공정성 규칙

- 변수는 퍼즐 시작 전에 이름과 예상 영향을 예고한다.
- 무예고 랜덤으로 완성 경로를 파괴하지 않는다.
- 변수 조합이 강제로 요구하는 추가 조작은 최대 2회다.
- 변수 적용 후의 실제 최단 해법이 `N+2`를 넘는 보드는 생성하지 않는다.
- 종료나 불러오기로 보드 또는 변수 조합을 다시 뽑을 수 없어야 한다.

현재 3×3 PoC에는 변수가 없다. 변수는 후속 4×4 보드에서 구현한다.

## 8. 승인 보드

### 3×3 조작 학습 보드

- 역할: 노선 회전, 분기 전환, `표지보다 공식 기록을 신뢰한다`는 원칙 학습
- 변수: 없음
- 실제 최단 `N`: 4회
- 정밀 기준: 6회
- 가짜 목적지 확정 시 위험 사례 표시

최단 조작:

```text
분기 스위치: 서↔동 → 서↔북       1회
중앙 곡선:   북·동 → 동·남         1회
우측 곡선:   동·남 → 서·북         2회
합계                                  4회
```

### 4×4 마지막 단서 보드

- 역할: 조사 단서, 3인 무전, 위험도·고착도 변수를 종합하는 최종 검증
- 변수 없는 실제 최단 `N`: 8회
- 정밀 기준: 10회
- 완료 결과: `안전 노선 검증 기록`

승인 변수:

| 변수 | 입력 | 추가 필수 조작 | 효과 |
|---|---|---:|---|
| 목적지 혼선 | 위험도 | 0 | 안전·가짜 목적지 표시가 유사해 공식 기록으로 판별 |
| 노선 고착 | 괴이 고착도 | +1 | 잠긴 곡선 타일을 강이준의 해제 장치로 먼저 해제 |
| 경로 흔들림 | 높은 위험도+고착도 | +1 | 예고된 직선 타일이 최초 완전 연결 후 한 번 뒤틀림 |

고착과 흔들림이 함께 발생해도 실제 최단은 10회로 정밀 기준 안에 있어야 한다.

### 최신 흐름에 따른 정리

과거 보드 문서에는 `3×3 학습 보드 → 본 조사 → 4×4 보드`처럼 읽히는 구간이 있다. 이후 사용자 승인으로 **미니게임은 조사 페이즈 마지막의 단일 대표 미니게임**으로 확정되었다.

따라서 최종 구현에서는 다음 중 하나로 정리해야 한다.

```text
조사 페이즈
→ 노선 복원 미니게임 진입
→ 내부 3×3 조작 학습 단계
→ 내부 4×4 최종 검증 단계
→ 마지막 단서
→ 회수 페이즈
```

또는 온보딩 검증 후 3×3을 제거하고 4×4만 사용한다.

3×3을 별도의 중간 미니게임 완료로 처리한 뒤 다시 조사로 돌아가는 구조는 최신 승인 흐름과 맞지 않는다. 현재 3×3 구현은 **조작·연결·저장 경계를 확인하기 위한 PoC**로 본다.

## 9. 현재 `main` 구현 상태

### 구현 파일

- `scripts/minigames/route_restore_game.gd`
- `scripts/scenes/minigame_scene.gd`

### 현재 동작

- 기존 저승역 미니게임 ID `minigame_frequency_sync`를 씬 런타임에서 `route_restore` 타입으로 덮어쓴다.
- 보호 경로 `data/episodes/episode_001_afterlife_station.json`은 수정하지 않았다.
- 저승역 기존 미니게임 진입 포인트에서 3×3 노선 복원 PoC가 열린다.
- 곡선 회전, 스위치 전환, 경로 확인, 초기화를 지원한다.
- 마우스와 키보드를 지원한다.
- 안전 목적지 연결 시 항상 정상 완료한다.
- 완료 횟수에 따라 `최적 복원`, `정밀 복원`, `일반 복원`을 기록한다.
- 가짜 목적지 확정 시 위험 사례를 보여주고 보드와 횟수를 유지한다.
- 끊긴 경로는 조작 실수로 안내하며 위험 사례로 등록하지 않는다.
- 빨간 우산 `rain_dodge`와 다른 미니게임 타입은 기존 분기를 유지한다.

### 현재 저장 경계

- 조사 씬이 미니게임 씬으로 이동하기 전에 기존 사건 상태와 현재 씬 경로를 저장한다.
- 미니게임 진행 중 타일 상태와 횟수는 저장하지 않는다.
- 미니게임 화면에서 저장을 유발하던 장면 이동 버튼을 제거했다.
- 정상 완료 시 기존 `save_minigame_result()` 흐름으로 결과를 저장한다.

## 11. Codex 권장 작업 순서

### 1단계 — 현재 PoC 검증과 차단 오류 수정

먼저 실제 `main` 상태를 검증한다.

1. `git status`와 최신 커밋을 확인한다.
2. 위 필수 파일을 읽고 사용자 변경을 보존한다.
3. `godot --headless --path . --quit` 또는 프로젝트 기준 headless 명령을 실행한다.
4. `scenes/minigame_scene.tscn`과 저승역 조사 진입 경로를 실행한다.
5. 3×3 보드의 4회 최적 해법을 실제 입력으로 확인한다.
6. 5~6회 정밀, 7회 이상 일반 결과를 확인한다.
7. 초기 상태에서 가짜 목적지를 확정해 위험 사례와 보드 유지 여부를 확인한다.
8. 끊긴 경로가 위험 사례가 아니라 조작 안내로 처리되는지 확인한다.
9. 미니게임 중 종료 후 이어하기에서 초기 보드·0회로 돌아오는지 확인한다.
10. 완료 후 조사 복귀와 기존 회수 진입이 막히지 않는지 확인한다.
11. 빨간 우산 `rain_dodge`가 깨지지 않았는지 회귀 확인한다.
12. 보고서·DB에 저승역 결과가 어떤 제목과 내용으로 저장되는지 확인한다.

이 단계에서는 실행을 막는 오류, 입력 불가, 저장 중복, 다른 사건 회귀만 수정한다. 승인되지 않은 대규모 구조 변경은 하지 않는다.

### 2단계 — 코드 영향 분석 보고

PoC 검증 후 다음 통합에 필요한 실제 파일과 위험을 보고한다.

- 3×3을 4×4 앞의 내부 튜토리얼 단계로 연결하는 방법
- 4×4 보드 데이터를 코드 상수, 리소스, JSON 중 어디에 둘지
- 위험도·괴이 고착도 임계값을 현재 상태 값과 연결하는 방법
- 변수 조합을 종료·불러오기에도 고정하는 방법
- 마지막 단서와 위험 사례를 현재 저장 스키마 안에서 표현할 수 있는지
- 기존 `minigame_frequency_sync` ID를 유지하면서 플레이어 노출 의미를 이관하는 방법
- 결과 화면, 사건 보고서, DB가 새 제목·등급·위험 사례를 동일하게 표시하는 방법
- 보호 JSON 변경이 필요한 최소 범위

새 저장 필드가 필요하다면 바로 추가하지 말고 기존 구조로 가능한 대안과 마이그레이션 영향을 먼저 제시한다.

### 3단계 — 최소 end-to-end 통합

사용자 승인 후 다음 순서로 좁게 구현한다.

```text
조사 단서 조건 충족
→ 저승역 노선 복원 진입
→ 내부 3×3 학습 또는 짧은 안내
→ 4×4 최종 보드
→ 위험도·고착도 변수 적용
→ 마지막 단서 `안전 노선 검증 기록`
→ 조사 복귀가 아니라 회수 페이즈 진입
```

이 단계에서도 저승역 외 사건과 전체 저장 시스템을 재설계하지 않는다.

### 4단계 — 콘텐츠·문서 동기화

런타임 흐름이 검증된 뒤에만 보호 데이터와 공식 문서를 갱신한다.

- `data/episodes/episode_001_afterlife_station.json`
- `docs/GAME_DESIGN_DOCUMENT.md`
- `docs/URBAN_LEGEND_GAME_DESIGN.docx` 생성기 `--build`, `--check`
- `README.md`
- `MVP_ROADMAP.md`
- `TEST_CHECKLIST.md`
- `docs/PROJECT_CONTEXT.md`

GDD를 수정하면 DOCX를 직접 편집하지 말고 생성기를 사용한다.

## 12. 1단계 완료 기준

- [ ] Godot headless 프로젝트 로드가 오류 없이 끝난다.

- [ ] 가짜 목적지 확정이 위험 사례를 표시한다.
- [ ] 위험 사례 뒤 현재 보드와 조작 횟수가 유지된다.

- [ ] 미니게임 중 저장 버튼이나 디버그 장면 이동 수단이 노출되지 않는다.
- [ ] 완료 전 종료·불러오기 후 초기 보드·0회로 복구된다.
- [ ] 완료 결과가 한 번만 저장되고 중복 효과가 적용되지 않는다.
- [ ] 완료 후 기존 조사·회수 흐름으로 돌아갈 수 있다.
- [ ] 빨간 우산 미니게임에 회귀가 없다.
- [ ] 미검증 항목과 남은 위험을 명시한다.
