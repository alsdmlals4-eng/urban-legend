# Codex 통합 제안서 - MVP-043 괴이 기록국 콘텐츠 재구성 및 저승역 end-to-end PoC

## 문서 상태

- 목적: 이 대화에서 확정된 **총기획, 콘텐츠 재구성 원칙, 첫 10분 PoC 명세, 미니게임 공통 규칙, 저승역 노선 복원 구현 방향**을 Codex에게 하나의 작업 기준으로 전달한다.
- 상태: **사용자 승인 통합 제안서 / 현재 구현과 목표 설계를 함께 기록한 후속 구현 기준 문서**
- 연결 Issue: #36 `MVP-043 콘텐츠 방향 v0.9 통합 및 저승역 튜토리얼 PoC`
- 기준일: 2026-07-14
- 현재 구현 기준: `main`

이 문서는 단순한 3×3 퍼즐 구현 지시서가 아니다. 프로젝트의 전체 콘텐츠 방향을 먼저 정의하고, 그 방향을 검증하는 첫 번째 end-to-end 사례로 저승역 PoC를 사용한다.

Codex는 반드시 아래 세 층을 구분한다.

```text
총기획과 사용자 승인 방향
→ 콘텐츠 재구성 목표
→ 현재 main의 PoC 구현 사실
```

현재 코드에 존재한다고 해서 최종 설계로 간주하지 않는다. 반대로 승인 설계에 적혀 있다고 해서 구현 완료로 보고하지 않는다.

# 1. 프로젝트 총기획

## 1.1 게임 핵심 한 문장

> **인간의 마음에서 되살아나는 괴이의 반복 규칙을 조사하고, 현재 출현을 안정화한 뒤 다음 피해를 막을 괴이 매뉴얼을 완성하는 현대 오컬트 조사 어드벤처.**

플레이어 경험 중심 표현:

> **단서를 연결해 괴이의 규칙을 알아내고, 내가 검증한 대응법이 공식 기록으로 남는 게임.**

## 1.2 세계관 핵심 명제

괴이는 인간의 공포, 후회, 집착, 죄책감, 기억과 소문에서 발생한다. 사람의 마음과 이야기가 남아 있는 한 괴이는 완전히 죽지 않으며, 조건이 다시 갖춰지면 다른 장소와 형태로 재출현할 수 있다.

괴이 기록국의 목적은 괴이를 처치하거나 영구 제거하는 것이 아니다.

```text
현상을 조사한다.
→ 출현 조건과 반복 규칙을 확인한다.
→ 현재 출현을 안정화한다.
→ 규칙을 입증하는 잔향과 증거를 회수한다.
→ 전조·금기·위험 사례·검증된 대응을 괴이 매뉴얼로 기록한다.
→ 재출현 때 더 적은 피해로 더 빠르게 대응한다.
```

기관의 성과 기준은 다음 질문이다.

> 같은 괴이가 다시 나타났을 때, 더 적은 피해로 더 빠르게 대응할 수 있는가?

## 1.3 공식 용어

| 기존 표현 | 공식 표현 | 의미 |
|---|---|---|
| 괴담기록국 | **괴이 기록국** | 플레이어 노출 공식 기관명 |
| 사건 해결·종료 | **안정화 상태** | 현재 출현이 통제된 상태 |
| 실패 기록·패턴 학습 | **위험 사례** | 오대응과 관측 결과를 다음 판단 근거로 남긴 기록 |
| 괴이 핵·매개체 | **잔향** | 괴이의 규칙을 입증하는 회수 증거 |
| 사건 보고서 | **괴이 매뉴얼 작성·갱신** | 플레이어가 완성하는 최종 지식 보상 |

내부 파일명, 저장 키, 구 경로는 호환성을 위해 유지할 수 있다. UI, 대사, GDD, 결과 화면은 최종적으로 공식 용어에 맞춘다.

## 1.4 핵심 재미

이 게임의 보상은 전리품 폭발이나 반복 점수 팝업보다 **인지적 보상**에서 나온다.

```text
불길함
→ 관찰
→ 의심
→ 가설
→ 결단
→ 현장의 즉시 반응
→ 이해와 안도
→ 괴이 매뉴얼 완성
```

반드시 제공할 보상:

- 예상이 맞았다는 즉각적인 현장 변화
- 서로 다른 기록이 하나의 규칙으로 연결되는 순간
- 선택 직후 대사, 음향, 배경, 상태 기록이 바뀌는 피드백
- 미확인 항목이 추정 또는 현장 검증으로 바뀌는 기록 변화
- 요원의 짧은 인정과 보조 관찰
- 실패가 위험 사례로 전환되어 다음 판단에 실제로 도움이 되는 경험
- 플레이어가 직접 알아낸 대응법이 매뉴얼에 등록되는 소유감

피해야 할 것:

- 의미 없는 재화와 점수 팝업
- 장르와 맞지 않는 과도한 승리 연출
- 정답을 자동으로 공개하는 장비 또는 연구
- 첫 10분에 운영·경제·신뢰도·세력 설명을 몰아넣는 구성
- 실패를 시간 손실과 강제 재시작으로만 처리하는 구성
- 괴이를 HP, 공격, 처치, 사망 중심으로 해석하는 확장

# 2. 전체 게임 흐름과 튜토리얼 목표

## 2.1 시나리오 공통 흐름

모든 시나리오는 아래 흐름을 기본 계약으로 사용한다.

```text
괴이 조사 시작
→ 조사 페이즈
→ 사건 고유 미니게임
→ 마지막 단서 확보
→ 회수 페이즈
→ 안정화 상태
→ 잔향 회수
→ 괴이 매뉴얼 작성·갱신
```

각 단계의 질문:

| 구간 | 핵심 질문 |
|---|---|
| 조사 페이즈 | 이 괴이는 어떤 규칙으로 작동하는가? |
| 미니게임 | 조사에서 세운 가설이 실제 현장에서 통하는가? |
| 마지막 단서 | 현장 검증으로 무엇이 확인되었는가? |
| 회수 페이즈 | 확인한 규칙으로 현재 출현을 어떻게 안정화할 것인가? |
| 괴이 매뉴얼 | 다음 출현 때 무엇을 보고 어떻게 대응해야 하는가? |

미니게임과 회수 페이즈는 같은 일을 반복하면 안 된다.

- 미니게임: 가설의 현장 검증
- 회수 페이즈: 검증된 규칙을 이용한 최종 안정화

## 2.2 신규 캠페인 목표 흐름

현재 구현:

```text
신규 캠페인
→ 반일 계획
→ 조사·휴식·의뢰 배치
→ 사건 선택
→ 현장 조사
```

승인된 목표:

```text
신규 캠페인
→ 2~3분 오프닝
→ 저승역 3인 고정 투입
→ 조사·단서·위험 사례·미니게임·회수 튜토리얼
→ 저승역 안정화 상태
→ 저승역 반복 안내 잔향 회수
→ 괴이 매뉴얼 초안 완성
→ HQ 복귀
→ 첫 자유 반일 계획
```

첫 사건이 끝나기 전에는 다음을 설명하지 않는다.

- 세력 관계 단계
- 시장과 가격
- 연구 트리
- 10일 종료 계산
- 소모품 제한
- 모든 요원 능력치
- 신뢰도 세부 수치
- 편성 최적화

## 2.3 오프닝

2~3분 안에 다음만 전달한다.

1. 괴이는 인간의 마음과 기억에서 태어난다.
2. 괴이는 완전히 죽지 않는다.
3. 괴이 기록국은 현재 출현을 안정화하고 대응 절차를 기록한다.
4. 플레이어는 요원의 관찰과 기록을 읽고 최종 판단을 내린다.

승인 문구:

> 사람의 마음과 기억이 남아 있는 한,
> 언젠가 다시 나타난다.
>
> 그렇기에 괴이 기록국은 괴이를 없애는 기관이 아니다.
> 현상을 조사하고, 현재의 출현을 안정화하며,
> 다음 피해를 막을 대응 절차를 기록해 인류를 수호하는 집단이다.

오프닝 끝에서 저승역 긴급 보고로 전환한다.

# 3. 첫 10분 PoC 총명세

이 PoC의 목적은 퍼즐 하나를 만드는 것이 아니다. 플레이어가 첫 10분 안에 게임의 핵심 동사를 한 번씩 경험하도록 만드는 것이다.

## 3.1 시간 계약

| 시간 | 장면 | 플레이어 행동 | 즉시 보상 |
|---|---|---|---|
| 0:00~1:00 | 괴이 콜드 오픈 | 반복되는 이상 현상 관찰 | 세계관 갈고리 |
| 1:00~2:30 | 괴이 기록국 소개·긴급 보고 | 기관 목적과 플레이어 역할 확인 | 목표 이해 |
| 2:30~5:00 | 저승역 첫 조사 | 의미가 다른 조사 선택 | 첫 단서와 요원 반응 |
| 5:00~7:00 | 기록 비교 | 두 기록의 공통점·모순 연결 | 첫 추론 보상 |
| 7:00~9:00 | 가설 검증 | 정보에 근거한 행동 선택 또는 노선 판단 | 현장의 즉시 변화 |
| 9:00~10:00 | 결과 기록 | 성공 또는 위험 사례 확인 | 매뉴얼 항목 변화와 다음 질문 |

## 3.2 필수 경험

첫 10분 안에 다음이 모두 발생해야 한다.

- 시작 후 3분 안에 의미 있는 선택 1회
- 단서 2개 이상 확보
- 서로 다른 기록의 공통점 또는 모순 연결
- provisional hypothesis 또는 같은 역할의 가설 선택
- 행동으로 가설을 검증
- 성공 또는 위험 사례에 따른 즉시 현장 반응
- 기록 상태가 `미확인 → 추정 → 현장 검증` 중 하나로 변화
- 요원이 정답을 말하지 않고 플레이어의 판단을 인정

## 3.3 첫 의미 있는 선택

첫 조사 선택은 단순한 앞/뒤 이동이 아니라 서로 다른 종류의 근거를 선택하게 한다.

추천 구조:

```text
A. 공식 운행 기록을 먼저 확인한다.
B. 비정상 안내방송 원본을 먼저 확인한다.
```

두 선택은 진행을 막지 않으며, 먼저 얻는 관점과 요원 반응이 다르다. 이후 반대편 기록도 확보해 비교하게 한다.

## 3.4 성공과 실패

실패는 진행 차단이 아니다.

```text
올바른 가설 검증
→ 현장 규칙 확인
→ 더 명확한 마지막 단서
→ 유리한 회수 진입

잘못된 가설 또는 행동
→ 위험 사례 등록
→ 무엇을 비교해야 하는지 새 근거 제공
→ 불리하지만 진행 가능한 상태
```

강제 실패 튜토리얼은 사용하지 않는다.

# 4. 콘텐츠 재구성 원칙

## 4.1 재구성의 목적

현재 저승역은 구조적으로 재사용 가능한 요소가 많지만, 사건의 중심 인과가 승인 설계와 다르다.

현재 중심:

```text
존재하지 않는 종착지
+ 검은 승차권 핵
+ 피해자의 이름·숫자 폐주파수
+ 반사 신경형 리듬 판정
```

목표 중심:

```text
목적지 구간의 공백
+ 듣는 사람이 채우는 개인별 목적지
+ 안내 종료 전 이동과 공간 초기화
+ 공식 운행 기록을 이용한 안전 노선 복원
```

따라서 전체 구조를 버리는 방식이 아니라 **기존 ID, 저장 키, 노드, 화면 흐름을 최대한 보존하면서 플레이어가 읽는 의미와 사건 인과를 재작성**한다.

## 4.2 콘텐츠 분류 방식

모든 기존 요소를 다음 중 하나로 분류한다.

| 분류 | 의미 |
|---|---|
| 유지 | 구조와 의미를 그대로 사용 |
| 문구 수정 | 역할은 유지하고 플레이어 노출 표현만 변경 |
| 역할 변경 | 내부 ID는 유지하지만 사건 안에서 기능을 바꿈 |
| 교체 | 기존 기능이 목표 경험과 맞지 않아 새 내용으로 대체 |
| 삭제 후보 | 중복되거나 잘못된 인과를 강화해 제거 검토 |

## 4.3 보호 원칙

- 기존 저장 키와 내부 ID를 가능한 한 유지한다.
- `data/episodes/*`는 실제 코드 영향 분석과 사용자 승인 없이 대규모 수정하지 않는다.
- 현재 저장 스키마로 가능한 방법을 먼저 찾는다.
- 새 필드가 필요하면 대안, 마이그레이션, 구 저장 호환 영향을 먼저 보고한다.
- 기존 빨간 우산, 폐주파수 방송국 흐름을 임의로 재작성하지 않는다.
- 현재 구현된 사용자 변경을 되돌리지 않는다.

## 4.4 단서·힌트·위험 사례의 분리

- 단서: 객관적으로 확보한 사건 근거
- 힌트: 어떤 기록을 비교해야 하는지 알려주는 방향
- 위험 사례: 잘못된 행동에서 확인된 규칙

같은 내용을 세 범주에 중복 등록하지 않는다.

예시:

```text
단서: 녹음 원본의 목적지 구간은 비어 있다.
힌트: 사람들이 기억한 목적지보다 음성이 끊기는 위치를 비교한다.
위험 사례: 안내 종료 전에 움직이면 시간은 흐르고 위치만 초기화된다.
```

# 5. 저승역 콘텐츠 총설계

## 5.1 정체성

한 줄 약속:

> **막차를 놓친 사람에게 다시 돌아갈 기회를 들려주지만, 그 안내를 따르면 영원히 같은 승강장으로 돌아오는 괴이현상.**

발생 감정:

- 마지막 기회를 놓쳤다는 후회
- 반드시 돌아가야 한다는 귀환 강박
- 돌아갈 곳을 잃을지 모른다는 공포
- 시간을 되돌리고 싶다는 미련
- 조금만 빨랐다면 달라졌을 것이라는 자책

괴이의 정체:

> **돌아갈 마지막 기회를 놓치고 싶지 않은 마음이 안내방송이라는 규칙으로 나타난 괴이.**

## 5.2 출현 조건

1. 정규 막차 운행이 종료되었다.
2. 역 또는 승강장에 고립된 사람이 있다.
3. 그 사람이 특정 장소로 돌아가야 한다는 강한 감정을 품고 있다.
4. 실제 운행 기록에 없는 마지막 안내방송이 시작된다.

## 5.3 핵심 규칙

- 안내방송의 목적지 구간은 실제 녹음에서 비어 있다.
- 방송을 들은 사람은 그 공백을 자신이 가장 돌아가고 싶은 장소로 채워 듣는다.
- 같은 방송을 들은 사람도 서로 다른 목적지를 기억한다.
- 자신이 들었다고 믿은 목적지를 받아들이고 안내가 끝나기 전에 이동하면 규칙이 성립한다.
- 시간은 계속 흐르지만 공간 위치만 승강장 시작점으로 초기화된다.
- 반복할수록 최근 기억과 방향감각이 흐려진다.
- 개인이 인식한 목적지를 공식 경로로 적용하면 폐쇄 순환 또는 같은 승강장 복귀가 발생한다.

플레이어 핵심 질문:

> **왜 같은 방송을 들은 사람들이 서로 다른 목적지를 기억하는가?**

질문 발전:

1. 방송이 실제 목적지를 말했는가?
2. 기록 장치가 일부 음성을 놓쳤는가?
3. 괴이가 사람마다 다른 방송을 들려주는가?
4. 듣는 사람이 방송의 빈칸을 스스로 채우는가?
5. 어떤 행동이 목적지를 받아들인 것으로 판정되는가?

## 5.4 핵심 동사

- 비교한다: 운행 기록, 방송 원본, 증언, CCTV, 행동 시점을 대조한다.
- 기다린다: 조급함에 반응하지 않고 안내와 전조를 끝까지 관찰한다.
- 재구성한다: 개인적 목적지 대신 검증된 공식 절차와 노선을 복원한다.

# 6. 저승역 3인 고정 작전

저승역은 편성 선택 없이 다음 세 명을 고정 투입한다.

| 요원 | 위치 | 역할 | 제한 |
|---|---|---|---|
| 오현 | 방송실·공식 기록 단말기 | 방송 원본, 목적지 공백, 공식 운행 기록, 식별음, 안내 종료 시점 무전 | 안전 목적지를 대신 선택하지 않는다. |
| 권나래 | 피해자와 뒤틀린 노선 내부 | 피해자 호송, 현재 위치, 현장 표지, 피해자 상태 보고 | 피해자의 기억을 객관 사실로 확정하지 않는다. |
| 강이준 | 현실 승강장 경계 | 현실 출구 유지, 선택 노선 고정, 고착 구간 해제 보조 | 플레이어 대신 타일을 조작하지 않는다. |

기본 작전 흐름:

```text
오현의 객관 기록
→ 권나래의 현장 관찰
→ 피해자의 개인적 목적지 주장
→ 플레이어의 노선 판단과 조작
→ 강이준의 경계 고정
→ 현장의 즉시 반응
```

고정 편성의 스토리 근거:

- 방송과 공식 기록을 확인할 인원 1명
- 피해자를 보호하며 내부를 이동할 인원 1명
- 현실 출구와 뒤틀린 노선 경계를 유지할 인원 1명

세 역할이 동시에 필요하므로 첫 사건에서는 3인 전원 투입이 작전상 필수다.

현재 PoC는 역할을 문구로만 표현한다. 실제 편성 강제, 컷인, 초상화, 음성 연출은 별도 구현 대상이다.

# 7. 저승역 조사 페이즈 재구성

## 7.1 승인된 근거 역할

최종 목표는 조사 페이즈에서 4개의 일반 단서를 확보하고, 미니게임 결과를 5번째이자 마지막 단서로 만드는 것이다.

### 조사 단서 1 — 피해자의 마지막 기록

- 피해자는 특정 목적지로 가는 마지막 열차를 탔다고 확신한다.
- 실제 메시지는 목적지보다 시각과 귀환 강박을 보여준다.

### 조사 단서 2 — 반복 안내방송 원본

- 목적지가 들어갈 구간에 음성 공백이 있다.
- 사람의 기억과 원본 데이터가 일치하지 않는다.

### 조사 단서 3 — 서로 다른 목적지 기록

- 피해자, 목격자, 요원, 전광판 또는 기기가 같은 방송에서 서로 다른 목적지를 기록한다.
- 괴이가 하나의 객관적 목적지를 방송한 것이 아님을 보여준다.

### 조사 단서 4 — 공식 운행 기록

- 막차 이후 추가 운행과 목적지는 존재하지 않는다.
- 정상 역명, 노선, 식별음, 운행 종료 시각을 제공한다.

### 미니게임 마지막 단서 — 안전 노선 검증 기록

> 공식 운행 기록에 맞춘 노선은 공간 초기화 없이 현실 승강장으로 연결되었다. 개인이 인식한 목적지는 실제 방송 원본에 존재하지 않았다.

## 7.2 기존 5개 단서 ID 재사용 제안

보호 데이터 수정 전 실제 파일을 확인한 뒤 아래 의미 이관을 검토한다.

| 기존 내부 ID | 목표 역할 | 처리 제안 |
|---|---|---|
| `clue_last_message` | 피해자가 믿은 목적지, 마지막 기회에 대한 후회와 귀환 강박 | 역할 재작성 |
| `clue_black_ticket` | 개인 목적지와 잔향이 각인된 물리 증거 | 필수 핵 단서에서 보조 증거로 역할 변경 |
| `clue_repeating_announcement` | 목적지 공백이 있는 방송 원본 | 핵심 단서로 재작성 |
| `clue_missing_terminal_sign` | 사람·기기마다 서로 다른 목적지 표시 | 역할 재작성 |
| `clue_staff_room_log` | 공식 막차·정상 안내·식별음 기준 | 역할 재작성 |

공간 초기화와 안내 종료 전 이동은 일반 단서 ID로 억지 저장하지 않고 위험 사례로 관리하는 안을 우선한다.

## 7.3 힌트 역할

힌트는 정답 행동을 직접 말하지 않는다.

1. 현장 안내 시각과 공식 운행표를 비교한다.
2. 방송 내용보다 음성이 끊기는 위치를 확인한다.
3. 여러 사람이 기억한 목적지를 대조한다.
4. CCTV의 시각보다 인물 위치 변화를 확인한다.
5. 공간 초기화 직전 행동과 방송 종료 시점을 함께 기록한다.

금지 문구:

- 안내가 끝날 때까지 움직이지 마십시오.
- 목적지는 존재하지 않습니다.
- 빈 구간에 정상 안내를 재생하십시오.
- 정답 노선을 선택하십시오.

# 8. 저승역 위험 사례

## 위험 사례 01 — 조기 이동

> 안내방송이 끝나기 전에 이동한 인원은 승강장 진입 지점으로 되돌아왔다. 시간 기록은 유지되었으나 공간 위치만 초기화되었다.

## 위험 사례 02 — 개인 목적지 확정

> 개인이 들었다고 주장한 목적지를 현장 사실로 적용하자 해당 노선이 폐쇄 순환으로 변했다. 다른 기록에서는 같은 목적지가 확인되지 않았다.

## 위험 사례 03 — 미검증 방송 재생

> 출처가 확인되지 않은 안내 음성을 스피커로 재생하자 반복 범위가 확대되었다. 현장 방송의 증폭은 대응 절차로 사용할 수 없다.

위험 사례는 정답을 직접 말하지 않는다. 다음에 무엇을 비교하고 어떤 행동을 피해야 하는지 근거를 남긴다.

# 9. 미니게임 공통 시스템

## 9.1 역할

시나리오당 대표 미니게임은 1개다. 미니게임은 조사 페이즈 끝의 **마지막 현장 검증**이며, 결과가 마지막 단서가 된다.

- 미니게임 전에 확보한 단서가 플레이에 도움이 되어야 한다.
- 미니게임 조작만 잘해서 사건 규칙을 알 수 있어서는 안 된다.
- 미니게임 성공은 회수 정답을 자동 공개하지 않는다.
- 미니게임 완료 뒤 별도의 회수 페이즈가 이어진다.

## 9.2 조작 횟수

횟수 포함:

- 타일 90도 회전
- 분기 방향 전환
- 고착 해제 장치 조작
- 노선 구조를 실제로 바꾸는 사건 고유 행동

횟수 제외:

- 타일 선택만 하기
- 단서 다시 보기
- 무전 다시 듣기
- 화면 이동과 확대·축소
- 경로 미리보기
- 이동 또는 결과 확정 버튼

되돌리기는 이전 횟수를 취소하지 않는다. `처음부터 다시`는 현재 시도의 보드와 횟수만 초기화한다.

## 9.3 최적·정밀 기준

변수 없는 기본 보드의 실제 최단 조작 횟수를 `N`으로 둔다.

```text
기본 최적 횟수: N
정밀 해결 기준: N + 2
```

| 결과 | 조건 | 진행 |
|---|---|---|
| 최적 해결 | `N` 이하 | 정상 진행, 최적 기록과 특별 대사 |
| 정밀 해결 | `N+2` 이하 | 정상 진행, 정밀 해결 보상 |
| 일반 해결 | `N+2` 초과 | 정상 진행, 추가 보상 없음 |
| 위험 사례 | 잘못된 목적지·규칙·경로 확정 | 관측 결과 기록 후 재조정 |

횟수 초과는 실패가 아니다. 일반 해결에도 핵심 서사, 마지막 단서, 회수 진입을 보장한다.

플레이 중에는 현재 조작 횟수와 정밀 기준만 표시한다. 실제 최단 `N`은 완료 후 공개한다.

## 9.4 정밀 해결 보상

기본 완료 보장:

- 마지막 단서
- 회수 페이즈 진입
- 괴이 매뉴얼 필수 항목
- 사건 핵심 서사

`N+2` 이내 보상:

- 정밀 해결 기록
- 잔향 기록 품질 또는 보고서 완성도 상승
- 피해자 후유증 완화
- 회수 페이즈 부담 소폭 감소
- 전용 요원 반응

`N` 이하 추가 표현:

- 최적 기록 표식
- 특별 대사
- 별도 큰 성장 보상 없음

금지 보상:

- 회수 정답 자동 공개
- 필수 단서 독점
- 핵심 이야기 독점
- 다음 사건 정답 힌트
- 장기 밸런스를 크게 흔드는 영구 능력치

# 10. 위험도·괴이 고착도 변수

## 10.1 위험도

위험도는 정보 교란과 즉시 변화형 변수를 만든다.

- 가짜 목적지 또는 가짜 표식
- 미리보기 노이즈
- 충돌하는 무전·표시
- 예고된 경로 흔들림

## 10.2 괴이 고착도

기존 내부 `anomaly_stability`는 플레이어 노출에서 **괴이 고착도**로 표현하는 안을 우선한다. 값이 높을수록 괴이가 공간과 규칙에 강하게 박혀 있다.

- 특정 타일 또는 장치 고착
- 우회 해제 조작
- 정상 경로 일부 차단
- 구조 재고정 저항

실제 현재 값의 증감 방향과 이름이 반대라면 UI 변환 또는 내부 의미 이관 방안을 먼저 보고한다.

## 10.3 변수 발생 단계

| 상태 | 변수 구성 |
|---|---|
| 위험도·고착도 모두 낮음 | 변수 없음 |
| 둘 중 하나가 높음 | 약한 변수 1개 |
| 둘 다 높음 | 약한 변수 2개 또는 구조 변수 1개 |
| 둘 다 매우 높음 | 강한 변수 1개 + 약한 변수 1개 |

공정성 규칙:

- 변수는 퍼즐 시작 전에 이름과 예상 영향을 예고한다.
- 정답 타일과 안전 목적지는 공개하지 않는다.
- 무예고 랜덤으로 완성 경로를 파괴하지 않는다.
- 변수 조합이 강제로 요구하는 추가 조작은 최대 2회다.
- 변수 적용 후의 실제 최단 해법이 `N+2`를 넘는 보드는 출현시키지 않는다.
- 종료·불러오기로 보드와 변수 조합을 다시 뽑을 수 없어야 한다.

## 10.4 조사 결과 연결

```text
위험한 조사·오대응
→ 위험도 상승
→ 목적지 혼선·경로 흔들림 강화

핵심 기록 미확보·대응 지연
→ 괴이 고착도 상승
→ 노선 고착·해제 장치 필요

충분한 근거·안전한 조사
→ 변수 억제 또는 대응 정보 확보
```

단서는 변수를 자동 삭제하기보다 정체를 예고하거나 대응 장치를 보여주는 방향을 우선한다.

# 11. 저승역 노선 복원 미니게임

## 11.1 서사 구조

- 오현은 방송실과 공식 기록을 확인하며 무전한다.
- 권나래는 피해자를 데리고 뒤틀린 노선 내부를 이동한다.
- 강이준은 현실 승강장 출구와 플레이어가 선택한 노선을 고정한다.
- 플레이어는 두 현장의 보고와 조사 단서를 종합해 안전 목적지와 이동 시점을 판단한다.

노선을 연결하는 것만으로 끝나지 않는다.

최종 조건:

1. 공식 기록으로 확인된 안전 목적지를 선택한다.
2. 폐쇄 순환하지 않는 경로를 연결한다.
3. 안내방송 종료 후 이동을 확정한다.

## 11.2 3×3 조작 학습 보드

목적:

- 타일 회전과 분기 전환 학습
- `피해자 증언보다 공식 기록을 기준으로 판단한다`는 원칙 학습
- 경로 확인, 위험 사례, 저장 경계를 검증하는 기술 PoC

기준:

- 변수 없음
- 실제 최단 `N=4`
- 정밀 기준 `6회`
- 가짜 목적지 확정 시 위험 사례
- 위험 사례 후 현재 보드와 조작 횟수 유지

검증된 최단 조작:

```text
분기 스위치: 서↔동 → 서↔북       1회
중앙 곡선:   북·동 → 동·남         1회
우측 곡선:   동·남 → 서·북         2회
합계                                  4회
```

## 11.3 4×4 마지막 단서 보드

목적:

- 조사 단서와 3인 무전을 종합
- 위험도·고착도 변수 대응
- 실제 피해자 탈출 경로 검증
- 마지막 단서 `안전 노선 검증 기록` 생성

기준:

- 변수 없는 실제 최단 `N=8`
- 정밀 기준 `10회`

승인 변수:

| 변수 | 입력 | 추가 필수 조작 | 효과 |
|---|---|---:|---|
| 목적지 혼선 | 위험도 | 0 | 안전·가짜 목적지 표기가 유사해 공식 기록으로 판별 |
| 노선 고착 | 괴이 고착도 | +1 | 잠긴 타일을 강이준의 해제 장치로 먼저 해제 |
| 경로 흔들림 | 높은 위험도+고착도 | +1 | 예고된 직선 타일이 최초 완전 연결 후 한 번 뒤틀림 |

고착과 흔들림이 함께 발생해도 실제 최단은 10회로 정밀 기준 안에 있어야 한다.

## 11.4 3×3과 4×4의 최종 관계

최신 승인 흐름에서 시나리오당 대표 미니게임은 1개다. 따라서 3×3을 조사 중간의 별도 미니게임으로 완료 처리하면 안 된다.

권장 최종 구조:

```text
조사 페이즈 완료
→ 노선 복원 미니게임 진입
→ 내부 3×3 조작 학습 단계
→ 짧은 무전 전환
→ 내부 4×4 최종 검증 단계
→ 마지막 단서 생성
→ 회수 페이즈 직행
```

대안:

- 실제 테스트에서 3×3이 불필요하면 첫 진입 안내와 4×4만 사용한다.

현재 `main`의 3×3 구현은 최종 콘텐츠가 아니라 기술 PoC다.

# 12. 미니게임 저장·종료·복구

## 12.1 확정 원칙

- 미니게임 중 수동 저장과 자동 저장을 하지 않는다.
- 미니게임 진입 직전에 복구용 체크포인트를 저장한다.
- 정상 완료 후 다음 페이즈 진입 시 다시 저장할 수 있다.

## 12.2 비정상 종료와 불러오기

게임 강제 종료, 충돌, 전원 종료, 프로세스 중단 또는 미니게임 중 불러오기가 발생하면 다음 실행에서 미니게임 시작 지점으로 돌아간다.

초기화:

- 타일·장치 방향
- 현재 조작 횟수
- 현 시도의 이동 위치
- 현 시도에만 존재하는 임시 연출

유지:

- 미니게임 진입 전 단서
- 진입 전 위험도와 괴이 고착도
- 적용 보드 ID
- 적용 변수 조합 또는 결정론적 시드
- 사건 진행 상태
- 고정 요원 편성

같은 체크포인트를 불러오면 같은 보드와 같은 변수 조합으로 시작한다. 종료·불러오기로 유리한 보드나 변수를 다시 뽑을 수 없어야 한다.

정상 종료 안내:

> 미니게임 진행 상황은 저장되지 않습니다. 다시 시작하면 이 미니게임의 처음부터 진행합니다.

초기 PoC에서는 타일별 진행 상태를 저장하지 않는다. 새 저장 필드는 실제 기존 구조로 표현할 수 없는 경우에만 별도 승인 후 검토한다.

# 13. 회수 페이즈와 안정화

미니게임에서 피해자를 탈출시켰다고 저승역 전체가 안정화된 것은 아니다.

미니게임:

- 안전 경로와 공간 반복 규칙 검증
- 피해자 탈출
- 마지막 단서 확보

회수 페이즈:

- 저승역이 다시 개인 목적지를 생성하지 못하게 공식 절차 적용
- 방송 공백과 정상 식별음 대조
- 현실 노선 경계 고정
- 현재 출현 안정화
- 잔향 포집

승인 안정화 절차:

1. 비정상 안내가 끝날 때까지 이동하지 않는다.
2. 방송에서 목적지 공백 구간을 식별한다.
3. 공식 운행 기록과 정상 역명·노선 식별음을 대조한다.
4. 공백 구간에 검증된 식별음을 적용한다.
5. 개인별로 들리던 목적지를 공식 기록으로 덮는다.
6. 현실 노선과 출구를 고정한다.
7. 공간 반복이 중단되면 안정화 상태로 전환한다.
8. 스피커와 기록 장치에 남은 잔향을 포집한다.

표기:

> **저승역 — 안정화 상태**

회수 대상:

> **저승역 반복 안내 잔향**

검은 승차권은 괴이 자체나 핵이 아니다. 노선 검증 과정에서 개인 목적지와 잔향이 각인된 물리 증거 또는 포집 매체로 역할을 바꾼다.

# 14. 괴이 매뉴얼

저승역 완료 뒤 기존 보고서와 기록국 DB를 다음 항목을 가진 괴이 매뉴얼로 재구성한다.

- 괴이명
- 발생 감정
- 관측 현상
- 출현 조건
- 초기 전조
- 위험 행동
- 위험 사례
- 검증된 대응
- 마지막 단서
- 회수한 잔향
- 안정화 상태
- 재출현 가능성
- 미확인 항목

항목 상태는 복잡한 퍼센트보다 다음 표현을 우선한다.

```text
미확인 / 추정 / 현장 검증
```

저승역 등록 초안:

- 발생 감정: 마지막 기회를 놓쳤다는 후회와 귀환 강박
- 초기 전조: 막차 이후 안내, 공식 기록에 없는 목적지, 서로 다른 목적지 인식
- 위험 행동: 안내 종료 전 이동, 개인 증언 확정, 미검증 방송 재생
- 위험 사례: 조기 이동 공간 초기화, 개인 목적지 폐쇄 순환, 방송 증폭
- 검증된 대응: 공식 기록 대조, 목적지 공백 확인, 정상 식별음과 안전 노선 복원
- 마지막 단서: 안전 노선 검증 기록
- 회수 잔향: 저승역 반복 안내 잔향
- 안정화 상태: 현재 공간 반복 중단
- 재출현 가능성: 유사 감정과 조건이 쌓인 다른 역에서 재출현 가능

사건 종료 문구:

> 승강장의 반복은 멈췄다.
> 그러나 저승역이 사라진 것은 아니다.
>
> 같은 후회와 같은 이야기가 다시 조건을 갖춘다면,
> 다른 역에서 같은 안내가 시작될 수 있다.
>
> 현장에서 확인한 대응 절차를 괴이 매뉴얼에 등록한다.

결과 표기:

```text
현재 출현 안정화
피해자 구조 완료
공간 반복 중단
저승역 반복 안내 잔향 회수
괴이 매뉴얼 초안 작성 완료
재출현 가능성: 존재
```

- 일반 단서 4개와 미니게임 마지막 단서 구조
- 위험 사례를 현재 저장 구조로 기록하는 방법
- 검은 승차권 역할 변경 영향

### 고정 편성

- 저승역에서 오현·권나래·강이준을 강제하는 최소 방법
- 기존 편성·대표 요원·신뢰도·저장 영향
- 튜토리얼 이후 자유 편성과 충돌 여부

### 미니게임

- 3×3을 내부 튜토리얼로 연결하는 방법
- 4×4 보드 데이터를 코드 상수, Resource, JSON 중 어디에 둘지
- 자동 최단 해법 검증 방법
- 위험도·괴이 고착도 임계값 연결
- 변수 조합 결정과 재시작 고정 방법
- `minigame_frequency_sync` 내부 ID 유지 방법

### 회수·결과·매뉴얼

- 미니게임 완료 후 회수 페이즈 직행 방법
- 공식 식별음과 노선 고정 규칙으로 회수 의미를 바꾸는 방법
- 마지막 단서가 회수 판단 근거로 실제 사용되게 하는 방법
- 결과 화면, 완료 보고서, DB를 괴이 매뉴얼로 재구성하는 방법
- 잔향과 정밀 해결 보상 저장 방법

새 저장 필드가 필요하면 바로 구현하지 않는다. 기존 구조 대안과 마이그레이션 영향을 먼저 제시한다.

## 단계 3 — 첫 10분 vertical slice

사용자 승인 후 가장 작은 end-to-end 범위로 구현한다.

# 18. 전체 수용 기준

## 총기획

- [ ] 플레이어가 괴이가 왜 완전히 죽지 않는지 설명할 수 있다.
- [ ] 플레이어가 괴이 기록국이 괴이 매뉴얼을 만드는 이유를 설명할 수 있다.
- [ ] 사건 완료가 영구 제거가 아니라 안정화 상태로 읽힌다.
- [ ] 잔향이 전리품이 아니라 대응 증거로 읽힌다.

## 첫 10분

- [ ] 시작 후 3분 안에 의미 있는 선택이 나온다.
- [ ] 10분 안에 단서 두 개를 연결한다.
- [ ] 가설을 한 번 행동으로 검증한다.
- [ ] 성공 또는 위험 사례의 즉시 반응을 본다.
- [ ] 요원은 근거를 제공하지만 정답을 대신 선택하지 않는다.

## 콘텐츠 구조

- [ ] 단서, 힌트, 위험 사례가 서로 다른 역할을 가진다.
- [ ] 미니게임 결과가 마지막 단서가 된다.
- [ ] 미니게임과 회수 페이즈가 같은 행동을 반복하지 않는다.
- [ ] 검은 승차권은 괴이 핵이 아니라 잔향 증거로 읽힌다.

## 노선 미니게임

- [ ] 3×3과 4×4가 하나의 대표 미니게임 안에서 정리된다.
- [ ] 변수 없는 보드의 최단 횟수 `N`이 자동 탐색 또는 전수 검증된다.
- [ ] 모든 허용 변수 조합의 실제 최단이 `N+2` 이하다.
- [ ] 횟수 초과가 실패나 진행 차단을 만들지 않는다.
- [ ] 잘못된 목적지 또는 규칙 적용만 위험 사례가 된다.
- [ ] 단서 열람과 무전 재확인은 조작 횟수를 소비하지 않는다.

## 저장·복구

- [ ] 미니게임 중 저장 UI가 노출되지 않는다.
- [ ] 진입 전 체크포인트가 저장된다.
- [ ] 완료 전 종료·불러오기 후 초기 보드·0회로 돌아온다.
- [ ] 같은 보드와 같은 변수 조합이 유지된다.
- [ ] 종료·불러오기로 보상이나 변수를 재추첨할 수 없다.
- [ ] 완료 결과가 한 번만 저장된다.

## 회수·매뉴얼

- [ ] 마지막 단서가 회수 판단 근거로 실제 사용된다.
- [ ] 저승역이 공식 식별음과 노선 고정으로 안정화된다.
- [ ] `저승역 반복 안내 잔향`이 회수된다.
- [ ] 위험 사례와 검증된 대응이 괴이 매뉴얼에 기록된다.
- [ ] 일반 해결도 핵심 매뉴얼과 이야기를 모두 얻는다.

## 회귀

- [ ] 빨간 우산 미니게임에 회귀가 없다.
- [ ] 기존 저장 파일이 안전하게 불러와진다.
- [ ] 기존 완료 사건 보고서와 DB가 깨지지 않는다.
- [ ] 저승역 외 사건의 의미가 임의로 바뀌지 않는다.
