# Codex Goal - MVP-020

## 기준

- Issue: #27 `[MVP-020] 5개 단위 회고·규칙 다이어트·시스템/UI 전환 기준 정리`
- 주의: Issue #20은 오생성/중복 종료 이슈이므로 사용하지 않는다.

## 작업 전 확인

`docs/DOCUMENTATION_MAP.md`의 읽기 순서를 따른 뒤, Issue #27과 아래 파일을 확인한다.

- `MVP_ROADMAP.md`
- `TEST_CHECKLIST.md`
- `docs/MVP_STATUS_AUDIT.md`
- `docs/DOCUMENTATION_MAP.md`
- 필요 시 `README.md`

## 작업

Issue #27을 기준으로 MVP-020 문서 정리를 수행한다.

1. `MVP_ROADMAP.md`의 MVP-020 기준선과 MVP-021~030 시스템/UI 중심 로드맵을 확인·정리한다.
2. `TEST_CHECKLIST.md`에 MVP-020 회고/문서 정리 확인 항목이 충분한지 점검한다.
3. `docs/MVP_STATUS_AUDIT.md`에 MVP-015~019 회고, 유지/축소 규칙, 남은 위험이 정리되어 있는지 확인한다.
4. `docs/DOCUMENTATION_MAP.md`가 실제 MVP-020 Issue #27과 이 Goal을 가리키는지 확인한다.
5. 문서 간 충돌이 있으면 최소 수정한다.

## 제한

- 새 괴담 에피소드, 새 미니게임, 새 회수 시스템을 추가하지 않는다.
- 코드 리팩터링과 저장 구조 변경은 하지 않는다.
- HTML 대시보드를 GitHub 소스로 관리하지 않는다.
- Issue, 로드맵, 규칙 문서에 이미 있는 내용을 Goal에 장문으로 반복하지 않는다.

## 완료 기준

- MVP-015~019 회고와 MVP-021~030 전환 방향이 문서에서 충돌 없이 확인된다.
- 핵심 규칙과 줄일 중복이 분리되어 있다.
- 다음 MVP-021이 `메인 UI / 사건 준비 UI 리디자인 1차`로 명확하다.
- 코드 변경이 없거나, 있다면 이유가 명확하다.
