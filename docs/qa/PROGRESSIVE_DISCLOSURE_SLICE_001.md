# UX-PD-001 사건 준비 점진적 공개 검증

> 상태: `IMPLEMENTED_2A / GODOT_4_7_1_FULL_PASS / DOCS_SYNCED / PLAYER_TEST_PENDING`
> 기준선: `MVP-043 + CORE-VALIDATION-001 + UX-PD-001` / `Ver 4.2` / 저장 Schema `mvp-039`
> 기획 원본: `docs/planning/PROGRESSIVE_DISCLOSURE_PLAN.md`

## 1. 검증 목적

첫 사건 준비에서 플레이어가 일정·사건·편성부터 이해하고, 장비·외부 접점·기록은 기능 손실 없이 필요할 때 펼칠 수 있는지 검증한다.

## 2. 구현 계약

- `사건`, `편성` 탭은 항상 표시한다.
- 신규 상태에서는 `장비`, `외부 접점`, `기록` 탭을 접는다.
- 대시보드 네 번째 버튼은 접힌 상태에서 `보조 준비 도구`, 펼친 상태에서 기존 `외부 의뢰`로 동작한다.
- 공개 버튼은 기능을 해금하거나 비용을 소비하지 않고 표시 상태만 바꾼다.
- 공개 경험은 기존 `seen_hint_ids`의 `preparation_secondary_tools_opened`로 저장한다.
- 완료 보고서·장비·기록·소모품·수락 의뢰가 있는 저장은 자동으로 전체 도구를 표시한다.
- 저장 Schema와 시장·장비·세력 수치는 변경하지 않는다.

## 4. 자동 검증

전용 테스트 `tests/progressive_disclosure_preparation_test.gd`는 다음을 확인한다.

1. 준비 화면에 기존 5개 탭이 모두 생성된다.
2. 사건·편성은 항상 보인다.
3. 첫 신규 상태에서는 장비·외부 접점·기록이 숨겨진다.
4. 공개 버튼이 표시된다.
5. 버튼을 누르면 보조 탭 전체와 외부 의뢰 바로가기가 복원된다.
6. 기존 힌트 상태에 공개 ID가 저장된다.
7. 준비 화면 재진입 뒤에도 공개 상태가 유지된다.
8. 1280×720에서 대시보드와 탭 영역이 화면 경계를 넘지 않는다.

연결 회귀:

- `preparation_schedule_ui_test.gd`
- `preparation_agent_ui_test.gd`
- `daily_episode_ui_test.gd`
- 전체 `tests/run_godot_regression.sh`

## 6. 적대적 검토

### 보조 기능이 잠긴 것으로 오해할 위험

버튼 문구와 툴팁에서 장비·외부 접점·기록을 즉시 표시한다는 점을 명시한다. 별도 해금 조건이나 비용을 두지 않는다.

### 진행 저장의 기능이 다시 접힐 위험

공개 ID 외에도 실제 완료 보고서·장비·기록·장착·소모품·수락 의뢰를 검사해 자동 공개한다.

### 새 저장 상태가 기존 저장을 깨뜨릴 위험

새 필드를 추가하지 않고 기존 `seen_hint_ids`를 사용한다. `SAVE_VERSION`은 `mvp-039`를 유지한다.

### 대시보드가 720p에서 더 길어질 위험

새 설명 패널을 추가하지 않고 기존 2×2 액션 그리드의 네 번째 버튼을 교체한다.

### 시장·장비 사용 기회가 줄어드는 위험

기능 자체는 첫 화면에서 한 번의 명시적 공개만 요구한다. 보유 콘텐츠가 있거나 한 번 펼친 뒤에는 자동으로 전체 표시한다.
