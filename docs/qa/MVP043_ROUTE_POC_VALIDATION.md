## 구현 범위

- 기존 저승역 `minigame_frequency_sync` 진입 경로를 3×3 노선 복원 PoC로 런타임 전환한다.
- 새 `route_restore_game.gd`가 곡선 타일 회전, 분기 스위치 전환, 경로 검사, 초기화를 담당한다.
- 시작점에서 공식 안전 목적지까지 연결하고 가짜 목적지와 분리되면 완료한다.
- 기본 최적 횟수는 4회, 정밀 해결 기준은 6회다.
- 가짜 목적지로 이동 명령을 내리면 위험 사례를 표시하고 현재 보드와 조작 횟수를 유지한다.
- 조작 횟수 초과는 실패가 아니며 일반 복원으로 완료한다.
- 미니게임 중 저장·디버그 장면 이동 버튼을 제거하고 저장 불가 안내를 표시한다.
- 미니게임 진입 전에 조사 씬이 저장한 체크포인트를 사용하므로 비정상 종료 후에는 같은 미니게임의 초기 보드로 돌아간다.

## 정적 검증

### 기본 보드 최단 횟수

회전 상태를 전수 확인한 결과, 다음 조합만 4회에 안전 목적지와 연결되고 가짜 목적지와 분리된다.

```text
분기 스위치: 서↔동 → 서↔북       1회
중앙 곡선:   북·동 → 동·남         1회
우측 곡선:   동·남 → 서·북         2회
합계                                  4회
```

따라서 PoC의 `N=4`, 정밀 기준 `N+2=6`과 일치한다.

### 저장 경계

- 조사 포인트는 미니게임 씬으로 이동하기 전에 현재 씬 경로와 사건 상태를 저장한다.
- 미니게임 진행 중에는 새 저장을 만들 수 있는 내비게이션 버튼을 노출하지 않는다.
- 정상 완료 시에만 미니게임 결과가 저장되고 조사 복귀 버튼이 열린다.
- 완료 전 프로세스 종료 시 타일 방향과 조작 횟수는 저장되지 않는다.

## Godot 수동 확인 순서

1. 저승역에서 플랫폼 안내방송을 조사하고 기존 현장 판정 조사 포인트로 진입한다.
2. 화면 제목이 `저승역 노선 복원`, 중앙 제목이 `노선 복원 보드`인지 확인한다.
3. 초기 상태에서 바로 `경로 확인`을 눌러 가짜 목적지 위험 사례가 표시되는지 확인한다.
4. 위험 사례 뒤 보드와 조작 횟수가 초기화되지 않는지 확인한다.
5. 분기 스위치 1회, 중앙 곡선 1회, 우측 곡선 2회 조작 후 안전 경로를 확정한다.
6. 4회 완료가 `최적 복원`으로 기록되는지 확인한다.
7. 일부 타일을 더 돌려 5~6회 완료하면 `정밀 복원`, 7회 이상이면 `일반 복원`인지 확인한다.
8. 미니게임 도중 게임을 종료하고 이어하기를 눌러 초기 보드·0회 상태로 돌아오는지 확인한다.
9. 정상 완료 후 조사로 복귀해 기존 단서·회수 흐름이 이어지는지 확인한다.
10. 빨간 우산의 `rain_dodge` 미니게임이 기존 방식으로 유지되는지 회귀 확인한다.

## 현재 제한

- 보호 경로인 `data/episodes/episode_001_afterlife_station.json`은 이번 PoC에서 수정하지 않았다. 기존 미니게임 ID와 효과를 유지하고 씬에서 저승역 ID만 노선 퍼즐로 전환한다.
- 4×4 마지막 단서 보드, 위험도·괴이 고착 변수, 마지막 단서 데이터 교체는 아직 구현하지 않았다.
- 3인 역할은 제목·설명·현장 문구에 반영했으며 별도 캐릭터 컷인과 음성 연출은 이후 작업이다.
- GitHub 연결 환경에서는 Godot headless와 실제 화면 입력을 실행하지 못했다. 위 수동 확인을 통과하기 전에는 플레이 검증 완료로 간주하지 않는다.
