# MVP-043 저승역 UI·주인공·괴이 매뉴얼 검증

> 검증일: 2026-07-15
>
> 기준: `docs/CODEX_GOAL_MVP_043.md`, 커밋 `c17d58c`, Issue #36
>
> 범위: 반응형 조사 UI, 페이지형 매뉴얼·5개 후보, 주인공·서포트, 3×3·4×4 표시 경계, 결과·DB 연결

## 구현 확인

- 저승역 조사 화면을 최대 1600px 중앙 안전 프레임과 3열 구조로 교정했다.
- 기존 F2 자유 배치 설정이 핵심 조사 구조를 재부모화하거나 이동하지 않도록 제외했다.
- 선택 가능한 지점 또는 후보가 없어도 다른 조사 지점 복귀와 HQ 복귀 경로를 유지한다.
- 성공·실패 사례와 5개 후보를 표시 카탈로그로 제공하고 보호 사건 JSON 구조는 변경하지 않았다.
- 비활성 후보는 제거 이유를 표시하며, 위험 행동 뒤에는 같은 목록으로 복귀한다.
- `selected_agent_ids` 순서를 주인공 1명과 서포트 최대 2명의 호환 인터페이스로 사용한다.
- 반일 일정은 주인공 한 명만 필요·소비하며 서포트는 독립 일정을 요구하지 않는다.
- 자동 지원 시도는 성공·실패 모두 기존 `used_agent_supports`에 남아 재진입 재추첨을 막는다.
- 노선 복원은 3×3 학습과 4×4 최종 검증을 한 컨트롤 안에서 구분하며 최종 단계만 결과를 저장한다.
- 완료 결과와 기록국 DB는 기존 미니게임 결과 상세값으로 안전 노선 기록과 매뉴얼 페이지를 표시한다.

## 실제 장면 검수

- 1280×720 조사 기본, 사례 팝업, 위험 사례 후, 4×4 최종 검증 캡처를 생성해 승인 참고 화면과 직접 비교했다.
- 조사 기본 화면에서 상단 HUD, 세 열, 첫 후보, 복귀 버튼이 화면 안에 있고 한국어 본문과 제거 근거가 내부 스크롤로 유지되는 것을 확인했다.
- 4×4 화면에서 비교 근거, 보드, 현장 반응 열과 기록 오버레이 입력 차단을 확인했다.

## 미검증·남은 위험

- 1920×1080과 1918×943은 자동 좌표 계약을 통과했으나 실제 장시간 마우스·키보드 플레이는 미검증이다.
- 첫 10분의 실제 소요 시간과 플레이어가 사례 비교 원칙을 이해하는지는 사용자 플레이테스트가 필요하다.
- 다른 사건 콘텐츠는 재작성하지 않았으며 공용 흐름 smoke만 확인했다.
- GDD DOCX는 생성 해시·문단·표·이미지 구조 검사를 통과했다. 현재 실행 환경에 LibreOffice/Word가 없어 페이지 PNG 렌더 검수는 미검증이다.

## 저장 경계

- 새 저장 필드, 저장 버전, 마이그레이션 없음.
- 보호 사건 JSON 구조 변경 없음.
- 주인공·서포트는 기존 `selected_agent_ids`, 지원 시도는 기존 `used_agent_supports`를 재사용한다.

## 사용자 확인 순서

1. 신규 캠페인에서 저승역 조사 화면에 진입한다.
2. 조사 장소를 바꾸고 플랫폼 스피커의 5개 후보와 페이지형 매뉴얼을 확인한다.
3. 실패사례를 열어 오답 비활성화 이유와 남은 후보 수를 확인한다.
4. 위험 후보를 선택해 위험 사례가 남고 같은 목록으로 복귀하는지 확인한다.
5. 3×3 학습 뒤 4×4 최종 검증과 결과·DB 매뉴얼을 확인한다.
6. HQ에서 주인공을 바꾸고 주인공에게만 반일 일정이 필요한지 확인한다.
