# MVP-043 저승역 노선 복원 퍼즐 보드 설계 v0.1

> 상태: **사용자 승인 설계 / 구현 미반영**
>
> 기준일: 2026-07-14
>
> 상위 규칙: `docs/MINIGAME_SYSTEM_SPEC.md`
>
> 적용 범위: 저승역 3인 고정 작전, 3×3 조작 학습 보드, 4×4 마지막 단서 보드, 위험도·괴이 고착 변수, 무전 연출, 마지막 단서와 회수 연결

## 1. 한 줄 정의

오현이 방송실에서 공식 운행 기록과 목적지 공백을 확인하고, 권나래가 피해자를 호송하며 현재 위치를 보고한다. 플레이어는 두 보고를 대조해 노선 타일을 최소한으로 조정하고, 강이준이 고정한 현실 출구까지 안전 경로를 복원한다.

## 2. 사건 내 위치

```text
저승역 조사 페이즈
→ 조사 단서 4개 확보
→ 3×3 조작 학습 보드
→ 본 조사 진행
→ 4×4 노선 복원 보드
→ 마지막 단서 `안전 노선 검증 기록`
→ 저승역 회수 페이즈
→ 현재 출현 안정화
→ 반복 안내 잔향 회수
```

3×3 보드는 조작법과 `표지보다 공식 기록을 신뢰한다`는 원칙을 가르친다. 4×4 보드는 조사 결과와 현장 변수를 종합해 마지막 단서를 만드는 사건의 최종 조사 행동이다.

## 3. 고정 편성과 역할

| 요원 | 위치 | 역할 | 정답 공개 경계 |
|---|---|---|---|
| 오현 | 방송실·기록 단말기 | 목적지 공백, 공식 운행 기록, 안내 종료 시점 보고 | 안전 목적지를 직접 선택하지 않음 |
| 권나래 | 피해자와 노선 내부 | 현재 위치, 현장 표지, 피해자 상태 보고 | 피해자의 증언을 객관 사실로 확정하지 않음 |
| 강이준 | 현실 승강장 경계 | 선택한 노선 고정, 출구 유지, 고착 구간 해제 보조 | 플레이어 대신 타일을 조작하지 않음 |

무전 리듬은 다음 순서를 따른다.

```text
오현의 객관 기록
→ 권나래의 현장 관찰
→ 피해자의 개인적 목적지 주장
→ 플레이어의 노선 조작
→ 강이준의 경계 고정
→ 현장의 즉시 반응
```

## 4. 조작 규칙

### 입력

- 타일 선택 후 확인: 90도 시계 방향 회전
- 분기 스위치 선택: 두 경로 상태 전환
- 고착 해제 장치 선택: 잠긴 타일을 조작 가능 상태로 변경
- 경로 확인: 현재 연결 상태만 검사하며 조작 횟수에 포함하지 않음
- 이동 명령: 연결 결과를 현장에 적용하며 조작 횟수에 포함하지 않음

v0.1은 역회전을 제공하지 않는다. 실제 최단 횟수 `N`은 시계 방향 회전만을 기준으로 계산한다.

### 횟수 계산

- 곡선 타일 90도 회전: 1회
- 직선 타일 가로·세로 전환: 1회
- 분기 스위치 전환: 1회
- 고착 해제: 1회
- 단서·무전·경로 미리보기·확정 버튼: 0회

되돌리기는 이전 횟수를 취소하지 않는다. `처음부터 다시`를 선택하면 해당 시도의 보드와 조작 횟수만 초기화한다.

## 5. 기호

| 기호 | 의미 |
|---|---|
| `S→` | 호송팀 시작점, 동쪽 연결 |
| `D✓` | 공식 기록으로 확인된 안전 목적지 |
| `D?` | 피해자 기억에 반응한 가짜 목적지 |
| `━` | 가로 직선 노선 |
| `┃` | 세로 직선 노선 |
| `└` | 북·동 곡선 |
| `┌` | 동·남 곡선 |
| `┐` | 남·서 곡선 |
| `┘` | 서·북 곡선 |
| `SW→` | 시작 상태가 가짜 목적지 방향인 분기 스위치 |
| `SW↑` | 안전 노선 방향으로 전환된 분기 스위치 |
| `■` | 이동 불가능한 폐쇄 구간 |

## 6. 3×3 조작 학습 보드

### 목적

- 타일 회전과 분기 스위치 조작을 한 번씩 경험한다.
- 피해자가 주장하는 목적지가 연결 가능해 보여도 공식 기록과 다를 수 있음을 보여준다.
- 시간제한과 현장 변수 없이 기본 규칙을 학습한다.

### 초기 배치

```text
┌────┬────┬────┐
│ ■  │ ■  │ D✓ │
├────┼────┼────┤
│ ■  │ └  │ ┌  │
├────┼────┼────┤
│ S→ │SW→ │ D? │
└────┴────┴────┘
```

좌표 기준:

| 좌표 | 타일 | 초기 상태 | 목표 상태 | 필요 조작 |
|---|---|---|---|---:|
| `(3,2)` | 분기 스위치 | 서↔동, 가짜 목적지 방향 | 서↔북 | 1 |
| `(2,2)` | 곡선 | 북·동 `└` | 동·남 `┌` | 1 |
| `(2,3)` | 곡선 | 동·남 `┌` | 서·북 `┘` | 2 |

### 목표 배치

```text
┌────┬────┬────┐
│ ■  │ ■  │ D✓ │
├────┼────┼────┤
│ ■  │ ┌  │ ┘  │
├────┼────┼────┤
│ S→ │SW↑ │ D? │
└────┴────┴────┘
```

### 횟수

- 변수 없는 실제 최단 횟수 `N`: **4회**
- 정밀 해결 기준 `N+2`: **6회 이내**
- 현장 변수: 없음

회전 상태 전수 탐색 기준으로 안전 목적지와 연결되고 가짜 목적지와 연결되지 않는 최초 상태가 4회에서 확인된다.

### 연출 순서

1. 피해자가 `D?` 방향을 자신의 집으로 가는 막차라고 주장한다.
2. 오현이 해당 역명이 공식 운행 기록에 없다고 보고한다.
3. 권나래가 두 방향 모두 표지판은 보이지만, 가짜 방향에는 선로 연결음이 없다고 보고한다.
4. 플레이어가 스위치를 북쪽으로 전환하고 곡선 두 개를 연결한다.
5. 강이준이 복원된 노선 경계를 짧게 고정한다.
6. 권나래가 피해자와 한 구간 이동한 뒤 본 조사로 복귀한다.

### 잘못된 목적지 확정

스위치를 초기 상태로 둔 채 `D?`로 이동시키면 다음 위험 사례를 등록한다.

> **위험 사례 — 개인이 인식한 목적지를 공식 경로로 적용하면 노선이 같은 승강장으로 되돌아간다.**

이후 보드는 초기화하지 않고 스위치 상태와 조작 횟수를 유지한 채 재조정한다. 튜토리얼에서는 위험도 상승을 최소 연출로만 표시하고 사건 진행을 차단하지 않는다.

## 7. 4×4 마지막 단서 보드

### 목적

- 조사 단서 4개를 실제 행동에 사용한다.
- 오현과 권나래의 서로 다른 정보 범위를 대조한다.
- 위험도와 괴이 고착도에 따른 변수를 적용한다.
- 완료 결과를 마지막 단서 `안전 노선 검증 기록`으로 남긴다.

### 초기 배치

```text
┌────┬────┬────┬────┐
│ ■  │ ■  │ ┐  │ D✓ │
├────┼────┼────┼────┤
│ ■  │ └  │ ┌  │ ■  │
├────┼────┼────┼────┤
│ ■  │ ━  │ ■  │ ■  │
├────┼────┼────┼────┤
│ S→ │SW→ │ ━  │ D? │
└────┴────┴────┴────┘
```

### 목표 배치

```text
┌────┬────┬────┬────┐
│ ■  │ ■  │ ┌  │ D✓ │
├────┼────┼────┼────┤
│ ■  │ ┌  │ ┘  │ ■  │
├────┼────┼────┼────┤
│ ■  │ ┃  │ ■  │ ■  │
├────┼────┼────┼────┤
│ S→ │SW↑ │ ━  │ D? │
└────┴────┴────┴────┘
```

### 타일별 최소 조작

| 좌표 | 타일 | 초기 상태 | 목표 상태 | 최소 조작 |
|---|---|---|---|---:|
| `(4,2)` | 분기 스위치 | 서↔동 | 서↔북 | 1 |
| `(3,2)` | 직선 | 가로 `━` | 세로 `┃` | 1 |
| `(2,2)` | 곡선 | 북·동 `└` | 동·남 `┌` | 1 |
| `(2,3)` | 곡선 | 동·남 `┌` | 서·북 `┘` | 2 |
| `(1,3)` | 곡선 | 남·서 `┐` | 동·남 `┌` | 3 |
| 합계 |  |  |  | **8** |

### 횟수

- 변수 없는 실제 최단 횟수 `N`: **8회**
- 정밀 해결 기준 `N+2`: **10회 이내**
- 최적 해결: 8회 이하
- 정밀 해결: 9~10회
- 일반 해결: 11회 이상

회전 상태 전수 탐색 기준으로 안전 목적지와 연결되고 가짜 목적지와 연결되지 않는 최초 상태가 8회에서 확인된다.

## 8. 현장 변수

3×3 조작 학습 보드에는 변수를 적용하지 않는다. 4×4 마지막 단서 보드에서만 조사 결과에 따라 변수를 적용한다.

### 변수 A — 목적지 혼선

- 주요 입력: 위험도
- 추가 필수 조작: 0회
- 효과: `D✓`와 `D?`의 역명이 같은 글꼴과 색으로 표시되며, 공식 기록 아이콘만 구분 근거로 남는다.
- 단서 완화: 공식 운행 기록을 확보하면 `D✓`에 기록 일치 아이콘을 표시한다.
- 공정성: 안전 목적지 자체를 숨기지 않고, 어떤 기록이 신뢰 가능한지만 판단하게 한다.

오현:

> “두 표시는 같은 역명을 내보내고 있습니다. 화면 문구가 아니라 공식 식별 번호를 확인하세요.”

### 변수 B — 노선 고착

- 주요 입력: 괴이 고착도
- 추가 필수 조작: +1회
- 효과: `(2,3)` 곡선 타일이 잠긴 상태로 시작한다.
- 대응: 강이준의 경계 해제 장치를 1회 조작한 뒤 정상 회전 가능하다.
- 예고: 퍼즐 시작 전에 검은 고정못과 잠금 아이콘을 표시한다.

강이준:

> “중앙 분기는 저승역 쪽에 박혀 있습니다. 먼저 고정못부터 풀어야 합니다.”

### 변수 C — 경로 흔들림

- 주요 입력: 높은 위험도와 괴이 고착도의 조합
- 추가 필수 조작: +1회
- 효과: 플레이어가 처음으로 안전 목적지까지 완전 연결한 순간 `(3,2)` 직선 타일이 `┃`에서 `━`로 한 차례 뒤틀린다.
- 대응: 직선 타일을 1회 다시 회전한다.
- 예고: 해당 타일에 검은 잔향이 모이며, 오현이 변동 예상 구간을 사전에 보고한다.
- 반복: 한 시도에 1회만 발생한다.

오현:

> “세 번째 구간의 파형이 불안정합니다. 연결 직후 한 번 더 틀어질 가능성이 있습니다.”

### 조합별 실제 최소 횟수

| 변수 조합 | 실제 최소 횟수 | 정밀 기준 10회 달성 가능 |
|---|---:|---|
| 없음 | 8 | 가능 |
| 목적지 혼선 | 8 | 가능 |
| 노선 고착 | 9 | 가능 |
| 경로 흔들림 | 9 | 가능 |
| 노선 고착 + 경로 흔들림 | 10 | 가능 |
| 목적지 혼선 + 구조 변수 | 구조 변수와 동일 | 가능 |

변수로 강제되는 추가 조작은 최대 2회다. 10회를 초과해야만 해결되는 변수 조합은 생성하지 않는다.

## 9. 무전 발생 지점

| 조건 | 발화자 | 전달 정보 |
|---|---|---|
| 퍼즐 진입 | 오현 | 목적지 공백과 공식 식별 번호 |
| 첫 타일 조작 | 권나래 | 피해자가 주장하는 목적지와 현장 표지 차이 |
| 가짜 목적지 연결 가능 | 피해자 | 개인 기억에 기반한 유혹 |
| 노선 고착 확인 | 강이준 | 해제 장치 필요 |
| 조작 횟수 절반 도달 | 오현 | 현재 노선이 괴이에 각인되고 있다는 경고 |
| 첫 완전 연결 | 권나래 | 이동 준비 완료 또는 경로 흔들림 발생 보고 |
| 정밀 기준 2회 전 | 강이준 | 횟수 압박이 아닌 서사적 경고만 제공 |
| 안전 경로 확정 | 오현 | 공식 기록과 실제 경로 일치 확인 |

`정밀 기준 2회 전` 안내는 남은 횟수를 대신 계산해 주지 않는다. UI에 현재 횟수와 정밀 기준이 이미 표시되므로, 대사는 현장 긴장만 보강한다.

## 10. 잘못된 경로 처리

### 가짜 목적지 확정

```text
가짜 목적지 연결
→ 이동 명령
→ 권나래와 피해자가 같은 분기점으로 복귀
→ 시간 경과, 위치 초기화 확인
→ 위험 사례 등록
→ 위험도 소폭 상승
→ 현재 타일 상태와 조작 횟수를 유지한 채 재조정
```

### 끊긴 경로 확정

호송팀은 이동하지 않는다. 권나래가 연결음이 없다고 보고하며 위험 사례는 등록하지 않는다. 이는 조작 실수이지 괴이 규칙에 대한 오대응이 아니기 때문이다.

### 순환 경로 확정

향후 분기·순환 타일이 추가될 경우에만 사용한다. v0.1 두 보드는 순환 구조를 필수로 사용하지 않으며, 가짜 목적지 복귀로 기본 위험 사례를 가르친다.

## 11. 마지막 단서

### 명칭

**안전 노선 검증 기록**

### 기본 완료 기록

> 공식 운행 기록과 일치하는 노선은 공간 초기화 없이 현실 승강장으로 연결되었다. 개인이 인식한 목적지는 실제 방송 원본에 존재하지 않았다.

### 위험 사례를 경험한 기록

> 개인이 인식한 목적지를 경로로 확정하자 시간은 계속 흘렀지만 호송팀의 위치가 출발 분기점으로 초기화되었다. 공식 식별 번호를 적용한 노선에서는 같은 현상이 발생하지 않았다.

### 정밀 해결 추가 기록

> 불필요한 노선 전환을 억제한 상태에서 공식 경로와 실제 이동 기록이 완전히 일치했다. 잔향 오염이 적어 재현 가능한 대응 절차로 등록할 수 있다.

검은 승차권은 미니게임 종료 시 목적지 문자가 사라지고 공식 식별 번호의 천공 자국만 남는다. 이는 괴이의 핵이 아니라 `저승역 반복 안내 잔향`이 각인된 물리 증거다.

## 12. 회수 페이즈 연결

미니게임 완료는 피해자 구조와 안전 경로 검증까지 담당한다. 현재 출현의 안정화는 별도 회수 페이즈에서 진행한다.

```text
안전 노선 검증 기록 확보
→ 저승역이 복원 노선을 다시 덮어쓰려 함
→ 목적지 공백·공식 식별 번호·위험 사례 확인
→ 정상 식별음 적용
→ 강이준이 현실 경계 고정
→ 반복 안내 중단
→ 현재 출현 안정화
→ 반복 안내 잔향 회수
```

정밀 해결 시 회수 시작 부담만 소폭 낮아진다. 회수 정답은 자동 공개하지 않는다.

## 13. UI 요구사항

PC 16:9 기준 권장 구조:

- 중앙: 노선 퍼즐 보드
- 좌측: 확보 기록 2~4개, 공식 식별 번호
- 우측 상단: 현재 조작 횟수 / 정밀 기준
- 우측 중단: 적용 변수와 예상 영향
- 하단: 최신 무전 2줄과 피해자 상태
- 보드 위: 시작점, 목적지 후보, 현재 연결 미리보기

항상 표시할 정보:

- 현재 횟수
- 정밀 해결 기준 `N+2`
- 적용 변수 이름
- 출발점과 목적지 후보
- 색상 외의 노선 모양·아이콘
- 무전의 텍스트 자막

플레이 중 `N`은 공개하지 않는다. 결과 화면에서만 실제 최적 횟수와 플레이어 기록을 비교한다.

- 퍼즐 도중 수동·자동 저장 불가
- 진입 직전 체크포인트 저장
- 강제 종료·불러오기 시 해당 보드의 첫 시작 지점으로 복귀
- 타일 방향·현재 횟수·현 시도 연출 초기화
- 단서·위험도·괴이 고착도·보드 ID·변수 조합 유지
- 같은 체크포인트는 같은 보드와 같은 변수 조합으로 재진입
- 정상 완료 후 마지막 단서 생성과 회수 페이즈 진입 시 저장 가능

## 16. 구현 수용 기준

- 3×3 보드는 변수 없이 4회가 실제 최단 해법이다.
- 4×4 보드는 변수 없이 8회가 실제 최단 해법이다.
- 고착과 흔들림이 함께 발생해도 실제 최단 해법은 10회 이하다.
- 가짜 목적지와 안전 목적지는 단서로 구분 가능하다.
- 타일 조작만 횟수로 계산되고 기록 열람은 계산하지 않는다.
- 최적·정밀·일반 완료 모두 마지막 단서와 회수 진입을 보장한다.
- 잘못된 목적지만 위험 사례를 만들고, 끊긴 연결은 단순 조작 오류로 처리한다.
- 세 요원은 각자의 역할을 수행하지만 최종 노선을 대신 선택하지 않는다.
- 미니게임 도중 저장할 수 없으며 불러오기는 동일 보드·동일 변수의 시작점으로 복귀한다.
- 구현 테스트에 보드 상태 전수 탐색 또는 동등한 최단 경로 검증을 포함한다.
