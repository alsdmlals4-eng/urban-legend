## 1. 게임 약속

| 질문 | 답 |
|---|---|
| 플레이어는 누구인가 | 괴이 기록국의 현장 판단 담당 요원 권나래 |
| 무엇을 하는가 | 관측 가능한 현상을 조사하고 규칙 가설을 세워 현재 출현을 안정화한다 |
| 무엇을 남기는가 | 다음 피해를 막는 공식 규칙·후보·위험 사례가 담긴 괴이 매뉴얼 |
| 무엇을 하지 않는가 | 괴이를 처치하거나 요원·장비가 정답을 대신 선택하지 않는다 |

핵심 경험은 **상황 설명 → 선택 → 짧은 결과 → 기록 → 다음 질문**의 반복이다. 실패는 단순한 손실이 아니라 이후 가설을 배제하거나 보완할 위험 사례를 남긴다.

## 2. 공식 용어

| 개념 | 플레이어 노출 명칭 | 의미 |
|---|---|---|
| 기관 | 괴이 기록국 | 괴이를 제거하지 않고 조사·안정화·기록하는 조직 |
| 사건 종료 | 안정화 상태 | 현재 출현이 통제된 상태 |
| 실패 기록 | 위험 사례 | 잘못된 판단과 당시 근거를 보존한 기록 |
| 회수 대상 | 잔향 | 안정화 후 남아 회수 가능한 현상 잔재 |
| 최종 기록 | 괴이 매뉴얼 | 공식 규칙·후보·위험 사례가 누적되는 지식 보상 |
| 안내자 | 기록관 아카 | 확보 정보만 정리하는 관제 안내자 |

내부 호환 ID나 과거 명칭이 남아 있더라도 플레이어 노출 문구는 위 용어를 우선한다.

## 3. 핵심 루프

```text
반일 계획
→ 사건·편성 확인
→ 현장 조사와 근거 확보
→ 규칙 판단
→ 안정화·잔향 회수
→ 보고서·괴이 매뉴얼 갱신
→ 다음 반일
```

| 루프 단계 | 플레이어 질문 | 시스템 결과 |
|---|---|---|
| 준비 | 이번 반일에 무엇을 조사하고 누구와 갈 것인가 | 일정·편성·보조 도구 선택 |
| 조사 | 무엇이 변했고 어떤 근거가 있는가 | 단서·위험·현재 효과 갱신 |
| 판단 | 어떤 가설과 대응이 근거에 맞는가 | 성공·실패와 위험 사례 생성 |
| 기록 | 무엇이 검증됐고 무엇이 남았는가 | 공식 규칙·후보·보고서·DB 갱신 |

## 4. 캠페인과 시간

| 항목 | 계약 |
|---|---|
| 기간 | 최대 10일 |
| 시간 단위 | 오전·오후의 순차 반일 |
| 주인공 일정 | 권나래의 일정만 필요·소비 |
| 서포트 | 최대 2명, 독립 일정 불필요 |
| 결과 확인 | 오전 결과 확인 전 오후 계획 불가 |
| 중단·재개 | HQ 복귀·저장·불러오기 후 같은 작전과 슬롯 보존 |

난수, 지원 시도, 세력 의뢰 게시판과 진행 중 작전은 불러오기로 재추첨되지 않아야 한다.

## 5. 인물 구조

| 구분 | 현재 구현 |
|---|---|
| 주인공 | 권나래 |
| 초기 요원 | 윤서하, 오현, 강이준, 한유리 |
| 외부 접점 | 박도윤, 이세린, 레이먼드 케인, 카밀라 바르가스 |
| 편성 | 권나래 + 서포트 0~2명 |

![괴이 기록국 요원](../assets/agents/kwon_narae_expressions.png)

선택하지 않은 요원은 자동 보조·관계 반응·현장 대사에 등장하지 않는다. 능력·장비·관계는 위험 관리와 표현 차이를 만들 수 있지만 핵심 추리 정답을 제공하지 않는다.

## 6. 사건 구조

| 사건 | 핵심 질문 | 구현 상태 |
|---|---|---|
| 저승역 | 반복되는 승강장과 안내의 규칙은 무엇인가 | 구현 확인 |
| 비 오는 골목의 빨간 우산 | 관측·거리·접촉 조건은 어떻게 피해를 만드는가 | 구현 확인 |
| 폐주파수 방송국 | 송출·청취·반응의 규칙은 무엇인가 | 구현 확인 |

사건 데이터와 ID의 책임 원본은 `data/episodes/`다. 기획 문서는 의미를 설명하지만 실제 선택·단서·수치·보상 ID를 복제하지 않는다.

## 7. 저승역 코어 검증

| 단계 | 플레이어 행동 | 기록 결과 |
|---|---|---|
| 전조 | 현재 관측 질문을 확인 | 판단 범위 형성 |
| 가설 | 가능한 규칙 설명을 선택 | 임시 가설 |
| 근거 | 확보한 지지·반증 근거를 선택 | 검증 강도 |
| 대응 | 현장 행동을 선택 | 안정화 성공·실패 |
| 승격 | 결과를 매뉴얼에 기록 | 공식 규칙·후보·위험 사례 |

현장 대응 성공과 공식 매뉴얼 승격은 분리한다. 올바른 대응이라도 가설·근거가 부족하면 후보로 남으며, 오대응은 이후 성공해도 위험 사례에서 삭제하지 않는다.

## 8. 준비 화면 정보 위계 — UX-PD-001 2A

| 계층 | 첫 준비 표시 | 공개 방식 |
|---|---|---|
| 사건·일상 | 항상 표시 | 기본 |
| 편성 | 항상 표시 | 기본 |
| 장비 | 최초 접기 | 보조 준비 도구 버튼 |
| 외부 접점 | 최초 접기 | 보조 준비 도구 버튼 |
| 기록 | 최초 접기 | 보조 준비 도구 버튼 |

점진적 공개는 기능 잠금이 아니다. 한 번 펼치면 기존 `seen_hint_ids`에 기억하며, 보고서·장비·기록·소모품·수락 의뢰가 있는 진행 저장은 자동으로 전체 기능을 표시한다. 일정·경제·장비·세력 상태는 기존 준비 화면과 `GameState`가 소유하고 표시 확장 계층은 가시성만 조정한다.
