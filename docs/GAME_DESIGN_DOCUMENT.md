## 1. 게임 약속

| 질문 | 답 |
|---|---|
| 플레이어는 누구인가 | 괴이 기록국의 현장 판단 담당 요원 권나래 |
| 조사에서 무엇을 하는가 | 관측 정보와 매뉴얼을 비교하고 규칙 가설에 지지·반박·미해결 근거를 연결한다 |
| 전투에서 무엇을 하는가 | 현장 이해도로 전조를 읽고 턴제 대응을 누적해 포획 창을 연다 |
| 무엇을 남기는가 | 공식 규칙·관측 패턴·위험 사례·연구 결과가 축적되는 괴이 매뉴얼 |
| 장기 목표는 무엇인가 | 다시 나타난 괴이를 더 안전하게 다루고 더 나은 서사 결과를 선택한다 |
| 무엇을 하지 않는가 | 자동 정답, 거짓 예측, 핵심 추리 난수화, 조사와 무관한 화력전, 권나래 교체 |

## 2. 대상 플레이어와 플레이 맥락

- 오컬트·괴담 서사를 좋아하지만 순수 비주얼 노벨보다 직접 판단하는 게임을 원하는 플레이어
- 정답을 찍기보다 관측·비교·설명하는 추리를 선호하는 플레이어
- 실시간 액션보다 낮은 입력 부담의 턴제 판단을 선호하는 플레이어
- 한 사건 18~25분의 명확한 수사-회수 완결을 원하는 PC/Steam 플레이어

초기 PoC는 하드코어 전투 최적화 플레이어나 대규모 경영 시뮬레이션 플레이어를 주 대상으로 삼지 않는다.

## 3. 핵심 루프

```text
[조사]
관측 → 매뉴얼 비교 → 선택지 배제 → 규칙 가설 카드 → 위험 검증 → 현장 이해도

[회수]
전조 → 해석 판정 → 대응 선택 → 괴이 반응 → 포획 창 → 회수·잔향

[기록]
가설·결과 대조 → 후보·공식 규칙·위험 사례 → 다음 출현의 괴이 매뉴얼
```

### 3.1 Micro / Session / Meta

| 층 | 반복 행동 | 보상 |
|---|---|---|
| Micro | 기록을 선택지에 연결해 모순 배제 | 가능성 축소와 즉시 ‘아하’ |
| Session | 규칙을 읽고 전조 대응으로 포획 창 열기 | 추리가 전투에서 증명되는 숙련감 |
| Meta | 성공·실패를 매뉴얼에 승격 | 다음 출현의 정보 우위와 새로운 분기 |

## 4. 조사 페이즈

### 4.1 기본 화면

1. 현재 상황·관측 가능한 전조
2. 대응 선택지 4개
3. 관련 괴이 매뉴얼 사례 2~3개 자동 제시
4. 규칙 가설 카드
5. 최소 상태: 권나래 체력, 괴이 위험도, 직전 반응

장비·세력·상세 수치·관계 태그는 조사 기본 화면에서 접고 필요할 때만 연다.

### 4.2 선택지 구조

| 구분 | 수량 | 판별 방식 |
|---|---:|---|
| 배제 가능한 오답 | 2 | 확정 규칙·위험 사례·현재 근거와의 모순 |
| 경쟁 가설 | 2 | 현재 정보에서 모두 합리적이며 검증 필요 |

오답은 농담 선택지나 장르 상식으로 고르는 선택지가 아니다. 일부 기록에는 맞지만 현재 전조나 조건 하나와 충돌해야 한다.

### 4.3 기록 연결

- 시스템은 관련 사례를 찾지만 적용 대상을 결정하지 않는다.
- 플레이어가 기록을 선택지에 직접 연결한다.
- 타당한 연결은 선택지를 배제한다.
- 부적합한 연결은 `이 기록만으로는 배제할 수 없다`고 표시한다.
- 배제 시도에는 체력·위험도 비용이 없다.
- 정답 선택지 강조, 자동 추천, 미확보 정보 노출은 금지한다.

### 4.4 규칙 가설 카드

| 필드 | 계약 |
|---|---|
| 규칙 문장 | 남은 선택지 또는 사건 문장 템플릿에서 선택·구성 |
| 지지 근거 | 최소 1개 필수 |
| 반박 근거 | 존재하면 연결, 숨기지 않음 |
| 미해결 질문 | 아직 검증하지 못한 조건 |
| 검증 결과 | 현장 반응과 회수 관측 |
| 기록 상태 | 후보 / 공식 규칙 / 위험 사례 |

자유 문장 입력은 PoC 필수가 아니다. 기존 4지선다와 기록 연결을 유지하면서 플레이어가 근거 관계를 직접 구성하도록 한다.

### 4.5 가설 검증

```text
경쟁 가설 선택
→ 현장 대응
→ 실제 괴이 반응
→ 체력·괴이 위험도·제약 갱신
→ 지지·반박·미해결 근거 갱신
```

- 오답은 정답을 공개하지 않는다.
- 실패는 반응 단서와 배제 근거를 남긴다.
- 선택지는 `기존 가설 1개 + 직전 반응으로 생긴 신규 변형 가설 1개`로 부분 갱신한다.
- 검증 반복은 가능하지만 위험도와 현상 심화로 사실상 제한된다.
- 위험도 최대에서 추가 검증을 막고 긴급 회수에 진입한다.

## 5. 지식 모델

### 5.1 괴이 매뉴얼 지식 - 영구

- 공식 규칙
- 관측 패턴
- 위험 사례
- 연구 확정 정보
- 괴이 전용 대응

### 5.2 현장 이해도 - 현재 출현별

| 단계 | 결정론적 승격 조건 | 회수 효과 |
|---|---|---|
| 미확인 | 유효한 규칙 근거 부족 | 전조 세부 해석 불가 |
| 단서 확보 | 지지 근거 1개 이상 | 낮은 확률로 관측 패턴 해석 |
| 유력 | 경쟁 가설 축소 + 반박 검토 | 높은 확률로 관측 패턴 해석 |
| 이해 | 규칙 문장·지지·반박·미해결을 설명하고 결정적 검증 완료 | 관측한 패턴 확정 해석 |

현장 이해도는 난수로 오르내리지 않는다. 재출현 때 영구 매뉴얼은 남지만 현재 장소·환경·변형에 대한 현장 이해도는 다시 쌓는다.

## 6. 논리와 확률의 분리

### 6.1 난수 금지 영역

- 단서 내용과 획득 계약
- 선택지의 논리적 모순
- 가설 카드의 지지·반박 관계
- 현장 이해도 승격
- 괴이의 실제 공격 패턴
- 포획 조건

### 6.2 난수 허용 영역

`단서 확보`와 `유력` 단계에서 이미 관측한 패턴의 세부 정보를 이번 턴에 해석하는가만 확률 판정을 사용한다.

| 판정 | 표시 |
|---|---|
| 실패 | `놈은 무언가 하려 한다.` |
| 성공 | `놈은 [행동명]을 하려 한다.` + 확인한 범위의 대상·범위·조건 |

- 거짓 행동명·대상·범위는 금지한다.
- 실제 패턴은 판정 전에 고정한다.
- `이해` 단계는 확정 해석한다.
- 저장·불러오기로 같은 판정을 재추첨하지 않는다.
- 판정 실패에도 최소 하나의 유효한 범용 대응이 존재한다.

## 7. 회수 페이즈

### 7.1 전투 형식

턴제 패턴 대응 전투다. 핵심 질문은 `이 전조가 무엇을 의미하며 이번 턴에 무엇을 지킬 것인가`다.

### 7.2 행동 문법

| 행동 | 역할 |
|---|---|
| 관찰 | 다음 패턴·상태를 더 읽지만 위험을 감수 |
| 방어·보호 | 불명 전조와 미관측 패턴의 피해 완화 |
| 지원 | 체력·위치·장비·봉쇄 조건 보조 |
| 규칙 대응 | 조사에서 확인한 금기·조건에 맞춘 전용 행동 |
| 봉쇄·포획 | 올바른 대응 누적 뒤 열린 포획 창 사용 |

공격력 경쟁을 중심으로 하지 않는다. 체력은 실패 예산이며 승리 조건은 괴이 HP 0이 아니라 포획 창을 여는 것이다.

### 7.3 미관측 핵심 패턴

- 첫 발동은 행동명을 확인할 수 없다.
- 미관측 핵심 전조임은 별도로 표시한다.
- 범용 방어·엄폐·보호 중 최소 하나는 항상 유효하다.
- 최초 관측 손실은 회복 가능해야 한다.
- 즉사, 강제 중상, 캠페인 소프트락, 히든 분기 영구 실패를 금지한다.
- 실제 발동 뒤 패턴을 매뉴얼에 등록하고 두 번째부터 이해도 판정을 적용한다.

### 7.4 포획 창

- 올바른 대응은 괴이의 안정도·봉쇄 조건·행동 가능성을 변화시킨다.
- 사건별 2~4개의 대응 조건을 충족하면 포획 창이 열린다.
- 잘못된 대응은 체력·위험도 비용과 위험 사례를 남긴다.
- 긴급 회수는 낮은 결과 등급과 잔류 위험을 감수하는 폴백이다.

## 8. 실패·부상·결과 등급 - CORE_SUPPORT

### 8.1 실패 비용

- 체력 감소
- 권나래·서포트 부상
- 괴이 위험도 상승
- 현상 심화와 행동 제약
- 사건 결과 등급 하락
- 위험 사례와 배제 근거 기록

실패는 정보 없이 자원만 잃게 하지 않는다.

### 8.2 권나래

- 고정 주인공이며 교체 기능이 없다.
- 권나래의 일정만 진행·소비한다.
- 경상은 위험 관리와 결과 등급에 작은 불이익을 준다.
- 중상도 강행 가능하지만 추가 부상과 낮은 등급 위험이 커진다.
- 반일 회복으로 빠르게 한 단계 완화한다.
- 다른 일정을 진행해도 약한 자동 회복이 누적된다.
- 부상으로 가능한 행동이 모두 사라지는 소프트락은 금지한다.

### 8.3 결과 등급

| 축 | 평가 |
|---|---|
| 회수 품질 | 포획, 임시 봉합, 잔류 위험 |
| 피해 관리 | 권나래·서포트·민간인 피해, 무리한 강행 |
| 지식 품질 | 가설 정확성, 지지·반박 근거, 공식·후보·위험 사례 구분 |

회수 성공과 정확한 이해를 분리한다.

## 9. 서포트 요원 계약 - CORE_SUPPORT

- 서포트는 최대 2명이며 독립 일정을 소비하지 않는다.
- 정답·필수 단서·최적 행동을 독점하지 않는다.
- PoC 이후 각 요원은 `관찰 질문 1개`, `위험 해석 1개`, `결과 반응 1개`로 차별화한다.
- 수치는 피해·지원 강도와 실패 복구만 조절한다.
- 자동 최적 요원 선택과 예측률 정답 보너스는 금지한다.

## 10. 포획·연구·히든 분기 - CORE_SUPPORT

### 10.1 포획·연구

- 회수 성공 괴이는 일정 기간 `포획` 상태가 되어 사건 배치에서 제외된다.
- 포획 기간에 매뉴얼 정보를 확정하고 연구도를 축적한다.
- 중요 연구 단계에서만 괴이 전용 대응을 해금한다.
- 포획 종료 뒤 같은 괴이가 다시 발현할 수 있으나 현재 출현은 새로 조사한다.

### 10.2 히든 분기

```text
포획 연구
→ 재출현 조사에서 히든 선택지
→ 조건 성립
→ 회수 전투의 전용 행동
→ 새로운 장면·기록·결과
```

- 일반 회수보다 반드시 높은 결과 등급이나 자원을 주지 않는다.
- 괴이 구제·민간인 보호·세계관 진실에는 유리할 수 있다.
- 기관 평가·포획 안정성·자원 중 일부를 교환하는 대안 결과로 설계한다.

## 11. 캠페인·의뢰·엔딩 - CORE_SUPPORT

### 11.1 기간제 챕터

- 제한된 반일 일정과 핵심 사건 마감일을 가진다.
- 마감 초과 시 불리한 긴급 조사·회수에 진입한다.
- 조기 해결 뒤 신규 대형 사건 없이 회복·연구·후일담을 진행한다.

### 11.2 챕터 사건 구성

- 핵심 사건 1개
- 연계 의뢰 최대 1개
- 독립 세력 의뢰 1개

의뢰는 이해도·전조 관측·피해자 구출 상태에 반응할 수 있지만 핵심 사건의 정답·필수 단서·이해도를 직접 지급하지 않는다.

### 11.3 미니게임 재활용

`route restore`, `rain dodge` 등은 다음 공통 인지 문법을 만족할 때만 재사용한다.

```text
전조 관측 → 규칙 적용 → 즉시 결과
```

조작 외형이 코어 인지 문법과 맞지 않으면 재활용하지 않고 `DEFER`한다.

### 11.4 보상

- 세력별 기본 신뢰도
- 소량의 잔향 파편 또는 연구도
- 우수 등급 선택 보너스: 잔향 파편 / 연구도 / 추가 신뢰도

의뢰 반복이 핵심 조사와 연구를 대체하지 않는다.

### 11.5 엔딩

- 괴이 구제
- 민간인 보호
- 기관 명령 준수
- 진실 공개

선악 점수가 아니라 반복한 책임 우선순위를 반영한다. 모든 히든 분기를 요구하는 단일 진엔딩은 금지한다.

## 12. UI·접근성

### PoC 기본 화면 우선순위

| 화면 | 전면 정보 | 상세 서랍 |
|---|---|---|
| 조사 | 현재 전조, 4개 선택지, 관련 매뉴얼, 가설 카드, 최소 체력·위험 | 장비, 세력, 상세 수치, 관계 |
| 회수 | 전조, 해석 결과, 대응 의도, 직전 결과, 포획 조건 | 세부 능력값, 자원 상세, 기록 전체 |
| 결과 | 회수 품질, 피해, 지식 품질, 매뉴얼 변화 | 전체 보상·통계 |

- 긴 한국어 자동 줄바꿈과 1280×720·1920×1080 가독성을 검증한다.
- 색상·음향만으로 전조와 위험을 전달하지 않는다.
- 입력 시간 제한이 있으면 일시정지·연장·비활성 대안을 제공한다.
- UI·컷인·표정은 상태를 표현하지만 진행을 대신 소유하지 않는다.

## 13. MDA·DDD 가드레일

### MDA

- Mechanics: 기록 연결, 가설 카드, 위험 검증, 이해도, 전조 해석, 대응, 포획, 매뉴얼
- Dynamics: 불확실성 축소 → 결단 → 반응 관찰 → 갱신 → 전조 예측 → 포획
- Aesthetics: 간파, 긴장, 숙련, 기록 소유감, 구제

### DDD

- 첫 의미 있는 보상은 5분 내 오답 1개 배제
- 기록 연결 결과는 즉시 표시
- 가설 검증은 한 장면 이내 피드백
- 조사 지식의 전투 보상은 다음 회수 턴 이내 확인
- 수치 팝업·랜덤 보상·수집률로 코어 보상을 대체하지 않는다.

## 14. 저장·호환 위험

새 코어는 캠페인·경제·엔딩 의미를 변경할 수 있는 고위험 목표다.

- CORE-MVP-001은 기존 세 사건과 저장을 직접 마이그레이션하지 않는 독립 PoC 경로를 사용한다.
- 현행 `mvp-039`와 `mvp-038` 이관을 보호한다.
- 실제 통합 전에 새 저장 버전, ID, 기본값, 이관, 롤백을 별도 승인한다.
- 보호 경로: `scripts/core/game_state.gd`, `data/episodes`, `project.godot`, `knowledge/base-pack`

## 15. MVP 재기준화

### CORE-MVP-001 - 현재 우선 작업

- 사건 1개, 18~25분
- 조사 장면 3개
- 핵심 단서 6개: 지지 3, 반박 2, 미해결 1
- 4개 선택지와 직접 배제 2개
- 규칙 가설 카드 2개
- 현장 이해도 4단계
- 회수 패턴 3개: 관측 가능 2, 미관측 핵심 1
- 턴제 회수 5~8턴
- 정상 포획 / 비용 있는 포획 / 긴급 포획
- 지원 시스템은 체력과 범용 방어만 사용

### CORE-MVP-002 - PoC 통과 뒤

- 권나래·서포트 부상
- 회복 일정과 약한 자동 회복
- 3축 결과 등급
- 포획 상태와 연구 2단계
- 괴이 전용 대응 1개

### CORE-MVP-003 - 선택 비용 검증

- 제한 반일 챕터 1개
- 핵심 사건 1개
- 연계 의뢰 1개, 독립 의뢰 1개
- 공통 인지 문법을 만족하는 기존 미니게임만 재활용
- 조기 해결 정리 기간과 마감 긴급 회수

### CORE-MVP-004 - 장기 보상 검증

- 연구 기반 조사 히든 선택지
- 회수 전용 행동
- 일반/히든 결과 비교
- 가치관 축 최소 2개
- 캠페인 결말 2종 이상

## 16. 플레이테스트·Production gate

### 사전 선언 지표

- 80% 이상이 사건 규칙과 대응 이유를 자기 말로 설명
- 70% 이상이 무작위 시도 없이 올바른 근거로 선택지 1개 이상 배제
- 70% 이상이 조사가 회수 전투를 읽기 쉽게 만들었다고 행동·인터뷰에서 확인
- 핵심 실패를 난수 탓으로 설명하는 비율 20% 이하
- 80% 이상이 매뉴얼을 자신이 만든 기록으로 인식
- 미관측 핵심 패턴에서 대응할 방법이 없었다는 보고 20% 이하

### 판정

- 통과: `KEEP / AMPLIFY`, CORE-MVP-002 진입
- 규칙 설명 통과·전투 연결 실패: 전조 UI·포획 행동 `CHANGE / RETEST`
- 매뉴얼이 자동 정답으로 인식: 가설 카드·근거 연결 `CHANGE / RETEST`
- 난수 불공정 반복: 중간 단계 확률 축소 또는 후보 정보 방식 `CHANGE`
- 조사와 회수가 별개로 인식: `HOLD`, 지원 시스템 확장 금지
