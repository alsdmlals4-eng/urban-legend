# Ver 3.5 로그 AI 동료 설계

## 목표

괴담기록국의 안내 AI `로그`를 텍스트 안내 패널이 아닌 등장인물로 승격한다. 로그는 전자기기 데스크와 현장 단말기를 통해 요원 대화에 자연스럽게 참여하고, 첫 튜토리얼·기록 대조·위험 경고를 맡는다.

플레이어가 얻어야 할 경험은 다음과 같다.

- 새 시스템을 별도 도움말보다 세계관 속 대화로 배운다.
- 로그를 사건을 함께 겪는 기록국 구성원으로 인식한다.
- 단서와 전조를 정리받되 정답이나 미확보 정보를 누설받지 않는다.
- 반복 플레이에서는 같은 안내가 흐름을 끊지 않는다.

## 캐릭터 방향

- 정체: 괴담기록국이 운용하는 기록·분석·통신 보조 AI
- 거점: 기록국 전자기기 데스크의 전용 단말기
- 현장 연결: 요원 휴대 단말기나 현장 모니터로 접속
- 성격: 친근하고 약간 능청스럽지만 위험 상황에서는 즉시 정확하고 진지해진다.
- 역할: 조작법 안내, 편성·장비 브리핑, 단서 대조, 전조 기록 비교, 결과 요약
- 금지: 정답 선택 지시, 미확보 단서 공개, 요원의 감정 장면을 끊는 농담, 같은 안내 반복

## 시각·음향 표현

제공된 참고 이미지에서는 `전자 화면 안의 단순한 픽셀 얼굴`이라는 원리만 가져온다. 금색 외곽 구조, 고유 도형과 구도는 복제하지 않는다.

신규 로그 이미지는 낡은 기록국 전자 단말기의 청록빛 화면에 나타나는 독창적인 픽셀 얼굴로 제작한다. 게임에서는 다음 세 표정을 사용한다.

- `normal`: 평상시 안내와 가벼운 농담
- `focus`: 기록 대조와 분석
- `warning`: 괴이 전조와 위험 경고

정식 대화에서는 요원 초상과 같은 화자 이미지 슬롯을 사용한다. 메인·준비·시장·DB에서는 데스크 단말기 이미지가 크게 보이고, 현장·회수에서는 휴대 단말기 화면처럼 축소한다.

로그의 첫 발언 직전에는 고유한 3음 접속 시그니처와 화면 연출을 한 번 재생한다. 같은 음형을 유지하되 상황에 따라 속도·음색·화면 색을 바꾼다.

- 평상시: 부드러운 3음 전자 차임, 커서 두 번 점멸, 픽셀 얼굴 조립
- 분석 시: 같은 음형에 짧은 라디오 잡음, 얼굴을 통과하는 스캔 라인, `focus` 표정 전환
- 위험 시: 낮고 빠른 경고 음색, 주황·적색 화면 반전, `warning` 표정 전환
- 통신 종료: 얼굴이 한쪽 눈을 찡긋하거나 작은 기록 아이콘으로 접히는 짧은 연출

전체 접속 연출은 첫 등장과 중요한 상태 변화에만 사용한다. 연속된 로그 대사에서는 초상 점등만 사용하고 효과음을 반복하지 않는다. 음향 없이도 화자명·표정·커서·색 변화로 같은 상태를 이해할 수 있어야 한다.

로그의 유머는 반복 효과음이 아니라 상황을 기록관답게 비트는 짧은 대사에서 표현한다. 위험 상황에서는 농담을 멈추고 즉시 핵심 경고를 전달한다.

## 참여 방식

`대본 참여 + 최초 시스템 안내` 혼합형을 사용한다.

### 대본 참여

사건 `field_nodes`의 대화 배열에서 `speaker: "로그"`를 일반 화자처럼 사용한다. 로그 대사는 사건의 공개된 단서와 현재 상태만 참조한다. 선택 후 3~5회 이어지는 대화에도 로그가 필요할 때 한 차례 참여할 수 있다.

### 시스템 안내

다음 상황의 첫 발생에만 짧은 로그 안내를 삽입한다.

- 새 게임과 이어하기
- 첫 요원 편성
- 첫 외부 접점과 시장 구매
- 첫 현장 선택과 기록 서랍
- 첫 단서 획득
- 첫 회수 전조·예측·오대응 학습
- 첫 사건 결과와 DB 기록

`seen_log_tutorial_ids`에 확인한 안내 ID를 저장한다. 같은 안내가 다시 필요한 경우에는 화면 가장자리의 한 줄 도움말로 축약하며 대화 진행을 중단하지 않는다.

## 화면별 역할

- 메인 메뉴: 첫 접속 인사, 새 게임과 이어하기 차이 안내
- 사건 준비: 사건 요약, 편성 부족, 장비·소모품·외부 접점 안내
- 시장: 관계 단계, 가격 변화, 장비와 소모품 차이 안내
- 조사: 기록국 연결 인사, 확보 단서 대조, 기록 서랍과 선택 결과 정리
- 회수: 전조 기록 비교, 예측 결과 전달, 과거 오대응 학습 회상
- 결과: 해결 등급·잔향 파편·새 기록 요약
- 기록국 DB: 새로 열린 기록과 과거 사건 비교 안내

로그는 화면마다 상시 떠 있는 팝업이 아니다. 대화할 때는 등장인물로 나타나고, 발언이 끝나면 장면의 시각적 중심을 요원과 괴이에게 돌려준다.

## 구조와 데이터

- 공통 `LogGuide` 표현 컴포넌트가 초상, 화자명, 표정, 점등, 알림음을 담당한다.
- 기존 대화 렌더러는 `speaker == "로그"`일 때 로그 이미지와 표정을 선택한다.
- `GameState`는 `seen_log_tutorial_ids` 조회·기록 API만 소유한다.
- 화면별 스크립트는 조건을 판단한 뒤 공통 컴포넌트에 안내 ID와 문장을 전달한다.
- 사건 고유 로그 대사는 에피소드 JSON에 두고 공통 시스템 안내는 별도 로그 안내 카탈로그에 둔다.
- 저장 마이그레이션에서 기존 저장은 빈 `seen_log_tutorial_ids`로 시작한다.

새 데이터나 API를 범용 대화 프레임워크로 확장하지 않는다. 기존 `field_nodes`, 대화 UI와 `GameState` 저장 구조를 최소 변경한다.

## 접근성·오류 처리

- 접속 시그니처 볼륨은 기존 효과음 설정을 따른다.
- 음소거 상태에서도 초상 점등과 화자명으로 동일 정보를 제공한다.
- 이미지 로드 실패 시 `로그` 이름과 텍스트 안내는 유지한다.
- 안내 ID가 이미 확인됐으면 자동 대화를 다시 열지 않는다.
- 저장 마이그레이션 실패 시 튜토리얼 확인 기록만 초기화하고 사건 진행은 보존한다.

## 버전과 문서

플레이어 노출과 저장 필드가 바뀌므로 다음 버전으로 갱신한다.

- 화면: `Ver 3.5`
- 문서: `MVP-035`
- 저장: `mvp-035`

벤치마킹 개선서에는 서사형 튜토리얼, 다이제틱 AI 동료, 반복 안내 억제, 알림음 접근성을 조사한다. 타 작품의 마스코트 외형·문구·성격을 복제하지 않는다.

## 검증 기준

- 로그 `normal/focus/warning` 이미지가 대화 화자 슬롯에 올바르게 표시된다.
- 평상·분석·위험 접속 연출이 같은 3음 모티프를 유지하면서 서로 구분된다.
- 첫 로그 발언과 중요한 상태 변화에만 전체 접속 연출이 실행되고 연속 대사에는 반복되지 않는다.
- 메인·준비·시장·조사·회수·결과·DB에서 각 역할의 첫 안내가 한 번만 표시된다.
- 저장·이어하기 후 확인한 튜토리얼이 다시 자동 재생되지 않는다.
- 로그가 미확보 단서와 정답 대응을 말하지 않는다.
- 음소거 및 이미지 누락 상태에서도 모든 안내를 읽을 수 있다.
- 기존 요원 대화, 3~5회 후속 대화, 기록 서랍과 전조 대응 흐름이 유지된다.
- 1280×720과 1920×1080에서 로그 초상이 장면·선택지·요원 이미지를 가리지 않는다.
- Godot 헤드리스 파싱, 기존 테스트, 신규 MVP-035 테스트와 주요 씬 실행을 통과한다.

## 공용 Base 승격 후보

- 튜토리얼 캐릭터는 세계관 속 역할과 시스템 안내 책임을 일치시킨다.
- 최초 상세 안내와 반복 축약 안내를 분리하고 확인 상태를 저장한다.
- 음향 신호는 시각·텍스트 신호를 함께 제공한다.
- 안내자는 플레이어가 아직 획득하지 않은 정보나 정답을 누설하지 않는다.
