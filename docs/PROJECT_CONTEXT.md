# Project Context

## 한 줄 정의

`괴담기록국`은 요원 팀이 상황을 읽고 조사 방식을 선택해 괴이의 규칙을 밝힌 뒤 안정화·회수하는 PC용 현대 오컬트 수사 어드벤처다.

## 현재 기준

- MVP-038 / Ver 3.8 / 저장 스키마 mvp-038
- 최대 10일, 오전·오후 순차 반일 캠페인
- 구현 사건: 저승역, 비 오는 골목의 빨간 우산
- 계획 사건: 폐주파수 방송국
- 두 구현 사건의 10일 캠페인 QA 완료; 다음 P0은 `MVP-040 폐주파수 방송국` 세로 슬라이스 설계다.
- 세력: 소문시장, 마도회, 퇴마사 계보
- 요원 능력: 제압·분석·방호·치료·교감
- 오현은 캠페인 QA 동안 `balanced` 성향의 임시 3인 편성을 허용한다. 저장 스키마와 사건 데이터는 바꾸지 않는다.

## 플레이 경험

- 메인 조사는 `상황 설명 → 선택 → 짧은 결과`가 중심이다.
- 플레이어는 고정 주인공이 아니라 요원 팀을 운용한다.
- 괴이는 처치하지 않고 단서로 규칙을 이해해 안정화·회수한다.
- 실패는 피해와 함께 다음 판단에 쓸 패턴 학습을 남긴다.
- 일상 에피소드는 선택형이며 필수 단서·회수·엔딩 진행을 대체하지 않는다.
- 목표 플레이타임은 필수 125~155분, 선택 콘텐츠 포함 180~220분이며 아직 실측 전이다.

## 표현 원칙

- UI는 대시보드보다 배경·캐릭터·현재 행동이 먼저 보이는 장면 중심 구조를 사용한다.
- `로그`는 전문성 70·귀여움 30의 디지털 유령 마스코트다.
- 대사는 일상에서 풍부하게, 조사에서는 짧게 사용한다.
- 피해·사망·후유증은 농담 소재로 삼지 않는다.

## 상태와 호환성

- `HIGH_RISK_DOMAIN`: 저장, 경제, 진행, 엔딩, 에피소드 의미 변경은 의미론적 고위험이다. 이는 파일 경로 자체를 보호하는 보호 경로와 구분한다.
- UI 표현은 가능한 기존 상태에서 계산하고 불필요한 저장 필드를 추가하지 않는다.
- 자동행동과 장비는 플레이어 선택이나 핵심 추리를 대체하지 않는다.
