# Project Context

## 한 줄 정의

`도시괴담 기록국`은 요원 팀이 상황을 읽고 조사 방식을 선택해 괴이의 규칙을 밝힌 뒤 안정화·회수하는 PC용 현대 오컬트 게임이다.

## 플레이 경험

- 메인 조사는 `상황 설명 → 선택 → 짧은 결과`가 중심이다. 로그는 시스템 메시지를 대신해 필요한 핵심 설명을 하며, 편성 요원은 조건에 맞을 때 1~2마디 반응한다.
- 일상 에피소드는 요원 2~3명의 관계와 세계관을 보여주는 선택 콘텐츠다. 건너뛰어도 필수 단서, 조사, 회수, 엔딩 진행에 영향이 없어야 한다.
- 플레이어는 고정 주인공 한 명이 아니라 요원 팀을 운용한다. 대표 요원과 지원 요원 역할을 화면과 규칙에 유지한다.
- 괴이는 처치 대상이 아니다. 단서로 규칙을 이해하고 위험을 관리해 안정화·회수한다.
- 목표 플레이타임은 필수 진행 약 125~155분, 일상·연구·세력 콘텐츠 포함 약 180~220분이다.

## 표현 원칙

- UI는 대시보드보다 배경·캐릭터·현재 행동이 먼저 보이는 장면 중심 구조를 사용한다.
- `로그`는 스마트폰 앱 속 디지털 유령 마스코트다. 전문성 70·귀여움 30으로 말하며 정답이나 미확보 정보를 누설하지 않는다.
- 대사는 일상에서 풍부하게, 조사에서는 짧게 사용한다. 피해·사망·후유증은 농담 소재로 삼지 않는다.

## 상태와 호환성

- 저장, 경제, 진행, 엔딩, 에피소드 JSON은 보호 상태다. 기존 저장 이관과 현재 진행 흐름을 먼저 확인한다.
- UI 표현 모드는 가능한 한 기존 진행 상태에서 계산하고 불필요한 저장 필드를 추가하지 않는다.
- 위험도 변화는 모든 괴이에 일괄 적용하지 않고 한 번에 1~2개를 기본으로 한다.
- 일상 이해도 보너스는 선택형 소량 보상이며 누적 상한을 둬 필수 노가다가 되지 않게 한다.

## 제작 역할

- DeepSeek: 저장소 조사, 반복 분석, 단순 비보호 초안, 구조·누설 1차 검사
- 외부 GPT: 대사, 튜토리얼, 캐릭터 감정선, 장문 기획과 UX 문구
- 이미지 모델: 콘셉트와 게임용 래스터 자산
- Codex: 작업 분류, 핵심 상태, 실제 diff, 충돌 조정, 실행 검증과 최종 품질

모든 외부 결과는 patch·review·manifest 계약으로 받고 Codex가 적용 여부를 결정한다.
