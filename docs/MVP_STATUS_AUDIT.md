# MVP Status Audit

## 점검 일자

2026-07-10

## 점검 목적

MVP-021 기준으로 메인 UI와 사건 준비 UI의 현재 구조를 확인하고, 두 사건 샘플 이후 시스템/UI 디벨롭 단계의 첫 작업 기준을 정리한다.

## 결론 요약

현재 큰 방향의 충돌은 없다.

MVP-021은 새 괴담 에피소드 추가가 아니라 `메인 UI / 사건 준비 UI 리디자인 1차`로 진행한다.

핵심 판단:

```text
플레이어가 다음에 무엇을 눌러야 하는지 망설이지 않고,
현재 사건·요원·장비·기록물 상태를 한눈에 이해해야 한다.
```

## MVP-021 UI 진단

| 화면 | 현재 상태 | 개선 방향 |
|---|---|---|
| 메인 메뉴 | 제목, 소개, DB, 새 게임, 이어하기, 저장 초기화, 테스트 버튼이 한 흐름에 있음 | 주요 행동과 개발/테스트 버튼 분리 |
| 메인 메뉴 상태 표시 | 저장 상태와 요원 상태가 텍스트 중심 | 시작 가능/불가능 이유를 더 읽기 쉽게 표시 |
| 사건 준비 화면 | 사건 선택, 요원, 장비, 기록물, 로그, 시작이 한 스크롤에 있음 | 카드/섹션 단위로 정보 위계 정리 |
| 장비/기록물 | 효과 텍스트는 있으나 현재 사건에 왜 유용한지 즉시 파악하기 어려울 수 있음 | 현재 사건 기준 설명 강화 |
| 모바일 세로 화면 | 스크롤 기반 구조 | 버튼 간격, 제목, 섹션 순서 점검 필요 |

## 유지할 흐름

- 요원 2명 이상 선택해야 시작 가능
- 새 게임 / 저승역 시작
- 이어하기
- 저장 초기화
- 기록국 DB 진입
- 준비 화면에서 사건 선택
- 장비 장착/해제
- 기록물 표시
- 조사 시작

## 제외할 범위

- 새 괴담 에피소드 추가
- 조사 화면 UI 개선
- 사건 보고서/DB UI 고도화
- 요원 시스템 확장
- 장비/기록물 효과 시스템 변경
- 저장 구조 변경
- 대형 아트/이미지 추가
- 복잡한 애니메이션 연출

## Base 승격 후보

- UI 리디자인 MVP는 기능 추가보다 정보 위계와 기존 흐름 보존을 우선한다는 원칙
- 개발/테스트 버튼을 플레이어 주요 행동과 분리하는 원칙

## 프로젝트 전용으로 유지할 내용

- 기록국 DB / 사건 준비 / 요원 편성 / 장비 / 기록물의 세계관 표현
- 저승역과 빨간 우산 2개 사건 기준
- Godot 파일 경로와 씬 구조

## 남은 위험

- 실제 Godot 화면에서 모바일 세로 기준 가독성은 로컬 실행으로 확인해야 한다.
- 코드가 생성형 UI 방식이므로 작은 레이아웃 수정도 버튼 연결을 깨뜨리지 않게 확인해야 한다.
- MVP-022 조사 화면 UI 개선과 범위가 섞이지 않도록 한다.
