# MVP Status Audit

## 점검 일자

2026-07-10

## 점검 목적

MVP-021 기준으로 메인 UI와 사건 준비 UI의 현재 구조를 확인하고, 두 사건 샘플 이후 시스템/UI 디벨롭 단계의 첫 작업 기준을 정리한다.

## 결론 요약

현재 큰 방향의 충돌은 없다.

MVP-021은 새 괴담 에피소드 추가가 아니라 `메인 UI / 사건 준비 UI 리디자인 1차`로 진행한다.

프로젝트의 1차 출시 목표는 **PC 기준 Steam 출시**다. 따라서 UI 판단은 모바일 세로가 아니라 PC 16:9, 마우스/키보드, Steam 데모 첫인상 기준을 우선한다.

핵심 판단:

```text
플레이어가 다음에 무엇을 눌러야 하는지 망설이지 않고,
현재 사건·요원·장비·기록물 상태를 한눈에 이해해야 한다.
```

- 조사 화면 UI 개선
- 사건 보고서/DB UI 고도화

- 모바일 세로 화면 최적화

## Base 승격 후보

- GPT가 매 작업에서 벤치마킹을 수행하고 개선점/수정점을 압축 반영하는 원칙
- UI 리디자인 MVP는 기능 추가보다 정보 위계와 기존 흐름 보존을 우선한다는 원칙
- 개발/테스트 버튼을 플레이어 주요 행동과 분리하는 원칙

## 프로젝트 전용으로 유지할 내용

- PC 기준 Steam 출시 목표
- Unity Asset Store / Tales of Tuscany Demo 고정 참고 링크
- 기록국 DB / 사건 준비 / 요원 편성 / 장비 / 기록물의 세계관 표현
- 저승역과 빨간 우산 2개 사건 기준
- Godot 파일 경로와 씬 구조

## 남은 위험

- 실제 Godot 화면에서 PC 16:9, 창 모드, 전체화면 기준 가독성은 로컬 실행으로 확인해야 한다.
- 코드가 생성형 UI 방식이므로 작은 레이아웃 수정도 버튼 연결을 깨뜨리지 않게 확인해야 한다.
- MVP-022 조사 화면 UI 개선과 범위가 섞이지 않도록 한다.
