# MVP Status Audit

## 점검 일자

2026-07-11

## 점검 목적

MVP-028 기준으로 PC/Steam 데모용 대화·조사·회수 UX를 정리하고, 조사 페이즈의 텍스트 노벨형 선택 진행과 요원 팀 기반 회수 UI 기준을 확정한다.

## 결론 요약

현재 큰 방향의 충돌은 없다.

MVP-028은 `PC/Steam UX·텍스트 노벨형 조사·팀 기반 회수 UI 정리`로 진행한다.

프로젝트의 1차 출시 목표는 **PC 기준 Steam 출시**다. 따라서 UI 판단은 PC 16:9, 마우스/키보드, Steam 데모 첫인상 기준을 우선한다.

핵심 판단:

```text
플레이어는 별도 주인공 1명을 조작하는 것이 아니라,
괴담기록국 요원 팀을 이끌며 상황을 읽고 선택하고 단서를 근거로 회수한다.
```

## MVP-028 UX 진단

| 화면 | 목표 구조 | 적용 기준 |
|---|---|---|
| 대화 화면 | 인물/현장 몰입 + 하단 대사창 + 주변부 보조 UI | 대화 읽기와 사건 분위기 우선 |
| 조사 화면 | 상황 텍스트 → 선택지 → 결과 텍스트 → 단서/상태 변화 | 텍스트 노벨형 선택 진행 |
| 회수 화면 | 좌상단 아군 요원 / 우상단 해결 단서 / 중앙 회수 연출 / 하단 행동 선택 | 전투가 아니라 팀 기반 안정화/회수 절차 |
| 요원 구조 | 별도 주인공 없음 | 플레이어는 괴담기록국 요원 팀을 운용 |
| 대표 요원 | 선택 요원 중 회수 담당 교체 가능 | 파티 전투가 아니라 현장 지휘/회수 담당 전환 |

## 유지할 흐름

- 저승역과 빨간 우산 2개 사건 흐름
- 요원 선택과 팀 기반 반응
- 조사 선택지, 단서/힌트, 상태 변화
- 괴이 안정화/회수 페이즈
- 완료 보고서와 기록국 DB
- 저장/이어하기/저장 초기화

## 제외할 범위

- 새 괴담 에피소드 추가
- 새 미니게임 추가
- 신규 캐릭터 대량 추가
- 대형 일러스트/애니메이션 제작
- 복잡한 파티 전투 시스템
- 모바일 세로 화면 최적화
- Steam 상점 페이지 실제 제작
- 전체 UI 프레임워크 교체

## Base 승격 후보

- 대화/조사/회수처럼 화면 역할이 다른 경우, 각 화면의 UX 기준을 별도로 정의한 뒤 전체 시선 흐름을 통일하는 원칙
- 고정 주인공이 없는 팀 기반 게임은 UI에서 `주인공 발화`보다 `팀 투입/요원 기여`를 우선 표현해야 한다는 원칙

## 프로젝트 전용으로 유지할 내용

- 별도 주인공 없음 / 괴담기록국 요원 팀 중심 진행
- 회수 페이즈 좌상단 아군 요원 / 우상단 해결 단서 구조
- 대표 요원 교체를 현장 지휘/회수 담당 전환으로 표현하는 기준
- 기록국 DB / 조사 포인트 / 단서 / 힌트 / 회수 표현
- 저승역과 빨간 우산 2개 사건 기준
- Godot 파일 경로와 씬 구조

## 남은 위험

- 텍스트 노벨형 조사로 전환하면서 기존 조사 포인트/조사 방법 데이터 구조를 과도하게 갈아엎을 위험이 있다.
- 대표 요원 교체가 복잡한 파티 전투 시스템으로 확대될 위험이 있다.
- 사용자 제공 이미지는 배치 원리 참고용이며, 이미지 스타일이나 특정 UI 스킨을 그대로 복제하면 안 된다.
- Issue #32~#35는 오생성 테스트 이슈로 닫혔으며 사용하지 않는다.
