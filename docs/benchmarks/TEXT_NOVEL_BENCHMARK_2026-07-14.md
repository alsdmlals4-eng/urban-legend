# 텍스트 노벨 벤치마킹과 MVP-041

## 범위와 검증 방법

- 확인일: 2026-07-14
- 1차 참고작: `활협전`, `화산의 딸`, `서울 2033`, `The Winter Tower`.
- 근거: 공식 스토어·Google Play 설명과 플랫폼 내 사용자 반응 표본. 평점·리뷰 수는 확인 시점의 표본이며 순위가 아니라 설계 가설을 점검하는 보조 근거다.
- DeepSeek `research-scout-flash` R0 읽기 전용 반대검토를 별도 수행했다. 저장·코드·문서 변경은 없었으며, 최종 판단은 Codex가 공식 출처와 실제 구현을 대조해 결정했다.
- 이번 MVP의 구현 범위: 세 사건 캠페인 QA와 문서 갱신. 게임 데이터, 저장 스키마, `GameState`, 에피소드 JSON은 변경하지 않는다.

## MVP-041 텍스트 노벨 UX 계약

| 시점 | 플레이어가 알아야 할 것 | 현재 구현의 근거 | 상태 |
|---|---|---|---|
| 반일·사건 선택 전 | 현재 반일, 선택 가능한 활동, 조사에 필요한 팀 배치와 시간 소모 | 순차 캠페인 일정과 준비 화면 | 구현 확인 |
| 조사 선택 전후 | 현재 상황, 확보 근거, 추론 방향, 다음 판단 | 조사 결과 블록 | 구현 확인 |
| 회수 선택 전후 | 전조, 연결 단서, 오대응 이유, 다음에 다시 볼 근거 | 회수 근거 서랍과 패턴 학습 | 구현 확인 |
| 사건 종료 뒤 | 확보 근거, 회수 결과, 요원 기여, 다음 판단 | 결과 `ReasoningSummary`, 사건 보고서·DB | 구현 확인 |

즉시 결과는 숫자 팝업이 아니라 장면·보고서 문맥 안에서 보여준다. 실패를 다음 판단의 근거로 바꾸는 정책은 `화산의 딸`의 완화된 실패와 프로젝트의 공정 추리 원칙을 함께 따른 프로젝트 선택이며, 네 참고작 전체의 공통 규칙이라고 일반화하지 않는다.

## 독립 반대검토와 경계

- DeepSeek 검토는 이 네 작품이 "보고서·DB와 다음 판단" 형태의 재열람 시스템을 직접 뒷받침하지 않는다고 지적했다.
- 이 지적을 채택한다. 보고서·DB는 벤치마킹에서 복제한 기능이 아니라 `괴담기록국`의 사건 기록 정체성과 기존 구현이다. 본 문서는 이를 1차 참고작의 공통 시장 규칙으로 주장하지 않는다.
- 기존 `Slay the Princess`, `Citizen Sleeper`, `Roadwarden`, `Sir Brante`는 추리·시간·실패 표현의 **보조 참고**로 유지한다. 이번 텍스트 노벨의 1차 채택 판단에는 사용하지 않는다.
