# 멀티모델 파이프라인 20건 평가

평가 일자: 2026-07-14
대상: DeepSeek `scout-flash` 읽기 전용 실제 실행 20건

## 표본

- 초기 실제 통합 조사 2건
- 동일한 격리·수집 계약으로 실행한 소형 정확성 조사 18건
- 자동 테스트용 `async-test`, 가짜 GPT 산출물, 이미지 fixture는 20건 표본에서 제외
- 18건은 각기 다른 파일의 정확한 `CODE` 값을 요구했고 보고서에 정답이 포함되는지 Codex가 확인

이 표본은 R0 조사 파이프라인의 실행 안정성을 평가한다. GPT 대사, 이미지 생성, R1/R2 구현을 섞은 생산 작업 표본은 아니므로 혼합 작업 전체의 절감률로 과대 해석하지 않는다.

작은 조사도 고정 에이전트 문맥 때문에 약 1.2만 토큰의 중앙값을 사용한다. 따라서 한 파일 20줄 이하의 명확한 수정이나 단일 사실 확인은 Codex가 직접 처리하고, 여러 파일 조사·목록·호출 관계처럼 문맥 회수가 큰 작업만 DeepSeek로 보내는 현재 임계값을 유지한다.

## Codex 토큰 절감 판단

Codex의 반사실 토큰 사용량을 동일 조건으로 재실행하지 않았기 때문에 절감률을 직접 측정할 수는 없다. 다만 20건 중 19건은 Codex가 원문 조사나 결과 문장을 작성하지 않고 계약 작성·정답 검수만 수행했다. R0 조사 구간의 작성 부담 감소는 매우 크지만, 실제 제작에는 Codex 구현과 외부 GPT·이미지 검수가 섞인다.

따라서 혼합 작업의 현재 운영 추정치는 **약 50%(범위 40~60%) 절감**으로 유지한다. 다음 재평가는 실제 생산 작업 20건을 provider와 위험도별로 나눠 수행한다.
