## 2026-07-13 MVP-034 공용 Base 반영안

- 전: 장비를 사건 결과에서 자동 해금했다. 후: 핵심 추리를 소모하지 않는 보조 장비만 별도 경제로 선택 구매한다.
- 전: 세계관 세력은 소개 데이터였다. 후: 관계 임계값으로 이벤트·의뢰·상품·가격을 해금한다.
- 전: 재화를 단순 가산할 수 있었다. 후: 모든 지급은 고유 보상 ID로 멱등 처리한다.
- 전: 소모품 반입 제한이 없었다. 후: 플레이 전 선택 가능한 종류·수량 상한을 명시한다.

프로젝트 고유 세력명·상품명·가격·보상량은 공용 Base 승격 대상에서 제외한다.

- 전: 대화와 조사를 화면 단위로 분리했다. 후: 플레이 경험이 연속될 때 장면 노드로 통합하고 기존 저장 진입점만 호환한다.
- 전: 예측 성공만 감쇠했다. 후: 성공 연속 억제와 실패 연속 구제를 함께 두며 실패 시 거짓 정보를 제공하지 않는다.
- 전: 실패는 피해 적용 중심이었다. 후: 실패 원인과 다음 판단에 사용할 학습 기록을 함께 남긴다.
- 전: 동료는 선택 수치 보정 중심이었다. 후: 플레이어 선택을 대체하지 않는 타이밍별 자동 보조로 운용한다.

프로젝트 고유 패턴명·수치·대사는 공용 Base 승격 대상에서 제외한다.

검증된 프로젝트 규칙을 다음 공용 규칙 후보로 기록한다. Base 원격 기준 커밋은 변경하지 않았으며, 원격 승격 시 사용자에게 이 전/후 차이를 보고한다.

- 전: 플레이어 노출 변경과 표시/저장 버전 갱신의 연결 기준이 명시되지 않았다.
  후: 플레이어가 확인할 수 있는 기능·UI·데이터 구조가 바뀌면 표시 버전과 저장 스키마 버전의 갱신 여부를 함께 심사한다.
- 전: 능력치 표시 범위에 대한 공용 원칙이 없었다.
  후: 능력치는 판단에 필요한 선택 순간에만 노출하고, 상시 노출은 프로필·준비 화면으로 제한한다.
- 전: 실패 후 진행 가능 여부만 확인했다.
  후: 조사 실패에도 관찰 정보 또는 다음 대응 근거를 남겨 학습 가능한 실패로 설계한다.

## 2026-07-13 MVP-035 공용 Base 반영안

- 전: 튜토리얼 안내자를 기능성 팝업으로만 다룰 수 있었다. 후: 서사형 안내자는 담당 시스템과 맞는 세계관 내 역할을 가진다.
- 전: 상세 안내의 반복 기준이 없었다. 후: 최초 상세 안내와 재방문용 짧은 안내를 분리하고 확인 상태를 저장한다.
- 전: 알림음만으로 상태를 전달할 수 있었다. 후: 음향 신호는 동등한 시각 또는 문구 신호와 함께 제공한다.
- 전: 안내자의 정보 공개 범위가 명시되지 않았다. 후: 플레이어가 아직 획득하지 않은 단서·정답·숨은 수치를 안내자가 누설하지 않는다.

프로젝트 고유 캐릭터명·대사·음정·에셋은 공용 Base 승격 대상에서 제외한다.

## 사용 규칙

일상적인 urban-legend 작업에서는 Base 원격 링크 대신 이 저장소의 공용 로컬 사본을 먼저 읽는다. Base 원격은 공용 규칙 동기화, 기준 차이 확인, 또는 공용 규칙 개선 작업에서만 확인한다.

단, 사용자 최신 지시로 Base 원본 확인이 요구되었거나 공용 규칙을 갱신하는 작업에서는 GitHub Base 원본을 먼저 확인한다.

## 2026-07-10 Base 공용 규칙 갱신 기록

이번 업데이트는 Base 원본의 공용 규칙을 직접 갱신했다.

- GPT가 매 작업에서 벤치마킹을 수행하게 한다.
- 벤치마킹에서 게임성, 디자인, UX, 아이디어, 시장성, 구현 방식 중 필요한 항목을 확인하게 한다.

- 프로젝트별 고정 참고 링크를 매 작업에서 확인하게 한다.

도시괴담 기록국 로컬 반영:

- 모바일 세로 화면 기준은 기본 목표에서 제외하고, 별도 요청 시에만 포함한다.
