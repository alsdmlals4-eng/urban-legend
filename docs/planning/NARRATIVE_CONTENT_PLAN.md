# 괴이 기록국 서사·대화·관계 기획서

> 문서 위치: `docs/planning/NARRATIVE_CONTENT_PLAN.md` | 프로젝트 방향: `docs/planning/PROJECT_DIRECTION.md` | 실제 사건 데이터: `data/episodes/`

## 1. 목적

이 문서는 괴이 사건의 상황지시문과 기록관 아카 대사, 사건 후일담, 기록국 일상, 외부 세력 교류, 관계 연속 이벤트를 **하나의 서사 체계**로 연결한다.

목표는 설명문으로 세계관을 전달하는 것이 아니다. 인물들이 조사하고, 기록하고, 쉬고, 충돌하고, 서로의 선택을 기억하는 장면을 통해 캐릭터성과 조직의 가치관을 보여준다.

## 2. 서사의 두 속도

### 현장 조사 — 짧고 기능적

```text
상황지시문
→ 현재 판단 질문
→ 선택
→ 짧은 현장 결과
→ 아카의 기록 정리
→ 다음 질문
```

- 상황지시문이 장면의 감각·위험·변화를 먼저 보여준다.
- 대사는 기록관 아카 중심으로 짧게 사용한다.
- 서포트 요원은 능력 발동, 위험 경고, 결정적 관찰처럼 역할이 있는 순간에만 한 문장을 말한다.
- 조사 중 잡담과 장문 감정 대화를 넣지 않는다.
- 아카와 요원은 확보되지 않은 단서와 정답을 말하지 않는다.

### 일상·후일담 — 인물과 세계를 충분히 보여줌

```text
생활 또는 사건 후 상황
→ 인물 간 대화와 행동
→ 주인공 태도 선택 2~3개
→ 상대 반응과 선택 기억
→ 인물·조직·세계관 정보 1개 공개
→ 다음 관계나 사건으로 이어지는 짧은 마무리
```

- 한 장면에서 새로운 핵심 정보는 하나만 남긴다.
- 선택은 정답·오답보다 권나래가 어떤 태도를 취하는지를 고르게 한다.
- 보상은 능력치보다 기록 문장, 관계 태그, 후속 장면, 표정·대사 해금에 둔다.

## 3. MVP-044 승인 범위

MVP-044 전달 패키지는 다음을 기획 완료 상태로 다룬다. GitHub `main` 구현 완료를 의미하지 않는다.

### 메인 괴이 3편

1. 저승역
2. 비 오는 골목의 빨간 우산
3. 폐주파수 방송국

각 사건에 다음을 보강한다.

- 콜드 오픈과 장면 진입 상황지시문
- 확보 정보에 맞춘 기록관 아카 대사
- 주요 판단 전후 선택 문구와 결과
- 지원 요원 자동 능력 발동 대사
- 미니게임 진입·성공·실패 피드백
- 안정화 상태·잔향 회수·괴이 매뉴얼 갱신 문구

### 선택형 서사

- AFTER 7편
- DAILY 9편
- FACTION 9편
- 총 25편

이 수량은 승인 패키지의 목표 범위다. 구현 전 현재 `data/daily_episodes.json`, DB 기록, 이벤트 ID와 충돌을 감사하고 작은 묶음부터 적용한다.

## 4. 에피소드 유형

### AFTER — 사건의 의미를 되짚는 후일담

- 피해자·잔향·실패 기록의 이후
- 사건 규칙이 인물에게 남긴 감정
- 대응 절차에 무엇을 기록할지
- 기록국과 다른 조직의 과거 대응 흔적

후일담은 사건의 정답을 다시 설명하는 장면이 아니라, 사건을 어떤 책임으로 기억할지 보여준다.

### DAILY — 기록국에서 살아가는 사람들

- 보고서 작성, 장비 정비, 식사, 야간 당직, 휴식
- 요원들의 업무 방식과 사소한 습관
- 조직 규정과 괴이 노출의 일상적 위험
- 주인공과 요원의 개인적 거리 변화

일상 장면은 캐릭터의 인간적인 매력을 보여주는 중심 콘텐츠다.

### FACTION — 책임 범위가 다른 사람들과의 교류

- 소문시장의 정보와 사생활
- 마도회의 보존 가치와 연구 윤리
- 퇴마사 계보의 제거·봉인 철학
- 프리랜서의 계약·생존·구조 기준

외부 세력은 선악 구도가 아니라 “누가 어디까지 책임지는가”의 차이로 표현한다.

### RELATION — 선택을 기억하는 연속 이벤트

- 한 장면의 태도가 다음 대사와 행동으로 이어진다.
- 관계 변화는 숫자 팝업보다 말투, 시선, 정보 공유, 도움 요청으로 보인다.
- 요원끼리의 갈등과 협력도 주인공 관계만큼 중요하게 다룬다.

## 5. 캐릭터 목소리 기준

| 인물 | 말투와 행동 | 피할 표현 |
|---|---|---|
| 권나래 | 낮고 정돈된 관찰, 기록과 책임, 확인 질문 | 감정을 직접 장황하게 설명, 무조건적인 지휘관 어조 |
| 윤서하 | 직선적, 빠른 감정 반응, 행동이 앞섬 | 생각 없는 난폭함, 피해를 가볍게 여기는 농담 |
| 오현 | 정중하고 정확함, 비교·배제·공식 언어 | 기계적인 무감정, 모든 현상을 즉시 아는 천재 표현 |
| 강이준 | 짧고 실무적, 안전과 차폐를 행동으로 보여줌 | 과도한 군대식 구호, 감정이 전혀 없는 벽 |
| 한유리 | 가벼운 농담 뒤 불편한 진실, 감각적 비유 | 피해자의 기억을 장난처럼 소비, 모호함만 남기는 대사 |
| 박도윤 | 친근한 거래 말투, 핵심을 바로 주지 않음 | 단순 사기꾼, 모든 정보를 아는 만능 브로커 |
| 이세린 | 침착한 연구자, 자부심과 검증 태도 | 오만한 악역 과학자, 연구를 위해 피해를 무시하는 단선적 표현 |
| 레이먼드 | 계약·거리·생존 가능성 중심, 간결함 | 감정 없는 용병 클리셰, 무조건적인 폭력 해결 |
| 카밀라 | 강한 확신, 현장 경험, 잔류 저주와 연결 절단 | 종교적 광신만으로 설명, 기록을 전면 부정하는 인물 |
| 기록관 아카 | 전문성 70·귀여움 30, 기록 위치와 비교 기준 안내 | 정답 누설, 상시 농담, 장면의 감정을 대신 설명 |

## 6. 상황지시문 작성 기준

상황지시문은 플레이어에게 다음 세 가지를 동시에 제공한다.

1. 지금 보이는 감각적 변화
2. 이전 상태와 비교할 차이
3. 행동을 고를 수 있는 여지

권장 구조:

```text
장소·소리·빛·온도·사물의 변화
→ 인물 또는 장비의 구체적 반응
→ 현재 확인되지 않은 불안 요소
```

좋은 상황지시문은 “무섭다”라고 평가하지 않고 무엇이 달라졌는지 보여준다. 선택 결과를 미리 단정하지 않으며, 핵심 정보는 플레이어가 읽고 비교할 수 있는 문장으로 남긴다.

## 7. 기록관 아카 대사 규칙

### 기능

- 확보된 기록을 짧게 요약한다.
- 서로 비교할 항목을 알려준다.
- 현재 질문을 명확하게 만든다.
- 위험 사례가 무엇을 새로 증명했는지 정리한다.
- 다음 기록 위치를 안내한다.

### 정보 제한

아카는 다음을 말할 수 없다.

- 아직 조사하지 않은 장소의 결과
- 숨은 판정값과 성공 확률
- 최종 정답의 직접 지시
- 선택하지 않은 요원의 관찰
- 데이터에는 없고 작가만 아는 설정

### 길이

- 현장 기본: 1문장, 최대 2줄
- 위험 경고: 1문장 + 짧은 행동 요청
- 결과 정리: 선택·관측·새 기록을 2문장 이내
- 일상: 장면의 주인공이 아닐 때 1~2회만 개입

## 8. 주인공 선택지 기준

선택지는 권나래의 태도를 구분한다.

- 기록 우선
- 사람 우선
- 절차 우선
- 상대의 방식 인정
- 의심 유지
- 책임 분담

선택 문구는 결과를 숨기는 모호한 표현보다 행동과 태도를 예측할 수 있게 쓴다.

```text
“위험 사례로 그대로 남길게요.”
“먼저 다친 곳부터 확인해요.”
“다음에는 같은 선택을 하지 않게 만들면 돼요.”
```

큰 분기보다 상대 반응, 기억 태그, 후속 대사, 작은 정보 공유 차이를 우선한다.

## 9. MVP-045 관계 설계

### 관계 태그

- 신뢰
- 경계
- 부채
- 경쟁
- 공감
- 보호
- 균열

태그는 화면에 점수로 상시 표시하지 않는다. 내부 기존 신뢰 수치가 있다면 저장 호환을 위해 즉시 삭제하지 않고, 신규 선택 기억과 태그를 기존 상태에서 파생하거나 병렬 저장 없이 계산할 수 있는지 먼저 감사한다.

### 관계 변화 출력

- 같은 질문에 다른 말투로 답함
- 먼저 정보를 공유하거나 숨김
- 도움을 요청하는 대상이 달라짐
- 장면에서 서 있는 거리·표정·시선이 달라짐
- 후속 일상·후일담이 열림
- 보고서의 짧은 기여 문장이 달라짐

전투 보너스, 정답 후보 제거, 핵심 단서 자동 공개로 관계를 보상하지 않는다.

### 승인된 초기 범위

- 연속 이벤트 체인 12개
- 총 장면 약 30개
- 주인공–요원, 요원–요원, 외부 접점 관계를 포함

구현은 체인 하나를 처음부터 끝까지 연결하는 수직 단위로 진행한다.

## 10. 우선 관계 축

| 관계 | 시작 갈등 | 변화 방향 |
|---|---|---|
| 권나래 × 윤서하 | 기록 보존과 즉시 제거 | 충돌 → 인정 → 역할 분담 |
| 권나래 × 오현 | 불완전한 의심과 확정 공식 | 논쟁 → 공동 기록 기준 |
| 권나래 × 강이준 | 구조 실패를 개인 책임으로 짊어짐 | 보호 → 책임 분담 |
| 권나래 × 한유리 | 기억이 관계를 증명하는가 | 불안 → 현재 선택의 신뢰 |
| 윤서하 × 오현 | 즉시 행동과 계산 완료 | 경쟁 → 공동 전술 |
| 강이준 × 한유리 | 수리할 수 없는 감각 손상 | 서툰 거리 → 무언의 기준점 |
| 권나래 × 박도윤 | 정보 가치와 피해자 사생활 | 거래 → 조건부 신뢰 또는 경계 |
| 기록국 × 이세린 | 위험 자료의 보존 가치 | 소유권 충돌 → 공동 책임 기준 |
| 기록국 × 레이먼드 | 계약 밖 구조 | 실리 → 행동으로 드러나는 원칙 |
| 기록국 × 카밀라 | 제거와 기록 | 충돌 → 제거 후 남는 기록 인정 |

## 11. 데이터와 상태 소유

- 사건 규칙·단서·선택 결과: 기존 `data/episodes/*`
- 일상 카드와 완료 기록: 기존 일상 데이터·상태 구조
- 관계 선택 기억: 기존 저장과 이벤트 완료 상태를 감사한 뒤 최소 구조로 추가
- 표정·컷인·배치: 프레젠테이션 데이터가 담당하며 서사 상태를 소유하지 않음
- DB·보고서: 이미 발생한 결과를 요약하며 새로운 필수 사실을 만들지 않음

같은 선택 결과를 대사 JSON, UI 스크립트, 저장 코드에서 세 번 정의하지 않는다.

## 12. 구현 순서

1. 세 사건의 현재 장면·선택·결과 ID를 목록화한다.
2. 아카 표시명과 실제 내부 ID를 구분한다.
3. 사건 하나의 상황지시문·아카 대사·결과 문구를 수직 구현한다.
4. AFTER 1편을 사건 결과와 연결한다.
5. DAILY 1편과 FACTION 1편을 기존 일상 구조에 연결한다.
6. 저장·DB·재진입·중복 보상을 검증한다.
7. MVP-044 범위를 확장한다.
8. MVP-045에서 관계 체인 하나를 처음부터 끝까지 구현한다.
9. 데이터가 안정된 뒤 MVP-046 표정·컷인·연출을 연결한다.

## 13. 품질 수용 기준

- 한 장면의 중심 질문을 한 문장으로 말할 수 있다.
- 상황지시문이 선택 전에 필요한 관측 정보를 제공한다.
- 대사를 가려도 상황과 선택 흐름을 이해할 수 있다.
- 말투만 보고 주요 인물을 구분할 수 있다.
- 일상 장면이 설정 설명이 아니라 행동과 충돌로 세계관을 보여준다.
- 선택이 정답 점수보다 이후 반응으로 기억된다.
- 관계 장면이 필수 추리와 엔딩을 잠그지 않는다.
- 재진입·불러오기로 대사 보상·관계 태그가 중복 적용되지 않는다.
- 피해·사망·후유증을 가벼운 개그로 소비하지 않는다.
