# 괴이 기록국 적용 사례 라이브러리

> 문서 위치: `docs/planning/REFERENCE_CASES.md` | 방법: `docs/BENCHMARKING_REFERENCE_GUIDE.md` | 상세 판정: `PROJECT_CORE_STRESS_TEST_AND_BENCHMARK.md`

## 사용 원칙

외부 사례는 현재 결정을 바꿀 비교 차원으로만 사용한다. 제품 사실, 플레이어 반응, 행동 근거, 해석을 구분하고 표면 연출·문구·고유 규칙은 복제하지 않는다.

판정은 `ADOPT / ADAPT / AVOID / TEST / IGNORE`를 사용한다.

## 시장 포지션

기관형 도시괴담 조사만으로는 차별화가 부족하다.

> 괴담의 진실을 밝히는 게임이 아니라, 다시 나타날 괴이에서 살아남을 규칙을 직접 기록하고 전투에서 증명하는 팀 수사 게임.

## 벤치마크 카드

### CASE-EXT-01 - Urban Myth Dissolution Center

- **질문:** 기관형 도시괴담 조사와 어떻게 직접 차별화하는가?
- **관찰:** 기관·에피소드형 사건·정황 증거·온라인 기록을 활용한다.
- **적용:** 팀 운영, 실패 학습, 플레이어 작성형 장기 매뉴얼, 전조 기반 포획 전투의 인과를 전면화한다.
- **제외:** 기관 외형, 픽셀 연출, 해체 구조의 표면 복제.
- **판정:** `ADAPT`.
- **검증:** 첫 사건 뒤 플레이어가 ‘기관형 도시괴담’이 아니라 ‘규칙을 기록해 전투에서 증명’한다고 설명하는가?

### CASE-EXT-02 - PARANORMASIGHT

- **질문:** 텍스트 중심 사건에서 장소성이 규칙 추리에 어떻게 기여하는가?
- **관찰:** 실제 지역을 반영한 배경, 인물 대화, 초자연 규칙을 결합한다.
- **적용:** 한국 도시의 장소·공공 기록·소문을 전조와 규칙 근거로 사용한다.
- **제외:** 복잡한 복수 시점 플로우, 저주 대결 구조, 화면 외형 복제.
- **판정:** `ADAPT`.
- **검증:** 플레이어가 장소 정보를 분위기가 아니라 가설 근거로 사용했는가?

### CASE-EXT-03 - Golden Idol 시리즈

- **질문:** 플레이어가 규칙을 직접 만들었다는 감각을 어떻게 확보하는가?
- **관찰:** 관찰 정보를 수집하고 플레이어가 이론을 조립한다.
- **적용:** 규칙 가설 카드에 지지·반박·미해결 근거를 연결한다.
- **제외:** 단어 끼워넣기 외형, 브루트포스 가능한 후보 풀.
- **판정:** 원리 `ADOPT`, UI `ADAPT`.
- **검증:** 플레이어가 선택지 문구가 아니라 근거 관계로 규칙을 설명하는가?

### CASE-EXT-04 - Return of the Obra Dinn

- **질문:** 기록장이 결과 메뉴가 아니라 핵심 플레이 도구가 되려면?
- **관찰:** 기록 재열람과 논리적 추론이 진행 자체를 구성한다.
- **적용:** 매뉴얼 초안에 불확실성, 반증 가능성, 부분 확정을 보존하고 현장 선택지에 직접 연결한다.
- **제외:** 1비트 시각 스타일, 장면 재생 장치 복제.
- **판정:** `ADAPT`.
- **검증:** 플레이어가 현장 판단 전 매뉴얼을 자발적으로 열고 근거를 연결하는가?

### CASE-EXT-05 - No Case Should Remain Unsolved

- **질문:** 첫 공개 빌드의 범위와 감정적 종결을 어떻게 압축하는가?
- **관찰:** 제한된 대화·기억 조각을 연결하고 짧은 플레이 안에 정서적 종결을 제공한다.
- **적용:** 첫 PoC는 사건 1개로 제한하고 일반 포획·비용 포획·긴급 포획의 후일담 의미를 분명히 한다.
- **제외:** 모든 사건을 같은 기억 퍼즐로 제작.
- **판정:** `ADOPT`.
- **검증:** 18~25분 안에 규칙 이해와 사건 감정이 모두 남는가?

### CASE-EXT-06 - Lobotomy Corporation

- **질문:** 개체별 금기를 배우는 지식 축적을 낮은 손실 피로로 구현하려면?
- **관찰:** 처음에는 정보가 없고 관찰·시행착오로 개체 지식을 축적한다.
- **적용:** 괴이별 영구 매뉴얼과 전용 대응을 재출현의 실제 판단 자산으로 사용한다.
- **제외:** 영구 사망, 반복 노동, 시설 경영, 제압 중심 전투.
- **판정:** `ADAPT`.
- **검증:** 실패 뒤 포기보다 새 기록을 이용한 재도전을 선택하는가?

### CASE-EXT-07 - Disco Elysium

- **질문:** 실패가 진행 차단이 아니라 다른 정보와 반응이 되게 하려면?
- **관찰:** 실패와 선택이 다른 대화·경로·인물 반응을 남긴다.
- **적용:** 위험 사례가 보고서, 요원 반응, 다음 질문을 함께 변경한다.
- **제외:** 장문 내면 독백, 상시 스킬 목소리, 핵심 추리 정답을 좌우하는 난수.
- **판정:** `ADAPT`.
- **검증:** 실패 뒤 같은 행동 반복보다 새 근거 탐색이 증가하는가?

### CASE-EXT-08 - WORLD OF HORROR

- **질문:** 모듈형 괴담을 공통 문법으로 확장하면서 정체성을 지키려면?
- **관찰:** 강한 시각 언어와 모듈형 사건, 턴제 공포를 결합한다.
- **적용:** 전조·가설·지지·반박·위험 사례·포획의 공통 사건 문법을 사용한다.
- **제외:** 로그라이트 랜덤, 전투 중심, 과도한 불친절.
- **판정:** `ADAPT / AVOID`.
- **검증:** 신규 사건이 별도 조작 학습 없이 같은 인지 루프를 사용하는가?

## 내부 사례 재사용

### CASE-INT-01 - 저승역 가설·근거·대응

- 기존 단계형 판단과 매뉴얼 후보·공식·위험 사례를 CORE-MVP-001의 기반으로 재사용한다.
- 새 가설 카드는 자유 문장 입력을 강제하지 않고 기존 선택·근거 연결 구조를 확장한다.

### CASE-INT-02 - route restore / rain dodge

- 조작을 그대로 의뢰에 복제하지 않는다.
- `전조 관측 → 규칙 적용 → 즉시 결과` 공통 인지 문법이 유지되는 경우에만 재사용한다.
- CORE-MVP-003 전에는 활성 확장하지 않는다.

### CASE-INT-03 - 준비 화면 점진 공개

- 첫 사건에는 조사·가설·전조·포획에 필요한 정보만 전면 노출한다.
- 시장·세력·장비·연구·관계 수치는 기능 삭제가 아니라 단계적 공개 대상으로 둔다.

## 외부 확인 기준

2026-07-23 기준 공식 상점 설명을 제품 사실의 기준으로 사용했다. 판매량·평점·리뷰 비율만으로 코어를 변경하지 않으며, 최신 제품 상태가 결정에 영향을 줄 때 원문을 다시 확인한다.
