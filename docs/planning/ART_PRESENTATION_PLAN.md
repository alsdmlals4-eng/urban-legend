# 괴이 기록국 아트·표정·컷인·이벤트 연출 기획서

> 문서 위치: `docs/planning/ART_PRESENTATION_PLAN.md` | 프로젝트 방향: `docs/planning/PROJECT_DIRECTION.md` | 실제 에셋: `assets/characters/`

## 1. 목적

이 문서는 캐릭터 아트, 배경, 대화 UI, 표정, 컷인, 화면 전환, 이벤트 연출을 같은 시각 언어로 묶는다.

연출의 목적은 장면을 화려하게 만드는 것이 아니라 다음을 더 빠르고 정확하게 읽게 하는 것이다.

- 지금 누가 말하고 있는가
- 어떤 감정과 태도가 변했는가
- 어떤 정보가 중요한가
- 선택의 결과가 무엇이었는가
- 관계와 세계관이 어떻게 달라졌는가

아트와 프레젠테이션 계층은 게임 진행·저장·관계 상태의 소유자가 아니다. 기존 데이터를 읽고 표현하며, 자산이 없거나 연출이 꺼져도 텍스트와 기본 UI로 모든 핵심 정보를 전달해야 한다.

## 2. 시각 정체성

### 기본 구성 비율

- 비주얼 노벨 기반: 약 65%
- 절제된 그래픽 노블 질감: 약 25%
- 클래식 셀 애니메이션 포인트: 약 10%
- 수채·먹 번짐: 특수 기록, 회상, 오염, 잔향 표현에만 제한

### 핵심 키워드

- 현대 오컬트
- 공공기관 기록물
- 야간 도시와 실내 형광등
- 절제된 청록 신호
- 위험 적색
- 오래된 종이와 잉크
- 인물의 표정과 시선이 먼저 읽히는 화면
- 장식보다 기능이 앞서는 기록 UI

### 금지 방향

- 유리처럼 번쩍이는 과도한 AI 렌더링
- 피부·의상·금속의 과잉 반사
- 인물보다 프레임 장식이 먼저 보이는 화면
- 읽을 수 없는 장식 한글과 이미지 안에 구운 UI 텍스트
- 모든 장면에 입자·흔들림·번쩍임을 반복
- 캐릭터별로 규칙이 다른 별도 UI
- 표정 변화 없이 대사만 길게 이어지는 정적 장면

## 3. 캐릭터 아트 기준

### 공통 규격

- 성인 자연 비율
- 2단계 셀 셰이딩
- 짙은 갈색 또는 남색 계열 선화
- 절제된 캐릭터 고유 강조색
- 약한 종이·잉크 질감
- 얼굴과 손의 가독성을 배경 디테일보다 우선
- 게임용 PNG에는 이름·소속·능력·대사를 굽지 않음

### 캐릭터별 시각 신호

| 인물 | 강조색·형태 | 연출에서 강조할 태도 |
|---|---|---|
| 권나래 | 청록, 기록 장비, 정돈된 실루엣 | 관찰, 판단, 책임, 잠깐의 망설임 |
| 윤서하 | 적색, 부적·격투 동작, 전진 실루엣 | 즉시 반응, 분노, 당황, 솔직한 인정 |
| 오현 | 보라, 공식·도형·정밀 장비 | 계산, 의심, 확정, 드문 감정 균열 |
| 강이준 | 황토·회갈색, 방패·차폐 장비 | 안정, 경계, 보호, 무언의 피로 |
| 한유리 | 청색·은색, 흐르는 감각선 | 장난, 감응, 불안, 기억의 단절 |
| 박도윤 | 녹색, 가벼운 정장·정보 단말 | 친근함, 계산, 회피, 진심이 드러나는 순간 |
| 이세린 | 보라·백색, 연구 장비·정제된 선 | 자부심, 호기심, 반론, 책임 인정 |
| 레이먼드 | 청회색, 실용 장비·거리 확보 | 상황 평가, 경계, 결심, 계약 밖 행동 |
| 카밀라 | 금색, 성물·절단 도구·강한 축 | 확신, 혐오, 경계, 기록의 필요 인정 |

## 4. 기본 자산 계층

캐릭터당 필요한 자산을 역할별로 분리한다.

1. **전신:** 편성·소개·기록 화면
2. **반신 초상:** 대화·일상·후일담
3. **표정 파생:** 대화 장면의 감정 변화
4. **조사 지원 컷:** 단서 비교·관측·차폐
5. **안정화 지원 컷:** 잔향 안정화·보호·구속
6. **특수 컷인:** 능력 발동·결정적 관계 장면

모든 자산을 처음부터 만들지 않는다. 장면 빈도와 감정 중요도를 기준으로 우선순위를 둔다.

## 5. 표정 시스템

### 목적

표정은 대사를 반복 설명하지 않고 다음을 보완한다.

- 말과 속마음의 차이
- 상대에 대한 거리
- 위험 인식 변화
- 선택을 받아들이는 태도
- 관계 태그가 대사에 미치는 미세한 변화

### 의미 키

표정 자산은 파일 번호보다 의미 키를 사용한다.

| 키 | 기본 의미 | 사용 예 |
|---|---|---|
| `neutral` | 평상·업무 | 기본 대화, 정보 전달 |
| `focus` | 집중·분석 | 기록 비교, 공식 계산 |
| `alert` | 경계·위험 인식 | 이상 징후, 경고 |
| `anger` | 분노·강한 반대 | 가치관 충돌 |
| `concern` | 걱정·보호 | 부상, 기억 손상 |
| `soft` | 안도·신뢰 | 관계 완화, 짧은 인정 |
| `awkward` | 당황·거리 조절 | 사적인 질문, 실수 |
| `hurt` | 상처·피로 | 실패 후, 과거 기억 |
| `resolve` | 결심 | 행동 전환, 책임 수용 |
| `uncanny` | 감응·괴이 영향 | 한유리, 아카 오류, 잔향 노출 |

모든 캐릭터가 모든 키를 가져야 하는 것은 아니다. 공통 의미 키에서 캐릭터별 표현 강도를 다르게 한다.

### 우선 제작 순서

1. `neutral`
2. `focus`
3. `alert`
4. 캐릭터 핵심 감정 2종
5. 관계 이벤트용 `soft` 또는 `hurt`
6. 특수 괴이 영향 표정

### 폴백

- 요청한 키가 없으면 동일 감정군의 가까운 키를 사용한다.
- 그것도 없으면 `neutral`로 폴백한다.
- 표정 자산 부재는 대사 진행과 선택을 막지 않는다.
- 폴백은 오류 팝업 대신 개발 로그와 QA 목록에 기록한다.

## 6. 대화 화면 구조

### 기본 시선 흐름

```text
배경과 공간
→ 현재 화자 얼굴·자세
→ 이름표와 대사
→ 상대 인물 반응
→ 선택지
→ 보조 기록·메뉴
```

### 기본 레이아웃

- 배경은 장면의 장소와 시간대를 먼저 전달한다.
- 주 화자는 화면 중심 또는 좌우 주요 슬롯에 배치한다.
- 대사창은 하단에 두되 인물 손과 핵심 소품을 과도하게 가리지 않는다.
- 이름표는 화자 구분용이며 장식 프레임을 최소화한다.
- 선택지는 대사창 바로 위 또는 장면 중심 하단에 일관되게 배치한다.
- 아카는 캐릭터 슬롯을 차지하기보다 단말·오버레이·작은 관제 패널로 구분한다.

### 인물 수

- 1인: 독백보다 행동·아카 응답·환경 변화 활용
- 2인: 기본 대화 구조
- 3인: 주 화자 1명, 반응자 2명; 동시 강조 금지
- 4인 이상: 군상 화면보다 컷 전환·반응 컷으로 분리

한 화면에 모든 인물을 작게 세우지 않는다.

## 7. 컷인 체계

컷인은 중요한 정보를 강조하는 짧은 프레젠테이션이다. 전신 일러스트를 단순 확대하는 것과 구분한다.

### 권장 6종

| 종류 | 목적 | 사용 시점 | 길이 원칙 |
|---|---|---|---|
| 능력 발동 | 누가 왜 기여했는지 표시 | 조사·안정화 자동 지원 | 0.8~1.5초 |
| 위험 경고 | 즉시 멈춰야 할 변화 | 잘못된 규칙 적용 직전·직후 | 0.5~1.0초 |
| 규칙 확인 | 가설이 현장에서 입증됨 | 핵심 단서 연결·검증 성공 | 1.0~2.0초 |
| 관계 반응 | 선택 기억이 드러남 | 신뢰·균열·보호의 결정적 순간 | 1.0~2.0초 |
| 잔향 오염 | 괴이 영향이 인물·기록에 번짐 | 특수 사건·감응 | 0.8~1.5초 |
| 매뉴얼 갱신 | 플레이어 기록의 완성 | 사건 후 핵심 항목 등록 | 1.0~2.0초 |

### 제한

- 같은 종류를 짧은 시간 안에 반복하지 않는다.
- 일반 대사마다 컷인을 사용하지 않는다.
- 컷인 중 선택지 입력을 받지 않는다.
- 컷인이 없어도 텍스트·상태 변화로 의미를 이해할 수 있어야 한다.
- 성공 연출을 위험 장르와 맞지 않게 축제처럼 만들지 않는다.

## 8. 이벤트 연출 비트

각 장면은 다음 비트 중 필요한 것만 사용한다.

```text
공간 제시
→ 인물 진입 또는 시선 이동
→ 핵심 대화
→ 작은 행동·소품 변화
→ 플레이어 선택
→ 표정·거리·음향 반응
→ 기록 또는 후속 태그
→ 잔향이 남는 마무리 이미지
```

### AFTER

- 사건 배경이나 회수 물품을 먼저 보여준다.
- 인물이 사건을 어떻게 기억하는지 행동으로 드러낸다.
- 마지막은 보고서 문장, 남은 소리, 장비의 이상처럼 사건의 잔향으로 닫는다.

### DAILY

- 생활 소품과 업무 습관으로 캐릭터를 소개한다.
- 큰 사건보다 사소한 어긋남과 관계 반응을 중심에 둔다.
- 공포 연출이 발생해도 일상 장면의 정서적 목적을 빼앗지 않는다.

### FACTION

- 상대 세력의 공간·도구·규칙을 먼저 보여준다.
- 대화는 가치관 충돌과 거래 조건을 중심으로 구성한다.
- 악역 입장 연출보다 서로 다른 책임 범위를 시각적으로 대비한다.

### RELATION

- 이전 선택을 회상 문구로 직접 설명하기보다 말투·거리·먼저 건네는 물건으로 보여준다.
- 관계 태그 변화는 팝업보다 장면 결과와 후속 대사로 확인한다.
- 중요한 관계 전환에서만 전용 표정·컷인을 사용한다.

## 9. 기록관 아카 연출

- 디지털 유령 마스코트지만 과도한 귀여움으로 공포를 깨지 않는다.
- 기본은 단말 화면, 작은 관제 패널, 신호선, 3음 접속 모티프다.
- `neutral`, `focus`, `warning`, `error`, `soft` 정도의 상태로 제한한다.
- 글리치 효과는 정보 손상·괴이 간섭처럼 의미가 있을 때만 사용한다.
- 오류 연출 중에도 실제 텍스트는 읽을 수 있어야 한다.
- 아카의 애니메이션은 안내 완료를 기다리게 만들지 않는다.

## 10. 배경과 공간 연출

### 기록국 HQ

- 기능적 공공기관과 비공식 오컬트 장비가 함께 존재한다.
- 정돈된 기록 구역과 임시 격리 구역의 대비를 사용한다.
- 일상 장면은 같은 장소라도 조명·시간·소품 배치로 반복 피로를 줄인다.

### 괴이 현장

- 익숙한 도시 공간에 한 가지 비정상 규칙을 명확하게 넣는다.
- 무서운 장식을 늘리기보다 이전 상태와 달라진 부분을 강조한다.
- 선택 결과는 조명, 소리, 사물 위치, 반복, 인물 반응으로 즉시 보여준다.

### 외부 세력 공간

- 소문시장: 정보가 상품인 임시성과 밀집감
- 마도회: 규격화된 연구와 보존 장치
- 퇴마사 계보: 사용 흔적이 남은 실전 도구와 봉인선
- 프리랜서 접점: 이동성, 계약 장비, 개인 규칙

## 11. 오디오와 텍스트 효과

- 중요한 신호는 음향과 시각·문구를 함께 제공한다.
- 캐릭터별 전체 음성보다 접속음, 호흡, 장비음, 짧은 반응음을 우선한다.
- 공포 효과음은 반복 횟수와 음량을 제한한다.
- 텍스트 흔들림·왜곡은 핵심 단어 일부에만 사용한다.
- 자동 진행·스킵이 도입될 경우 선택지·경고·기록 갱신에서 반드시 멈춘다.

## 12. 접근성

- 연출 감소 옵션: 줌, 흔들림, 슬라이드 속도 축소
- 점멸 제한: 번쩍임 대신 색 변화·테두리·문구 사용
- 텍스트 우선: 컷인·표정 없이도 핵심 정보 제공
- 색상 독립: 위험·성공·선택 가능 여부를 아이콘과 문구로 병행
- 키보드 포커스: 대사 진행, 선택, 기록 열기, 닫기 순서 일관
- 읽기 시간: 자동으로 사라지는 핵심 문구 금지
- 한국어 줄바꿈: 이름·조사 용어·수치 단위가 부자연스럽게 분리되지 않게 확인

## 13. 데이터와 구현 경계

- 대사 내용과 선택 결과: 서사 데이터
- 현재 감정 의미 키: 장면 프레젠테이션 데이터
- 실제 표정 파일 경로: 캐릭터 에셋 레지스트리
- 컷인 종류와 타이밍: 공용 프레젠테이션 레지스트리
- 관계 태그·사건 상태: 기존 게임 상태 또는 최소 관계 상태
- UI 스크립트: 데이터를 읽고 재생하며 결과를 새로 결정하지 않음

표정 이름, 컷인 파일, 장면 노드 경로를 대사 본문에 직접 하드코딩하지 않는다.

## 14. MVP-046 구현 순서

1. 현재 대화·일상·조사 화면의 공통 요소와 상태 소유자를 감사한다.
2. 기존 기능을 감싸는 공용 대화 스테이지를 최소 범위로 만든다.
3. 권나래·윤서하 2인 장면 하나에 표정 의미 키와 폴백을 연결한다.
4. 능력 발동 컷인 1종을 조사 또는 안정화 장면에 연결한다.
5. 관계 반응 컷인 1종을 일상 체인에 연결한다.
6. 모션 감소·점멸 제한·텍스트 폴백을 구현한다.
7. 1280×720, 1920×1080, 키보드·마우스·Esc를 검증한다.
8. 안정된 공용 체계를 나머지 캐릭터와 장면으로 확장한다.

## 15. 수용 기준

- 대사창보다 인물과 배경이 먼저 읽힌다.
- 현재 화자와 반응자가 혼동되지 않는다.
- 표정 변화가 대사의 감정과 항상 같은 방향만 반복하지 않는다.
- 컷인의 사용 이유를 장면별로 한 문장으로 설명할 수 있다.
- 연출을 꺼도 선택·결과·기록을 이해할 수 있다.
- 컷인과 오버레이가 입력 포커스를 빼앗거나 선택을 중복 실행하지 않는다.
- 자산이 없는 캐릭터도 기본 표정과 공용 UI로 정상 진행한다.
- 캐릭터별 색과 형태는 구분되지만 전체 작품의 선화·명암·질감은 통일된다.
- 이미지 안에 생성형 한글이나 게임 UI 문구가 구워지지 않는다.
