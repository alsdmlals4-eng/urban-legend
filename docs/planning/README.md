# 기획 상세 부록 인덱스

> 문서 위치: `docs/planning/README.md` | 활성 기획서 진입점: [`docs/DOCUMENTATION_MAP.md`](../DOCUMENTATION_MAP.md) | 상태 원본: [`docs/CURRENT_STATUS.md`](../CURRENT_STATUS.md)

이 파일은 더 이상 처음 읽는 활성 기획서가 아니다. 사용자는 Documentation Map 첫 화면의 다섯 활성 기획서만 먼저 읽는다. 이 폴더의 문서는 그 다섯 문서가 특정 수치, 대본, 사례, 인수 절차를 증명하기 위해 연결하는 **상세 부록**이다.

| 부록 | 필요할 때 여는 조건 | 활성 원본 |
|---|---|---|
| `PROJECT_DIRECTION.md` | 프로젝트 불변 조건·세계관 표현의 세부 근거 | 게임 기획서 |
| `NARRATIVE_CONTENT_PLAN.md` | 사건 대사·일상·관계 데이터 작업 | 게임 기획서 |
| `ART_PRESENTATION_PLAN.md` | 표정·컷인·대화 연출의 세부 기준 | 아트 기획서 |
| `ROADMAP_AND_HANDOFF.md` | 단계 의존성·인수 절차 | 프로그래밍 기획서 |
| `MVP044_046_EXECUTION_PLAN.md` | 과거 승인 MVP의 정확한 변경 경계 | 프로그래밍 기획서 |
| `REFERENCE_CASES.md` | 기존 벤치마킹 근거를 재사용할 때 | 게임/아트 기획서 |

고유 계약이 사라진 부록은 참조 검사를 한 뒤 [`docs/archive/backup/YYYY-MM-DD/`](../archive/backup/)에 원래 경로·사유·대체 문서를 남겨 보관한다. 현재는 링크 호환과 세부 계약을 보존하기 위해 이동하지 않는다.
