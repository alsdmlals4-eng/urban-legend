# UX-PD-001 점진적 공개·시스템 노출 축소

> 문서 위치: `docs/planning/PROGRESSIVE_DISCLOSURE_PLAN.md` | 현행 상태: `docs/CURRENT_STATUS.md` | 검증 기록: `docs/qa/PROGRESSIVE_DISCLOSURE_SLICE_001.md`

## 목적

첫 플레이부터 모든 시스템을 같은 중요도로 노출하지 않고, 현재 판단에 필요한 정보부터 보여준다. 기능을 삭제하거나 잠그는 것이 아니라 정보 위계와 진입 순서를 조정한다.

## 구현 단계

### 2A 사건 준비 화면 — 구현 완료

항상 표시:

- 사건·일상
- 편성

처음에는 접고 한 번의 명시적 선택으로 즉시 복원:

- 장비
- 외부 접점
- 기록
- 외부 의뢰 바로가기

공개 상태는 새 저장 필드 없이 기존 `seen_hint_ids`의 `preparation_secondary_tools_opened`를 사용한다. 완료 사건, 해금 장비·기록, 장착 장비, 소모품, 수락 의뢰가 있는 진행 저장은 자동으로 전체 도구를 표시한다.

### 2B 조사 정보 위계 — 다음 단계

- 현재 장면의 관측 정보와 선택을 1차 정보로 유지한다.
- 과거 로그·위험 사례·요원 상세는 보조 패널로 분리한다.
- 미확보 정보와 정답 추론은 노출하지 않는다.
- 기존 조사 상태·단서 ID·선택 ID·저장 계약은 바꾸지 않는다.

### 2C 결과 정보 위계 — 후속 단계

- 현장 결과와 다음 행동을 우선 표시한다.
- 상세 통계·장기 기록·매뉴얼 근거는 펼침 영역으로 분리한다.

## 보호 계약

- 저장 Schema `mvp-039` 유지
- 시장·장비·세력 의뢰·소모품·기록 데이터와 효과 유지
- 기존 사건·선택·보고서·DB ID 유지
- UI가 정답이나 최적 요원을 대신 선택하지 않음
- 1280×720과 1920×1080, 마우스·키보드 입력 지원

## 책임 경계

- 게임 상태와 경제 규칙: 기존 `preparation_scene.gd`와 `GameState`
- 노출 순서: `preparation_progressive_disclosure.gd`
- 검증: `progressive_disclosure_preparation_test.gd`

2A 병합 후 `investigation_scene.gd`의 화면 구성과 현재 책임 문서를 감사하고, 2B를 한 사건의 첫 장면부터 선택까지 작은 수직 단위로 구현한다.
