# 점진적 공개·시스템 노출 축소 기획서

> 문서 위치: `docs/planning/PROGRESSIVE_DISCLOSURE_PLAN.md` | 현재 상태: `docs/CURRENT_STATUS.md` | UI 구조: `docs/GODOT_NATIVE_UI_ARCHITECTURE.md`
>
> 기준선: `MVP-043 + CORE-VALIDATION-001 + UX-PD-001` / `Ver 4.2` / 저장 Schema `mvp-039`

## 1. 목적

괴이 기록국의 첫 준비 화면에서 일정·사건·편성·일상·장비·시장·세력 의뢰·기록물이 동시에 경쟁하지 않게 한다. 플레이어가 지금 필요한 결정부터 이해하고, 이미 존재하는 보조 시스템은 필요할 때 직접 펼치게 한다.

```text
현재 반일 이해
→ 사건과 주인공 일정 확인
→ 서포트 편성
→ 필요할 때 보조 준비 도구 펼치기
→ 장비·외부 접점·기록 활용
→ 현장 조사
```

점진적 공개는 기능 삭제나 콘텐츠 잠금이 아니다. 같은 화면 안에서 정보의 우선순위를 정하고, 사용자가 명시적으로 펼친 기능은 이후 방문에서도 유지한다.

## 2. 핵심 원칙

1. **현재 결정 우선:** 첫 준비에서 반드시 결정해야 하는 사건·일정·편성을 먼저 보여 준다.
2. **기능 보존:** 장비·시장·세력 의뢰·소모품·기록물의 규칙과 수치를 바꾸지 않는다.
3. **사용자 주도 공개:** 보조 도구는 한 버튼으로 즉시 펼칠 수 있으며 별도 비용·조건·튜토리얼 완료를 요구하지 않는다.
4. **내용이 있으면 자동 공개:** 이미 장비·기록·소모품·수락 의뢰·완료 보고서를 가진 저장은 보조 도구를 숨기지 않는다.
5. **기존 저장 재사용:** 새 저장 필드를 만들지 않고 기존 `seen_hint_ids`에 공개 경험만 기록한다.
6. **정보 감춤과 정답 감춤을 구분:** UI 복잡도는 낮추되 사건 규칙·확보 근거·현재 위험처럼 판단에 필요한 정보는 숨기지 않는다.

## 3. 준비 화면 정보 계층

### 항상 노출하는 핵심 계층

- 현재 날짜·반일·일정 단계
- 현재 사건과 사건 선택
- 권나래 일정
- 서포트 편성
- 조사 실행 상태와 저장
- 사건·일상 탭
- 편성 탭

### 최초에는 접고 필요할 때 공개하는 보조 계층

- 장비 탭
- 외부 접점 탭
- 기록 탭
- 외부 의뢰 대시보드 바로가기

보조 계층의 내부 기능은 기존과 동일하다. 탭을 숨긴 동안에도 데이터·가격·의뢰 상태·장착·기록 보상은 변경되지 않는다.

## 4. 공개 규칙

보조 계층은 다음 조건 중 하나라도 만족하면 표시한다.

- 사용자가 `보조 준비 도구` 버튼을 눌렀다.
- `preparation_secondary_tools_opened` 힌트 ID가 저장돼 있다.
- 완료 사건 보고서가 하나 이상 있다.
- 해금 장비 또는 기록물이 있다.
- 장착 장비가 있다.
- 소모품을 보유하고 있다.
- 수락 상태의 세력 의뢰가 있다.

첫 신규 캠페인처럼 위 조건이 하나도 없을 때만 장비·외부 접점·기록 탭을 접는다.

## 5. 상태 소유와 저장

- 화면 표시 상태 소유자: `scripts/scenes/preparation_scene.gd`
- 영속 공개 기억: 기존 `GameState.seen_hint_ids`
- 공개 ID: `preparation_secondary_tools_opened`
- 저장 버전: `mvp-039` 유지
- 새 경제·진행·사건 상태: 없음

사용자가 공개 버튼을 누르면 기존 `mark_hint_seen()`과 `save_game()`을 사용한다. 이 기록이 없는 과거 저장은 보유 콘텐츠 조건을 먼저 확인하므로 이미 진행 중인 저장의 기능이 다시 접히지 않는다.

## 6. 입력·접근성

- 공개 버튼은 기존 대시보드 액션 그리드의 네 번째 자리를 사용한다.
- 공개 전에는 `외부 의뢰` 바로가기를 숨기고 같은 자리에 `보조 준비 도구`를 표시한다.
- 공개 후에는 버튼이 사라지고 기존 `외부 의뢰` 바로가기가 복원된다.
- 핵심 탭은 키보드 탭 이동에서 항상 접근 가능해야 한다.
- 숨겨진 보조 탭은 포커스를 받지 않아야 한다.
- 공개 뒤 기존 탭 순서·마우스·키보드 동작을 유지한다.
- 1280×720에서 기존 대시보드 높이를 늘리지 않는다.

## 7. 이번 2A 범위

- 사건 준비 화면의 핵심 탭과 보조 탭 분리
- 명시적 일회 공개와 저장 왕복
- 진행 저장의 자동 공개
- 기존 준비·일상·편성 테스트 회귀
- 자동 회귀 진입점 추가

## 8. 이번 범위에서 하지 않는 것

- 시장·세력 의뢰·장비의 해금 시점 변경
- 경제 가격·초기 잔향 파편 변경
- 사건·일상 콘텐츠 삭제
- 새로운 튜토리얼 팝업 추가
- 회수·조사 화면의 정보 구조 변경
- 저장 Schema 증가
- 사용자가 펼친 기능을 다시 강제로 접기

## 9. 후속 단계

### 2B — 조사 화면의 선택 관련 정보만 노출

현재 선택과 무관한 능력·장비·상태 요약을 접고, 판단에 직접 필요한 근거·위험·현재 효과만 우선 표시한다.

### 2C — 결과 화면의 다음 행동 우선순위

보상 전체 목록보다 `무엇이 기록됐고 다음 판단에 무엇이 바뀌는가`를 먼저 보여 준다.

### 2D — 신규 플레이어 관찰 검증

5~8명이 첫 준비 화면에서 다음 질문에 답할 수 있는지 관찰한다.

- 지금 반드시 정해야 하는 것은 무엇인가?
- 장비·시장·기록은 어디서 펼치는가?
- 펼친 기능이 사라지거나 잠긴 것으로 오해하지 않는가?
- 첫 현장 진입까지 불필요한 왕복이 줄었는가?

## 10. 수용 기준

- 신규 상태에서 사건·편성 탭은 보이고 장비·외부 접점·기록 탭은 접힌다.
- 공개 버튼 한 번으로 기존 보조 탭이 모두 복원된다.
- 공개 선택은 저장 후 준비 화면 재진입에서도 유지된다.
- 보유 콘텐츠 또는 수락 의뢰가 있는 저장은 자동으로 전체 탭을 표시한다.
- 기존 준비 일정·요원 편성·일상 에피소드 테스트가 통과한다.
- 기존 시장·장비·의뢰·기록 데이터와 저장 Schema가 바뀌지 않는다.
- 활성 문서 참조 감사와 전체 Godot 회귀가 통과한다.
