# 점진적 공개·시스템 노출 축소 기획서

> 문서 위치: `docs/planning/PROGRESSIVE_DISCLOSURE_PLAN.md` | 현재 상태: `docs/CURRENT_STATUS.md` | UI 구조: `docs/GODOT_NATIVE_UI_ARCHITECTURE.md`
>
> 기준선: `MVP-043 + CORE-VALIDATION-001 + UX-PD-001 2A·2B` / `Ver 4.2` / 저장 Schema `mvp-039`

## 1. 목적

괴이 기록국의 준비·조사·결과 화면에서 모든 시스템이 같은 중요도로 경쟁하지 않게 한다. 플레이어가 **지금 판단해야 하는 정보와 행동**을 먼저 읽고, 과거 기록·상세 상태·보조 시스템은 필요할 때 직접 펼치게 한다.

점진적 공개는 기능 삭제나 콘텐츠 잠금이 아니다. 기존 데이터·규칙·저장 상태는 유지하고 화면의 정보 우선순위만 조정한다.

## 2. 핵심 원칙

1. **현재 결정 우선:** 현재 장면의 관측·대화·선택을 첫 계층에 둔다.
2. **기능 보존:** 장비·시장·세력 의뢰·기록·괴이 매뉴얼의 규칙과 수치를 바꾸지 않는다.
3. **사용자 주도 공개:** 보조 정보는 한 번의 명시적 버튼으로 즉시 열 수 있다.
4. **내용이 있으면 자동 공개:** 진행 저장의 기존 콘텐츠와 기능을 신규 상태처럼 숨기지 않는다.
5. **기존 저장 재사용:** 새 저장 필드를 만들지 않거나, UI 임시 상태는 저장하지 않는다.
6. **판단 정보 보존:** 근거·위험·현재 효과를 숨기지 않으며 미확보 정보와 정답은 새로 노출하지 않는다.
7. **상태 비소유 표시 계층:** 점진적 공개 helper는 게임 진행·경제·단서·선택 결과를 소유하지 않는다.

## 3. 2A — 사건 준비 화면 정보 계층

### 구현 완료

항상 표시:

- 현재 날짜·반일·일정 단계
- 사건·일상 탭
- 편성 탭
- 권나래 일정과 조사 실행 상태

처음에는 접고 필요할 때 공개:

- 장비 탭
- 외부 접점 탭
- 기록 탭
- 외부 의뢰 바로가기

공개 조건:

- 사용자가 `보조 준비 도구`를 눌렀다.
- `preparation_secondary_tools_opened`가 기존 `seen_hint_ids`에 있다.
- 완료 사건·해금 장비·기록·장착 장비·소모품·수락 의뢰가 있다.

상태 경계:

- 일정·편성·경제·진행: 기존 `preparation_scene.gd`와 `GameState`
- 표시 순서: `preparation_progressive_disclosure.gd`
- 저장 Schema: `mvp-039` 유지

## 4. 2B — 조사 화면 정보 계층

### 구현 완료 범위

저승역 조사 화면의 기본 화면에서는 다음을 우선한다.

- 현재 관측·대화
- 현재 선택
- 현재 조사 지점과 조사 방법
- 즉시 발생한 결과와 다음 행동

다음 정보는 명시적 버튼으로만 연다.

- 확보 규칙을 정리한 괴이 매뉴얼
- 과거 단서·힌트·학습 기록을 모은 현장 기록

### 상호 배타 규칙

- 기본 진입에서는 괴이 매뉴얼과 현장 기록이 모두 닫혀 있다.
- `괴이 매뉴얼 열기`를 누르면 매뉴얼만 연다.
- 매뉴얼을 연 상태에서 현장 기록을 열면 매뉴얼을 자동으로 닫는다.
- 현장 기록을 연 상태에서 매뉴얼을 열면 현장 기록을 자동으로 닫는다.
- 보조 패널을 닫아도 현재 관측·대화·선택은 유지된다.

### 상태 경계

- 조사 진행·단서·선택·위험·결과: 기존 `investigation_scene.gd`와 `GameState`
- 정보 위계: `investigation_progressive_disclosure.gd`
- 설치 진입점: 기존 공용 `ScenePresentation.apply_background()`의 `investigation` 역할
- 저장되는 새 UI 상태: 없음
- 다른 두 사건: 기존 직접 조사 레이아웃 유지

### 금지 방향

- 매뉴얼을 열었다는 이유로 단서나 규칙을 추가하지 않는다.
- 현장 기록이 정답·추천 행동·최적 요원을 자동 표시하지 않는다.
- 현재 위험·확보 근거·선택 결과를 보조 패널 안으로 숨기지 않는다.
- 조사 씬 전체를 새 상태 소유자로 교체하지 않는다.

## 5. 입력·접근성

- 보조 패널은 명시적 텍스트 버튼으로 열고 닫는다.
- 마우스와 키보드가 같은 경로를 사용한다.
- 동시에 두 보조 패널이 열려 현재 선택 영역을 압박하지 않는다.
- 1280×720과 1920×1080에서 현재 관측·선택이 화면 안에 남아야 한다.
- 긴 한국어 문장과 버튼 라벨이 잘리지 않아야 한다.
- 색상·음향만으로 열림 상태를 전달하지 않는다.

## 6. 자동 검증

- 준비 화면: `tests/progressive_disclosure_preparation_test.gd`
- 조사 화면: `tests/progressive_disclosure_investigation_test.gd`
- 전체 회귀: `tests/run_godot_regression.sh`

2B 전용 검증은 다음을 확인한다.

- 현재 관측·대화·선택이 기본 표시된다.
- 매뉴얼·현장 기록은 기본 접힘 상태다.
- 매뉴얼 버튼이 명시적으로 보인다.
- 두 보조 패널이 동시에 열리지 않는다.
- 기존 씬·단서·선택·저장 계약을 바꾸지 않는다.

## 7. 다음 단계

### 2C — 결과 화면의 다음 행동 우선순위

- 현장 결과와 다음 행동을 첫 계층에 둔다.
- 보상 전체 목록·장기 통계·상세 매뉴얼 근거는 펼침 영역으로 분리한다.
- 결과 화면·완료 보고서·DB가 같은 저장 데이터를 읽도록 유지한다.

### 외부 관찰 검증

5~8명의 신규 플레이어가 다음 질문에 답할 수 있는지 관찰한다.

- 지금 반드시 읽고 선택해야 하는 것은 무엇인가?
- 보조 준비 도구·괴이 매뉴얼·현장 기록은 어디서 여는가?
- 접힌 기능을 삭제되거나 잠긴 기능으로 오해하지 않는가?
- 첫 현장 진입과 첫 조사 선택까지 불필요한 왕복이 줄었는가?

## 8. 수용 기준

- 2A 준비 화면 계약과 2B 조사 화면 계약이 함께 유지된다.
- 기존 시장·장비·의뢰·기록·단서·선택 데이터가 바뀌지 않는다.
- 저장 Schema `mvp-039`가 유지된다.
- 활성 문서 참조 감사와 전체 Godot 회귀가 통과한다.
- 신규 플레이어 외부 관찰 테스트는 실행 전까지 `NOT_RUN`으로 보고한다.
