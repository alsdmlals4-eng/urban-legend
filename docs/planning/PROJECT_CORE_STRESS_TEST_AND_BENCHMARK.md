## 1. 결론

괴이 기록국의 코어는 다음 최소 계약으로 확정한다.

> **관측 가능한 단서로 괴이의 규칙 가설을 만들고, 위험을 감수한 검증으로 이해를 갱신한 뒤, 그 이해로 회수 전투의 전조를 읽어 포획 조건을 열고, 성공과 실패를 다음 출현의 괴이 매뉴얼로 기록하는 현대 오컬트 수사 RPG.**

가장 뾰족한 순간은 다음이다.

> **조사에서 읽은 규칙이 회수 전투의 전조로 실제 나타나고, 플레이어가 준비한 대응으로 포획 창을 여는 순간.**

캠페인 일정, 부상, 포획 연구, 세력 의뢰, 히든 분기, 엔딩 축은 승인된 방향이지만 프로젝트 정체성을 정의하는 최소 코어에는 포함하지 않는다. 이들은 코어를 증폭하거나 장기 보상을 제공하는 `CORE_SUPPORT`다.

상태 판정:

- 프로젝트 코어: `CORE_RECORDED`
- 적대적 검토: `STRESS_TESTED`
- 구현 상태: `POC_PENDING`
- Production gate: `HOLD_UNTIL_PLAYER_EVIDENCE`

1. **Attack:** 이 설계가 실제 플레이에서 실패했다고 가정하고 모순·불공정·범위 팽창을 찾음

## 5. 확정된 페어플레이 계약

### 5.1 논리와 확률의 분리

- 관측 가능한 단서, 선택지 모순, 가설의 지지·반박 관계는 난수 없이 판정한다.
- `현장 이해도` 단계는 수집·검증한 근거 계약으로 결정한다.
- 전조 해석 확률은 이미 관측된 패턴의 세부 정보를 이번 턴에 읽어내는가만 결정한다.
- 판정 실패는 `놈은 무언가 하려 한다.`이며 거짓 행동명은 절대 제공하지 않는다.
- 실제 공격 패턴은 판정 전에 고정되며 해석 판정이 공격 자체를 바꾸지 않는다.
- `이해` 단계는 이미 관측된 패턴을 확정 해석한다.
- 저장·불러오기로 해석 난수를 재추첨하지 않는다.

### 5.2 미관측 패턴 안전장치

- 첫 발동은 행동명을 공개하지 않는다.
- 범용 방어·엄폐·보호 중 최소 하나는 항상 유효하다.
- 최초 관측 피해는 즉사, 강제 중상, 캠페인 소프트락, 히든 분기 영구 실패를 만들지 않는다.
- 첫 발동 뒤 패턴과 결과가 매뉴얼에 등록되고 두 번째부터 이해도 해석을 적용한다.

### 5.3 규칙 가설 카드

가설 카드는 다음 필드를 가진다.

- 규칙 문장
- 지지 근거 1개 이상
- 반박 근거 0개 이상
- 미해결 질문 0개 이상
- 현장 검증 결과
- 기록 상태: 후보 / 공식 규칙 / 위험 사례

플레이어가 기록을 선택지에 연결해 두 오답을 배제한 뒤, 남은 가설에 근거 관계를 작성해야 `유력` 또는 `이해`로 승격할 수 있다.

## 6. 최소 코어와 지원 시스템

### 6.1 PROJECT_CORE - 바뀌면 다른 게임이 되는 계약

1. 관측 가능한 단서와 매뉴얼을 비교한다.
2. 플레이어가 규칙 가설과 근거 관계를 직접 구성한다.
3. 불완전한 가설에 위험을 걸어 검증하고 이해를 갱신한다.
4. 현장 이해도가 회수 전투의 전조 정보 우위로 변환된다.
5. 턴제 대응으로 포획 조건을 열며 괴이를 처치하지 않는다.
6. 성공과 실패를 괴이 매뉴얼의 다음 생존 규칙으로 남긴다.

### 6.2 CORE_SUPPORT - 코어를 증폭하지만 교체 가능한 구조

- 권나래 고정 주인공과 최대 2인 서포트
- 체력·부상·회복과 3축 결과 등급
- 포획·연구·괴이 전용 대응
- 기간제 챕터와 마감
- 세력 의뢰와 소량 자원
- 히든 분기와 가치관 엔딩
- 기록관 아카, DB, 보고서, 컷인·연출

### 6.3 CHANGEABLE

- 이해도 단계별 실제 확률
- 패턴 수, 전투 턴 수, 선택지 문구와 수량의 세부 조정
- 부상 지속 시간, 결과 등급 공식, 자원 수치
- 챕터 길이, 의뢰 수, 엔딩 축 가중치
- UI 배치와 재활용 미니게임 종류

## 7. MDA·DDD 분석

### Mechanics

단서 관측, 관련 매뉴얼 자동 제시, 선택지 배제, 가설 카드, 위험 검증, 이해도, 전조 해석, 턴제 대응, 포획 창, 매뉴얼 승격.

### Dynamics

불확실성을 줄임 → 현재 근거로 결단 → 괴이 반응을 관찰 → 가설을 수정 → 전조를 읽음 → 대응 타이밍을 선택 → 포획 → 지식을 보존.

### Aesthetics

- `Aha`: 규칙을 간파하는 쾌감
- `Dread`: 남은 불확실성에 위험을 거는 긴장
- `Mastery`: 조사 지식이 전투 정보 우위로 증명되는 숙련감
- `Authorship`: 내가 만든 매뉴얼이라는 소유감
- `Care`: 더 깊은 이해로 괴이와 일반인의 비극을 줄이는 감정적 보상

### Digital Dopamine Design

- 첫 의미 있는 보상: 첫 5분 안에 매뉴얼 근거로 오답 1개 배제
- 행동-피드백 지연: 근거 연결은 즉시, 가설 검증은 1개 장면 이내, 전조 적용은 다음 회수 턴 이내
- Micro: 선택지 배제와 새 반응 단서
- Session: 예상한 패턴을 읽고 포획 창 열기
- Meta: 공식 규칙·위험 사례·전용 대응·새로운 결말
- 금지: 수치 팝업 난사, 불필요한 랜덤 보상, 매뉴얼을 단순 수집률로 축소

## 8. SWOT와 행동 변환

### Strengths

- 조사와 전투가 이해도·전조로 직접 연결됨
- 성공과 실패가 모두 영구 지식이 됨
- 기록국과 플레이어 작성형 매뉴얼이 강한 역할 판타지를 제공
- 고정 주인공 권나래로 책임과 선택을 서사적으로 축적 가능

### Weaknesses

- 텍스트 추리와 턴제 전투 두 문법을 함께 온보딩해야 함
- 사건마다 공정한 단서·가설·패턴을 제작하는 콘텐츠 비용이 큼
- 플레이어 행동 증거가 없음
- 지원 시스템이 다시 전면으로 나오면 과밀화 가능

### Opportunities

- 기관형 도시괴담 경쟁작과 달리 ‘다시 나타날 괴이의 생존 규칙’을 직접 작성하는 포지션
- 한국 도시 장소·공공 기록·소문을 규칙 시스템과 결합
- 짧은 단일 사건 Steam 데모로 인과 루프를 선명하게 검증 가능
- 규칙 가설 카드가 사건 제작 표준과 사용자 생성형 토론 소재가 될 수 있음

### Threats

- Urban Myth Dissolution Center의 표면적 기관·도시괴담 비교
- 전투가 일반 RPG처럼 보이면 탐정 플레이어의 기대 이탈
- 확률 예측이 불공정하게 느껴질 경우 핵심 약속 붕괴
- 챕터·의뢰·연구·서사 확장으로 PoC 범위 재팽창

### SO / WO / ST / WT

- **SO:** 첫 공개 슬라이스와 마케팅 문장을 `전조를 읽고 포획한다`에 집중
- **WO:** 한 사건, 한 괴이, 세 패턴만 제작해 두 장르의 인과를 먼저 검증
- **ST:** 조사 가설 카드가 회수 화면의 전조 설명과 직접 연결되는 시각적 추적선을 제공
- **WT:** 플레이 증거 전까지 시장·세력·대규모 서사·복수 챕터 구현을 금지

## 11. PoC 계약

### 범위

- 사건 1개, 18~25분
- 조사 장면 3개
- 핵심 단서 6개: 지지 3, 반박 2, 미해결 1
- 4지선다 1회 이상, 직접 배제 2개
- 규칙 가설 카드 2개
- 현장 이해도 4단계
- 회수 패턴 3개: 관측 가능 2, 미관측 핵심 1
- 회수 전투 5~8턴
- 결과 3개: 정상 포획, 비용 있는 포획, 긴급 포획
- 지원 시스템은 체력과 범용 방어만 사용

### 성공 지표

- 80% 이상이 사건 규칙과 대응 이유를 자기 말로 설명
- 70% 이상이 무작위 시도 없이 올바른 근거로 선택지 1개 이상 배제
- 70% 이상이 조사가 회수 전투를 더 읽기 쉽게 만들었다고 행동과 인터뷰에서 확인
- 핵심 실패를 ‘난수 때문에 정답을 빼앗김’으로 설명하는 비율 20% 이하
- 80% 이상이 매뉴얼을 시스템 정답집이 아니라 자신이 만든 기록으로 인식
- 미관측 핵심 패턴에서 대응할 방법이 없었다는 보고 20% 이하

### Production gate

- 지표 통과: `KEEP / AMPLIFY` 후 CORE-MVP-002 진입
- 규칙 설명은 통과하지만 전투 인과가 약함: 전조 UI·포획 행동 `CHANGE / RETEST`
- 배제가 찍기 또는 매뉴얼 자동 정답으로 인식됨: 가설 카드·근거 연결 `CHANGE / RETEST`
- 난수 불공정이 반복됨: 중간 단계 확률 축소 또는 후보 정보 방식 `CHANGE`
- 조사와 전투가 별개로 인식됨: `HOLD`, 지원 시스템 확장 금지

## 12. KEEP / AMPLIFY / CHANGE / DEFER / REMOVE

### KEEP

- 안정화·회수·잔향·위험 사례·괴이 매뉴얼
- 관측 가능한 공정 정보와 실패 학습
- 권나래 고정 주인공과 단일 일정 주체
- 현장 짧은 대사 / 일상 충분한 대화
- 저장·ID·보호 경로 호환 원칙

### AMPLIFY

- 플레이어가 직접 작성하는 규칙 가설 카드
- 단서의 지지·반박·미해결 관계
- 조사 정보가 전투 전조로 변환되는 시각적 인과
- 괴이 매뉴얼의 재출현 판단 자산화
- 요원별 관찰 질문과 위험 해석

### CHANGE

- 핵심 단서·가설 정합성을 좌우하는 난수
- 자동 예측의 정답성 또는 거짓 정보
- 회수 화면의 수치 우선 위계
- 회수 전투의 HP·화력 중심 표현
- 제각각인 미니게임의 인지 문법

### DEFER

- MVP-044~046의 대량 서사·관계·연출 확장
- 시장·세력·연구 트리 확대
- 복수 챕터와 대규모 엔딩 조합
- 추가 괴이 수량 확정

### REMOVE

- 플레이어 판단을 대체하는 자동 최적 요원 선택
- 미관측 패턴의 즉사·강제 중상·영구 분기 실패
- 유사 괴이 자동 분류에 의한 이해도 전이
- 히든 분기를 항상 우월하게 만드는 보상 공식

## 13. Regression recheck

보호된 장점:

- 조사와 회수 전투의 직접 연결
- 합리적 확신 뒤 위험을 감수하는 재미
- 실패의 정보 가치
- 괴이별 영구 지식과 현재 출현별 갱신
- 서사적 히든 분기의 더 나은 결말 가능성
- 권나래 고정 주인공과 서포트 비독립 일정

새 결함 여부:

- 코어를 축소했지만 승인된 지원 시스템은 삭제하지 않고 후속 방향으로 보존함
- 확률을 제한했지만 `단서 확보/유력` 단계의 긴장과 범용 대응 선택은 유지함
- 가설 카드를 추가했지만 자유 문장 입력을 강제하지 않고 기존 4지선다·근거 연결 UI 안에서 구현 가능함
- 미관측 패턴을 보호했지만 최초 조우의 공포와 정보 획득 비용은 유지함

## 14. 남은 미검증

- 신규 플레이어가 실제로 두 오답을 근거로 배제하는지
- 중간 이해도 확률이 긴장으로 느껴지는지 불공정으로 느껴지는지
- 회수 전투 5~8턴이 추리의 증명으로 느껴지는지 반복으로 느껴지는지
- 서포트 요원이 수치 카드가 아닌 관찰 관점으로 기억되는지
- 포획 연구와 히든 분기가 실제 반복 플레이 동기가 되는지

이 항목들은 문서 검토로 확정할 수 없으며 CORE-MVP-001·002의 플레이테스트 증거가 필요하다.
