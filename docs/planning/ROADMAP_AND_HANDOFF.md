# 괴이 기록국 통합 로드맵·인수인계 기획서

> 문서 위치: `docs/planning/ROADMAP_AND_HANDOFF.md` | 현재 상태: `docs/CURRENT_STATUS.md` | 실행 로드맵: `MVP_ROADMAP.md`

## 1. 목적

이 문서는 새 기획자·ChatGPT·Codex가 **무엇이 구현됐고, 무엇이 승인 계획이며, 어떤 순서와 검증으로 다음 작업을 진행해야 하는지** 한 번에 파악하도록 한다.

세부 대사·컷인·데이터 값은 분야별 기획서와 실제 파일을 따른다. 이 문서는 단계 의존성, 진입 기준, 완료 기준, 인수인계 산출물을 책임진다.

## 4. MVP-044 — 사건·일상 서사 확장

### 목표

세 사건의 상황지시문과 아카 대사를 정교화하고, 사건 종료 뒤 인물과 세계관이 이어지는 AFTER·DAILY·FACTION 콘텐츠를 실제 게임 흐름에 연결한다.

### 플레이어 가치

- 현장에서 무엇이 달라졌는지 더 쉽게 읽는다.
- 기록관 아카가 관제 AI로서 일관된 역할을 가진다.
- 사건이 해결 화면에서 끝나지 않고 인물의 기억과 조직의 기록으로 이어진다.
- 초기 요원과 외부 접점의 말투·가치관을 체감한다.

### 진입 기준

- 현재 세 사건의 장면·선택·결과·회수 ID 목록이 확보됨
- 내부 `로그` 키와 플레이어 표시명 `기록관 아카`의 경계 확인
- 기존 일상 카드 구조·저장 필드·DB 기록 확인
- 보호 경로와 저장 회귀 범위 명시

### 구현 순서

1. 사건 1편의 상황지시문·아카 대사 수직 구현
2. 해당 사건 AFTER 1편 연결
3. DAILY 1편과 FACTION 1편을 기존 일상 구조에 연결
4. 저장·재진입·DB·중복 보상 검증
5. 패턴을 나머지 사건과 에피소드로 확장

### 완료 기준

- 세 사건의 대사와 상황지시문이 실제 ID에 연결됨
- 아카가 미확보 정보와 정답을 누설하지 않음
- 서포트 대사가 자동 능력·결정적 반응에 제한됨
- 선택형 서사가 반일·위험·의뢰·필수 단서를 변경하지 않음
- 완료 기록과 DB 문장이 저장 왕복 후 중복되지 않음
- 말투만으로 주요 인물을 구분할 수 있음

### 후속 전달 상태

- 안정된 장면 ID
- 에피소드 완료 ID
- 관계 선택 후보
- 표정·컷인이 필요한 중요 비트 목록

## 5. MVP-045 — 관계 태그·선택 기억·연속 이벤트

### 목표

주인공과 요원, 요원끼리, 기록국과 외부 접점의 관계가 한 장면에서 끝나지 않고 다음 대사와 행동으로 이어지게 한다.

### 플레이어 가치

- 선택이 점수 팝업보다 인물의 태도로 기억된다.
- 요원들이 수치 카드가 아니라 서로 영향을 주는 동료로 보인다.
- 외부 인물이 일회성 의뢰 NPC가 아니라 고유한 책임 철학을 가진다.

### 진입 기준

- MVP-044의 서사 ID와 완료 상태가 안정됨
- 기존 신뢰 수치·보조 이벤트·저장 상태 감사 완료
- 관계 태그를 새 저장 필드 없이 파생할 수 있는지 검토
- 첫 연속 체인 1개의 선행·분기·합류가 설계됨

### 구현 순서

1. 권나래 × 요원 체인 1개를 처음부터 끝까지 연결
2. 선택 기억 저장·불러오기·중복 방지 검증
3. 요원 × 요원 체인 1개 연결
4. 외부 접점 체인 1개 연결
5. 12개 체인·약 30장면으로 확장

### 완료 기준

- 관계가 신뢰·경계·부채·경쟁·공감·보호·균열 같은 태그와 대사 변화로 보임
- 숫자형 연애 호감도 UI가 없음
- 관계가 핵심 정답·전투 보너스·필수 진행을 소유하지 않음
- 동일 선택·장면 반복으로 태그와 보상을 중복 획득하지 않음
- 저장 후 이전 선택을 반영한 대사와 장면 해금이 유지됨
- 선택하지 않은 요원은 관계 반응에 등장하지 않음

### 후속 전달 상태

- 관계 태그 의미와 우선순위
- 장면별 감정 키
- 관계 전환 컷인 후보
- 인물 배치·거리·표정 차이 요구사항

## 6. MVP-046 — 대화 UI·표정·컷인·이벤트 연출

### 목표

기존 사건과 선택형 서사를 공용 대화 스테이지와 의미 기반 프레젠테이션 데이터로 표현해 장면 몰입·감정 가독성·접근성을 높인다.

### 플레이어 가치

- 누가 말하고 어떤 태도가 변했는지 즉시 이해한다.
- 중요한 능력·규칙·관계 순간이 절제된 컷인으로 기억된다.
- 연출을 줄이거나 자산이 없어도 텍스트로 모든 정보를 이해한다.

### 진입 기준

- 기존 대화·일상·조사 화면의 상태 소유자와 입력 흐름 감사
- MVP-044 대사 데이터와 MVP-045 관계 반응의 의미 키 확인
- 실제 자산 manifest와 누락 자산 목록 확인
- 1280×720 기준 목업과 구현 화면 차이 정리

### 구현 순서

1. 기존 기능을 감싸는 공용 대화 스테이지
2. 두 캐릭터·한 장면의 표정 의미 키와 폴백
3. 능력 발동 컷인 1종
4. 관계 반응 컷인 1종
5. 모션 감소·점멸 제한·텍스트 폴백
6. 입력·해상도·복귀 회귀
7. 캐릭터와 장면 확장

### 완료 기준

- 공용 UI가 기존 사건·일상 흐름과 저장 상태를 바꾸지 않음
- 표정은 의미 키로 요청되고 기본 표정 폴백이 작동함
- 컷인은 6종 체계 안에서 의미 있는 순간에만 사용됨
- 컷인·오버레이 중 선택지 중복 입력과 포커스 충돌이 없음
- 1280×720과 1920×1080에서 대사·선택·이름표가 잘리지 않음
- 연출 감소·색상 독립·텍스트 우선 접근성이 작동함

- 기존 세력 인물·관계 이벤트와 연결됨
- 고유 보상 ID와 저장 멱등성이 확보됨

- 필수 구간 플레이타임 실측
- 16:9 해상도와 입력 검수
- 저장·불러오기·종료 복구
- 스크린샷용 대표 장면 선정
- 소개 문구와 괴이 기록국의 핵심 약속 통일
- 접근성·옵션·크레딧·라이선스 확인

## 9. 역할별 인수인계

### 기획 AI / ChatGPT

- 현재 상태와 실제 파일의 차이를 먼저 확인한다.
- 플레이어 가치, 장면 질문, 포함·제외 범위, 수용 기준을 작성한다.
- 외부 사례는 원리를 추출하고 프로젝트 적용·제외 이유를 남긴다.
- 구현 문서에는 필요한 맥락만 전달하고 장문 조사 원문을 넣지 않는다.

### Codex

- ZIP의 `00_README`와 `IMP-00` 사전 감사부터 시작한다.
- 보호 경로와 사용자 변경을 보존한다.
- 가장 작은 end-to-end 변경을 구현한다.
- 실제 코드·데이터·씬·테스트로 완료를 검증한다.
- 구현되지 않은 계획을 완료 기능으로 문서화하지 않는다.

### 사용자

- 캐릭터성, 장면 감정, 화면 가독성, 플레이 감각을 승인한다.
- 저장·진행·경제·엔딩의 고위험 변경 범위를 결정한다.
- 대표 장면과 최종 미감의 우선순위를 확정한다.
