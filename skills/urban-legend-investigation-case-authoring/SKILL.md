# Urban Legend Investigation Case Authoring

괴이는 처치 대상이 아니다. 플레이어가 **관찰 가능한 전조를 읽고, 가설을 세우며, 근거를 확보해, 스스로 대응을 선택**하게 만들어 현재 출현을 안정화하고 잔향을 회수하는 현상이다.

## Read first

`AGENTS.md` → `docs/CURRENT_STATUS.md` → `docs/planning/PROJECT_DIRECTION.md` → `docs/GAME_DESIGN_DOCUMENT.md` → `docs/MINIGAME_SYSTEM_SPEC.md`(해당 시) → 대상 `data/episodes/*`와 관련 테스트.

## Case contract

| 요소 | 반드시 정의할 내용 |
|---|---|
| 현상과 금기 | 반복되는 규칙, 발동 조건, 플레이어가 볼 수 있는 전조 |
| 조사 질문 | 질문 목적, 가능한 답, 정보 변화, 정답을 직접 말하지 않는 이유 |
| 가설 | 플레이어가 비교 가능한 후보와 각 후보를 지지·배제할 관찰 |
| 근거 | 출처·획득 경로·해석·가설과의 관계·놓쳤을 때의 공정한 대체 경로 |
| 대응 | 선택 조건, 위험, 현장 결과, 안정화와 공식 규칙 승격의 차이 |
| 실패 | 위험 사례, 비용·학습, 재시도 또는 다른 근거 탐색 경로 |
| 매뉴얼 | `후보 / 공식 규칙 / 위험 사례` 상태와 중복 병합 규칙 |

## Authoring rules

- `상황 설명 → 선택 → 짧은 결과`의 리듬을 유지한다. 긴 설명으로 다음 클릭을 답처럼 지시하지 않는다.
- 핵심 대응은 요원·아카·장비·자동행동이 대신 고르거나 숨은 정답을 알려 주지 않는다.
- 대응이 맞아도 필요한 근거가 부족하면 공식 규칙으로 승격하지 않는다. 검증 대기 후보와 현장 안정화는 별도 상태다.
- 실패는 진행을 막지 않되 위험 사례와 배제 근거를 남긴다. 같은 실패를 새 행으로 중복하지 않고 시도 횟수와 사유를 갱신한다.
- 미니게임은 조사 마지막의 규칙 검증이며, 그 성공이 곧 안정화·회수 전체를 대체하지 않는다.
- 대사는 조사에서 짧고 기능적으로 사용한다. 아카의 표현은 안내하되 미확보 단서·정답·숨은 수치를 누설하지 않는다.
- 기존 사건 ID·저장 스키마·저승역 외 직접 대응 흐름을 불필요하게 바꾸지 않는다.

## Fairness review

각 사건을 다음 질문으로 공격한다.

1. 첫 선택 전에 플레이어가 읽을 수 있는 전조가 있는가?
2. 두 가설 이상이 처음에는 그럴듯하며, 근거로 하나를 배제할 수 있는가?
3. 실패 결과가 왜 틀렸는지와 다음 조사 방향을 알려 주되 정답을 자동 제공하지 않는가?
4. 정답이 운·외부 지식·숨은 수치가 아니라 사건 안의 증거로 결정되는가?
5. 공식 매뉴얼과 결과 화면이 같은 기록 상태를 소비하는가?

## Output

사건 목적, 규칙·전조, 조사 흐름, 가설·근거 표, 대응·결과 표, 위험 사례·매뉴얼 전이, 필요한 데이터·저장 영향, 정상·실패·경계 검증, 미확정을 분리한다.

## Failure conditions

- 괴이를 HP·공격·처치 중심의 전투 적으로 바꾼다.
- 자동 보조가 플레이어의 가설·대응 선택을 대체한다.
- 맞는 대응 하나만 강요하면서 근거 확인과 매뉴얼 검증을 생략한다.
- 위험 사례를 삭제하거나 같은 실패를 중복 기록한다.
