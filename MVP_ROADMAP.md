# MVP_ROADMAP

## 현재 기준

현재 문서 기준선은 `MVP-037`이다.

## MVP-037 작업 기준

- 최대 10일 동안 세 사건을 비선형으로 추적하는 데모 운영 코어를 사용한다.
- 필수 진행은 약 125~155분, 일상·연구·세력 콘텐츠 포함 전체 진행은 약 180~220분을 목표로 한다.
- 전 요원의 오전·오후 일정을 정한 뒤 조사에 들어가며 결과 화면에서 하루를 한 번만 마감한다.
- 미해결 괴이 위험도는 기본 1건만 상승하고, 높은 확산 상태에서만 2건까지 상승한다. 같은 사건을 2일 넘게 순환 대상에서 제외하지 않는다.
- 폭주는 게임 오버가 아니라 긴급 조사 또는 포기 판단으로 전환한다.
- 일상 콘텐츠의 사건 이해도는 최초 관람 +2, 짧은 후속 +1, 하루 1회, 사건별 최대 +10이며 단서·회수를 대체하지 않는다.
- `CampaignState`가 계산을 맡고 `GameState`는 저장과 화면 연결 파사드만 제공한다.
- 저장 기준은 `mvp-037`, 화면 표기는 `Ver 3.7`이다.

## MVP-035 작업 기준

- 괴담기록국 AI `로그`가 전자기기 데스크와 현장 단말기를 통해 주요 화면의 안내와 정보 전달을 맡는다.
- 로그도 요원처럼 초상·화자명·대사로 참여하며 평상·집중·경고 표현을 구분한다.
- 첫 상세 안내는 한 번만 재생하고, 이후에는 짧은 반복 힌트만 표시하며 확인 상태를 저장한다.
- 세 가지 알림음은 같은 3음 모티프를 공유하고 화면 색·문구 변화로 동일 정보를 함께 전달한다.
- 로그는 플레이어가 아직 확보하지 않은 단서, 대응 정답, 숨은 효과 수치를 공개하지 않는다.
- 저장 기준은 `mvp-035`, 화면 표기는 `Ver 3.5`이다.

## MVP-034 작업 기준

- 사건 준비의 외부 접점에서 소문시장 거래와 마도회·퇴마사 의뢰를 관리한다.
- 잔향 파편 지급은 고유 보상 ID로 멱등 처리한다.
- 관계 단계는 적대·경계·중립·우호·신뢰이며 시장 가격과 거래 잠금에 반영한다.
- 영구 장비는 공용 장비 슬롯, 소모품은 사건당 최대 2종·종류별 3개 반입 규칙을 사용한다.
- 저장 기준은 `mvp-034`, 화면 표기는 `Ver 3.4`이다.

## MVP-033 작업 기준

- 대화와 조사를 `field_nodes` 기반의 연속 현장 장면으로 통합한다.
- 회수는 전조 → 자동 예측 → 상황형 대응 → 괴이 행동 → 자동 보조 순으로 진행한다.
- 예측은 성공 연속 억제와 실패 연속 구제를 함께 두며 거짓 정보를 주지 않는다.
- 오대응은 피해와 함께 다음 판단에 사용할 패턴 학습 기록을 남긴다.
- 저장 기준은 `mvp-033`, 화면 표기는 `Ver 3.3`이다.

## 플랫폼 목표

현재 프로젝트의 1차 출시 목표는 **PC 기준 Steam 출시**다.

UI/UX 판단은 다음 기준을 우선한다.

- PC 16:9 화면
- 마우스/키보드 조작
- Steam 데모/상점 페이지에서 이해되는 첫인상
- 창 모드와 전체화면에서 모두 읽히는 정보 위계
- 패드 대응은 후순위 검토
- 모바일 세로 화면 최적화는 별도 요청 시에만 포함

```text
기존 2편은 유지하고, 비선형 운영 루프 검증을 위한 세 번째 사건
`폐주파수 방송국`을 추가해 3편 데모를 구성한다.
```

1. `저승역`
2. `비 오는 골목의 빨간 우산`
3. `폐주파수 방송국` (콘텐츠 구현 예정, 캠페인 상태는 미확인으로 선반영)

## MVP-028 작업 기준

MVP-028은 PC/Steam 데모 기준 UX 패스다. 핵심은 대화, 조사, 회수 화면이 서로 다른 역할을 갖되 하나의 게임처럼 읽히게 만드는 것이다.

핵심 목표:

```text
플레이어는 별도 주인공 1명을 조작하는 것이 아니라,
괴담기록국 요원 팀을 이끌며 상황을 읽고 선택하고 단서를 근거로 회수한다.
```

### 벤치마킹 반영 기준

- Unity Asset Store: UI 키트/템플릿의 카드형 배치, 기능별 패키징, 상점형 요약 방식 참고
- Tales of Tuscany Demo: 짧은 기능 소개, 팀/캐릭터 교체, 선택형 상호작용, 강한 첫인상 구성 참고
- 사용자 제공 이미지: 대화/텍스트 노벨/전투 UI의 화면 배치와 정보 위계 참고

### UX 전환 기준

| 화면 | 정리 기준 |
|---|---|
| 대화 화면 | 인물/배경 몰입, 하단 대사창, 주변부 보조 UI |
| 조사 화면 | 텍스트 노벨형 상황 설명 → 선택지 → 결과 → 단서/상태 변화 |
| 회수 화면 | 좌상단 아군 요원, 우상단 해결 단서/회수 근거, 하단 행동 선택 |

### 팀 기반 진행 기준

- 별도 고정 주인공 캐릭터를 두지 않는다.
- 플레이어는 괴담기록국 요원 팀을 운용한다.
- 대화/조사/회수는 선택 요원들의 관점, 반응, 지원으로 진행된다.
- 회수 페이즈에서는 선택 요원 중 대표 캐릭터를 교체할 수 있게 설계한다.
- 대표 교체는 전투 캐릭터 교체가 아니라 `현장 지휘/회수 담당 전환`으로 표현한다.

## MVP-032 작업 기준

- 메인 메뉴의 편성을 사건 준비로 옮기고, 사건 준비에서 2~3명 편성과 상세 프로필을 확인한다.
- 요원은 `제압·분석·방호·치료·교감`, 현재/최대 체력·정신력, 고유 장비·기술·면모를 가진다.
- 조사에서는 현재 선택에 관계된 능력과 담당 요원만 노출하며, 실패해도 다음 판단에 쓸 정보를 남긴다.
- 회수에서는 다섯 능력과 장비·기술 보정을 사용하고, 체력 0/정신력 0 시 행동 불능으로 처리한다.
- 저장 기준은 `mvp-032`, 화면 표기는 `Ver 3.2`로 유지한다.
