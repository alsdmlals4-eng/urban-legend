# MVP_ROADMAP

## 현재 기준

현재 문서 기준선은 `MVP-021`이다.

현재 MVP 상태 판단은 이 문서와 GitHub Issue #28, `docs/CODEX_GOAL_MVP_021.md`, `docs/MVP_STATUS_AUDIT.md`를 함께 기준으로 삼는다.

> GitHub Issue #20은 과거 오생성/중복 종료 이슈다. MVP-020 기준 이슈는 #27이고, MVP-021 기준 이슈는 #28이다.

## 플랫폼 목표

현재 프로젝트의 1차 출시 목표는 **PC 기준 Steam 출시**다.

따라서 UI/UX 판단은 모바일 세로 화면이 아니라 다음 기준을 우선한다.

- PC 16:9 화면
- 마우스/키보드 조작
- Steam 데모/상점 페이지에서 이해되는 첫인상
- 창 모드와 전체화면에서 모두 읽히는 정보 위계
- 패드 대응은 후순위 검토

## 전체 방향 전환

MVP-020부터는 새 괴담 에피소드를 계속 추가하지 않는다.

기준 방향:

```text
괴담 에피소드는 2편까지를 기준 샘플로 두고,
이후 MVP는 시스템·UI·안정화·데모 완성도를 높인다.
```

두 사건 샘플:

1. `저승역`
2. `비 오는 골목의 빨간 우산`

이 2개 사건으로 다음 핵심 루프를 검증한다.

```text
사건 준비
→ 요원 편성
→ 조사
→ 조사 방법 선택
→ 단서/힌트/상태 변화
→ 미니게임 또는 판정
→ 괴이 안정화/회수
→ 사건 보고서
→ 기록국 DB
→ 다음 사건 참고
```

## MVP-021 작업 기준

MVP-021은 새 기능 확장이 아니라 플레이어가 기존 루프를 더 쉽게 이해하도록 만드는 UI 정리 작업이다.

### 현재 UI 기준선

- `scripts/ui/main_menu.gd`: 프로젝트 소개, DB 진입, 새 게임, 이어하기, 저장 초기화, 테스트 씬 버튼이 한 화면에 세로로 배치되어 있다.
- `scripts/scenes/preparation_scene.gd`: 사건 선택, 요원 편성, 장비 장착, 참고 기록물, 로그 안내, 조사 시작이 한 스크롤에 표시된다.

### 목표

```text
플레이어가 다음에 무엇을 눌러야 하는지 망설이지 않고,
현재 사건·요원·장비·기록물 상태를 한눈에 이해한다.
```

### 벤치마킹 반영 기준

MVP-021부터 GPT는 매 작업에서 벤치마킹을 수행하고, 개선점/수정점을 간결한 컨텍스트로 반영한다.

고정 확인 대상:

- Unity Asset Store: 에셋/템플릿/툴 패키징, UI 키트 구성, 상점형 정보 구조 참고
- Tales of Tuscany Demo: Steam 위시리스트 유도, 데모 소개, 빠른 기능 소개, 개성 있는 선택지/미니게임 표현 참고

MVP-021 적용:

- PC/Steam 기준 첫 화면에서 `무엇을 하는 게임인지`와 `다음에 무엇을 누를지`가 즉시 보여야 한다.
- 개발용 테스트 버튼은 플레이어용 주요 행동과 분리한다.
- 준비 화면은 사건 선택/요원/장비/기록물을 점검표처럼 읽히게 한다.
