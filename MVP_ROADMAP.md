## 구현 순서

1. Codex에 대상 ZIP 하나만 업로드한다.
2. ZIP 최상단 `00_README_코덱스_전달사항.md`를 읽는다.
3. 해당 `IMP-00` 사전 감사에서 현재 `main`, 보호 경로, 저장 호환, 기존 ID를 확인한다.
4. 작은 end-to-end 패키지 하나를 구현하고 자동·수동 검증한다.
5. 완료된 범위만 `docs/CURRENT_STATUS.md`, 이 로드맵, `TEST_CHECKLIST.md`에 반영한다.

기본 의존 순서는 **MVP-044 → MVP-045 → MVP-046**이다. MVP-046의 공용 프레젠테이션 기반을 먼저 구현하는 경우에는 대사 데이터·관계 상태·저장 스키마를 변경하지 않는 독립 범위로 제한한다.

## 후속 장기 마일스톤

1. MVP-044~046의 서사·관계·연출을 세 사건 전체에서 회귀 검증한다.
2. 연구·장비·세력 선택이 정답 구매가 아닌 위험 관리로 작동하는지 재심사한다.
3. 선택 콘텐츠 포함 180~220분 플레이타임을 실측한다.
4. Steam 데모 후보 빌드, 입력·해상도·저장·스크린샷을 검수한다.
5. 플레이테스트로 정보량·난이도·연출 빈도를 확정한다.

## 유지 계약

- 오전 결과 확인 전 오후를 계획하지 않는다.
- 조사는 한 반일을 소비하며 HQ 중단·재개 시 같은 작전과 슬롯을 보존한다.
- 권나래의 일정만 필요·소비하고 서포트는 독립 일정을 요구하지 않는다.
- 저장 난수·지원 시도 상태를 유지해 불러오기 재추첨을 막는다.
- 장비·연구·자동행동·관계 이벤트가 핵심 추리를 대신하지 않는다.
- 괴이는 처치하지 않고 현재 출현을 안정화하고 잔향을 회수한다.
