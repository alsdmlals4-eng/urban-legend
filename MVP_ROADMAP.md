## MVP-028 작업 기준

MVP-028은 PC/Steam 데모 기준 UX 패스다. 핵심은 대화, 조사, 회수 화면이 서로 다른 역할을 갖되 하나의 게임처럼 읽히게 만드는 것이다.

핵심 목표:

```text
플레이어는 별도 주인공 1명을 조작하는 것이 아니라,
괴담기록국 요원 팀을 이끌며 상황을 읽고 선택하고 단서를 근거로 회수한다.
```

### 벤치마킹 반영 기준

- Unity Asset Store: UI 키트/템플릿의 카드형 배치, 기능별 패키징, 상점형 요약 방식 참고
- Tales of Tuscany Demo: 짧은 기능 소개, 팀/캐릭터 교체, 선택형 상호작용, 강한 첫인상 구성 참고
- 사용자 제공 이미지: 대화/텍스트 노벨/전투 UI의 화면 배치와 정보 위계 참고

### UX 전환 기준

| 화면 | 정리 기준 |
|---|---|
| 대화 화면 | 인물/배경 몰입, 하단 대사창, 주변부 보조 UI |
| 조사 화면 | 텍스트 노벨형 상황 설명 → 선택지 → 결과 → 단서/상태 변화 |
| 회수 화면 | 좌상단 아군 요원, 우상단 해결 단서/회수 근거, 하단 행동 선택 |

### 팀 기반 진행 기준

- 별도 고정 주인공 캐릭터를 두지 않는다.
- 플레이어는 괴담기록국 요원 팀을 운용한다.
- 대화/조사/회수는 선택 요원들의 관점, 반응, 지원으로 진행된다.
- 회수 페이즈에서는 선택 요원 중 대표 캐릭터를 교체할 수 있게 설계한다.
- 대표 교체는 전투 캐릭터 교체가 아니라 `현장 지휘/회수 담당 전환`으로 표현한다.

## MVP-032 작업 기준

- 메인 메뉴의 편성을 사건 준비로 옮기고, 사건 준비에서 2~3명 편성과 상세 프로필을 확인한다.
- 요원은 `제압·분석·방호·치료·교감`, 현재/최대 체력·정신력, 고유 장비·기술·면모를 가진다.
- 조사에서는 현재 선택에 관계된 능력과 담당 요원만 노출하며, 실패해도 다음 판단에 쓸 정보를 남긴다.
- 회수에서는 다섯 능력과 장비·기술 보정을 사용하고, 체력 0/정신력 0 시 행동 불능으로 처리한다.
- 저장 기준은 `mvp-032`, 화면 표기는 `Ver 3.2`로 유지한다.
