## 구현 완료선

### MVP-043 + CORE-VALIDATION-001

- 권나래 고정 주인공과 서포트 최대 2인
- 세 사건의 조사·판단·안정화·잔향 회수·보고서·DB
- 가설·근거·대응 단계형 저승역 판단
- 공식 규칙·검증 대기 후보·위험 사례의 저장·결과·DB 연결
- 3×3 학습과 4×4 최종 검증
- HQ 일정 비소모 일상 에피소드와 세력 의뢰

### UX-PD-001 2A

- 준비 화면에서 사건·편성을 1차 정보로 유지
- 장비·외부 접점·기록을 `보조 준비 도구`로 접고 즉시 복원
- 기존 `seen_hint_ids`를 사용해 공개 상태 저장
- 진행 저장의 기존 보조 콘텐츠를 감지해 자동 공개
- 경제·편성·일정 상태 소유자는 기존 코드 그대로 유지

## 실행 규칙

1. `docs/CURRENT_STATUS.md`와 책임 기획서를 먼저 확인한다.
2. 현재 `main`, 보호 경로, 저장 호환, 기존 ID를 감사한다.
3. 작은 end-to-end 단위 하나만 구현한다.
4. Godot import, 전용 테스트, 전체 회귀, 1280×720·1920×1080을 검증한다.
5. 완료된 범위만 상태·로드맵·체크리스트·책임 기획서에 반영한다.
6. 구현되지 않은 계획을 완료 기능으로 문서화하지 않는다.

## 유지 계약

- 오전 결과 확인 전 오후를 계획하지 않는다.
- 조사 중 HQ 중단·재개 시 같은 작전과 슬롯을 보존한다.
- 저장 난수·지원 시도·게시판 상태를 유지한다.
- 장비·연구·요원·관계·UI가 핵심 추리를 대신하지 않는다.
- 괴이는 처치하지 않고 현재 출현을 안정화하고 잔향을 회수한다.
- 저장 Schema 변경은 별도 승인과 이관 테스트 없이 수행하지 않는다.
