## 플랫폼 목표

현재 프로젝트의 1차 출시 목표는 **PC 기준 Steam 출시**다.

따라서 UI/UX 판단은 모바일 세로 화면이 아니라 다음 기준을 우선한다.

- PC 16:9 화면
- 마우스/키보드 조작
- Steam 데모/상점 페이지에서 이해되는 첫인상
- 창 모드와 전체화면에서 모두 읽히는 정보 위계
- 패드 대응은 후순위 검토

## 전체 방향 전환

MVP-020부터는 새 괴담 에피소드를 계속 추가하지 않는다.

기준 방향:

```text
괴담 에피소드는 2편까지를 기준 샘플로 두고,
이후 MVP는 시스템·UI·안정화·데모 완성도를 높인다.
```

두 사건 샘플:

1. `저승역`
2. `비 오는 골목의 빨간 우산`

이 2개 사건으로 다음 핵심 루프를 검증한다.

```text
사건 준비
→ 요원 편성
→ 조사
→ 조사 방법 선택
→ 단서/힌트/상태 변화
→ 미니게임 또는 판정
→ 괴이 안정화/회수
→ 사건 보고서
→ 기록국 DB
→ 다음 사건 참고
```

## MVP-022 작업 기준

MVP-022는 새 기능 확장이 아니라 플레이어가 조사 중 얻은 정보를 더 쉽게 추적하도록 만드는 UI 정리 작업이다.

### 현재 UI 기준선

- `scripts/scenes/investigation_scene.gd`: 코드로 UI를 생성한다.
- 현재 화면은 사건 상태, 조사 포인트, 조사 방법, 결과 텍스트, 힌트 텍스트, 단서 목록을 한 스크롤 흐름에 표시한다.
- 조사 방법 결과는 판정식, 상태 변화, 신뢰도, 이벤트가 한꺼번에 출력되어 정보 밀도가 높다.

### 목표

```text
조사 포인트를 누른 결과가 지나가는 텍스트가 아니라,
사건 기록으로 축적되고 다음 판단에 쓰인다는 느낌을 준다.
```

### 벤치마킹 반영 기준

GPT는 매 작업에서 벤치마킹을 수행하고, 개선점/수정점을 간결한 컨텍스트로 반영한다.

고정 확인 대상:

- Unity Asset Store: GUI/Tools/Templates처럼 기능을 명확히 분류하는 정보 구조 참고
- Tales of Tuscany Demo: 짧은 기능 소개와 즉시 이해되는 상호작용 안내 방식 참고

MVP-022 적용:

- 조사 화면은 상태, 조사 포인트, 결과 로그, 단서, 힌트, 회수 안내를 기능 영역으로 분리한다.
- 단서와 힌트는 같은 정보가 아니므로 UI에서도 분리한다.
- 조사 방법 결과는 `방법 → 성공/실패 → 새 정보 → 상태 변화 → 요원 반응 → 다음 행동` 순서로 읽히게 한다.
