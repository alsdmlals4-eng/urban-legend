## 실행 규칙

1. Codex에 대상 ZIP 하나만 업로드한다.
2. ZIP 최상단 `00_README_코덱스_전달사항.md`를 읽는다.
3. 해당 `IMP-00` 사전 감사에서 현재 `main`, 보호 경로, 저장 호환, 기존 ID를 확인한다.
4. `docs/planning/`의 분야별 기획서에서 플레이어 가치와 금지 방향을 확인한다.
5. 작은 end-to-end 패키지 하나를 구현하고 자동·수동 검증한다.
6. 완료된 범위만 `CURRENT_STATUS`, 이 로드맵, `TEST_CHECKLIST`, 분야별 기획서에 반영한다.

전달 문서가 존재한다는 이유만으로 `구현 확인`으로 표시하지 않는다.

7. 보고서·DB·괴이 매뉴얼

이 단계에서 필수 구간과 선택 콘텐츠 포함 플레이타임을 실측한다.

## 후속 장기 마일스톤

1. 연구·장비·세력 선택이 정답 구매가 아닌 위험 관리로 작동하는지 재심사한다.
2. 관계·세력 이벤트와 경제 선택의 연결 범위를 결정한다.
3. 필수 125~155분, 선택 포함 180~220분 목표를 실제 측정값으로 재조정한다.
4. Steam 데모 후보 빌드, 입력·해상도·저장·스크린샷·라이선스를 검수한다.
5. 플레이테스트로 정보량·난이도·연출 빈도를 확정한다.

## 유지 계약

- 오전 결과 확인 전 오후를 계획하지 않는다.
- 조사는 한 반일을 소비하며 HQ 중단·재개 시 같은 작전과 슬롯을 보존한다.
- 권나래의 일정만 필요·소비하고 서포트는 독립 일정을 요구하지 않는다.
- 저장 난수·지원 시도·게시판 상태를 유지해 불러오기 재추첨을 막는다.
- 장비·연구·자동행동·관계 이벤트가 핵심 추리를 대신하지 않는다.
- 관계는 수치형 연애 호감도가 아니라 선택 기억과 후속 반응으로 표현한다.
- 아트·표정·컷인·UI는 진행 상태를 표현하지만 대신 소유하지 않는다.
- 괴이는 처치하지 않고 현재 출현을 안정화하고 잔향을 회수한다.

## 최신화 규칙

- 큰 단계·MVP 종료 시 현재 기준과 다음 구현 순서를 갱신한다.
- 완료 상세는 로드맵 본문에 누적하지 않고 `docs/qa/` 또는 날짜별 백업으로 이동한다.
- 오래된 다음 작업이 남지 않도록 `docs/CURRENT_STATUS.md`와 함께 갱신한다.
- 기획 방향 변경은 관련 활성 기획서와 로드맵의 진입·종료 조건을 함께 수정한다.
- 5개 MVP마다 중복·구문서·깨진 참조·불필요한 기본 읽기 문서를 감사한다.
