## MVP-038 유지 계약

- 오전 계획·실행·결과 확인 뒤 오후를 계획한다.
- 조사는 반일을 소비하고 HQ 복귀 시 같은 작전과 슬롯을 보존한다.
- 세력 의뢰는 3칸 게시판의 파견형·회수 행동형과 공용 1d100 판정을 사용한다.
- 신규 캠페인은 조사 배치가 있을 때만 사건을 선택한다.
- 두 구현 사건은 `lead`, 폐주파수 방송국은 `unknown`으로 시작한다.
- 저장 난수 상태를 유지해 불러오기 재추첨을 막는다.

## 현재 작업 순서

| 우선순위 | 작업 | 완료 기준 |
|---:|---|---|
| P0 | 살아 있는 게임기획서 체계 | Markdown·DOCX 동기화, 현행·보관 문서 분리 |
| P0 | 추리 UX 검증 패스 | 회수 전 근거·힌트·패턴 학습·결과 보고서가 페어플레이 추리로 읽히는지 두 사건에서 확인 |
| 완료 | 세 사건 전체 캠페인 수동 QA | Day 8 자연 발견, 세 사건 보고서·DB·저장 왕복, 오대응 학습, 10일 마감 확인 |
| P1 | 연구·장비·세력 확장 | 정답을 판매하지 않는 선택과 밸런스 검증 |
| P1 | 로그·요원 문구 패스 | 미확보 정보 누설·중복·톤 불일치 제거 |
| P2 | Steam 데모 패키징 | 16:9·조작·스크린샷·빌드·저장 검수 |
| P2 | 플레이타임 실측 | 필수 125~155분, 전체 180~220분 목표 재조정 |

추리 UX 후보의 근거·채택·보류 이유는 `docs/benchmarks/GDD_MYSTERY_UX_BENCHMARK_2026-07-14.md`와 `docs/benchmarks/TEXT_NOVEL_BENCHMARK_2026-07-14.md`를 따른다. MVP-042는 기존 서울 2033 근거를 유지해 HQ 일상 3장을 구현했고, 반일·위험·의뢰·필수 추리를 바꾸지 않는 경계를 자동·화면 QA로 확인했다. 다음 P1은 연구·장비·세력 선택의 가격·보상 검증이다.

## 장기 마일스톤

1. 세 사건이 10일 캠페인에서 비선형으로 연결된다.
2. 일상·연구·세력 콘텐츠가 추리를 대체하지 않고 선택 폭을 만든다.
3. 저장 호환성과 사건 보고서가 데모 전 구간에서 안정적이다.
4. 사용자 플레이테스트로 난이도·정보량·플레이타임을 확정한다.
5. Steam 데모 후보 빌드와 소개 자료를 완성한다.

## 제외 범위

- 전통 RPG 파티 전투와 영구 사망
- 모바일 세로 화면 기본 최적화
- 온라인·랭킹·광고·결제
- 정답 판매형 시장과 과도한 파밍
- 전체 UI 프레임워크 교체

## 최신화 규칙

- 큰 단계·MVP 종료 시 현재 기준선과 다음 작업을 갱신한다.
- 플레이어에게 보이는 설계 변경은 GDD 로드맵과 함께 심사한다.
- 완료된 상세 계획은 기본 읽기에서 제외하되 삭제하지 않는다.
- 5개 MVP마다 중복·구문서·깨진 참조·스킬 효율을 감사한다.
- GDD가 갱신되면 DOCX를 재생성하고 두 파일을 사용자에게 보여준다.
