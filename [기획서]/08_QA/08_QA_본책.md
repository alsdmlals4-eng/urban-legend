## 검증 경로

- 문서 구조: `python tools/verify_migration_inventory.py --before docs/MIGRATION_INVENTORY_BEFORE.json --after docs/MIGRATION_INVENTORY_AFTER.json`
- 링크/Registry/발행: `python tools/build_design_documents.py --registry [기획서]/00_프로젝트_허브/DESIGN_DOCUMENT_REGISTRY.json --source-commit dd3c9a8776eb938eeeeb2f1319af6bfc4a135202`
- 게임 변경이 있을 때만: Godot editor import → 관련 headless test → runtime smoke.

## 2026-07-20 실행 관찰

- Godot 4.7.1 editor import는 통과했고, `accessibility_settings_test.gd` 단독 headless는 통과했다.
- `test_agent_system.gd` 같은 상태·캠페인 스크립트는 `--script`만으로 Autoload `GameState`를 로드하지 못한다. 대응 `.tscn`은 자체 종료하지 않아 headless 실행기가 대기한다.
- `test_save_guard.gd`는 독립 실행 테스트가 아니라 preload 보조 스크립트다. 자동 수집에서 제외해야 한다.
- 따라서 기존 전체 headless 묶음은 `NOT_RUN`이며, 이주 PR에서는 게임 코드를 바꾸지 않는다. 후속 별도 Issue에서 Autoload를 가진 테스트 실행기와 종료 계약을 정해야 한다.
