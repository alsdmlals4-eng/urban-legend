# 대사 작성 워크플로

이 문서는 `docs/AI_DELEGATION_WORKFLOW.md`의 대사·튜토리얼 전용 하위 규칙이다. 충돌 시 상위 문서의 보호 경로·입력 해시·산출물 검증 규칙을 먼저 적용한다.

이 문서는 대화문, 튜토리얼 안내문, 상황별 반응문을 수정할 때 Codex 토큰 사용을 줄이면서 문장 품질과 게임 데이터 안전성을 유지하는 프로젝트 전용 절차다.

## 역할

- 외부 GPT: 한국어 대사 작성, 캐릭터별 말투, 감정선, 장면 리듬을 담당한다.
- DeepSeek: 대사 위치 추출과 결과 검수를 담당한다. 새 대사를 대량 작성하지 않는다.
- Codex: 작업 지시문과 첨부 파일 목록을 만들고, 검수를 통과한 패치를 적용한 뒤 최소한의 구조·게임 검증을 수행한다.
- 사용자는 외부 GPT 실행과 결과 파일 전달을 담당한다.

내부 하위 에이전트는 Codex 사용량에 포함되므로 대사 작성의 기본 경로로 사용하지 않는다.

## 표준 순서

1. Codex가 현재 요청, 관련 장면, 캐릭터 설정, 수정 가능한 파일과 제외 범위를 확인한다.
2. Codex가 아래 템플릿을 현재 작업에 맞게 채우고 첨부 파일 목록과 함께 사용자에게 제공한다.
3. 사용자가 지시문과 원본 파일을 외부 GPT에 전달한다.
4. 외부 GPT는 채팅에 전체 파일을 출력하지 않고 `dialogue_rewrite.patch`와 `dialogue_review.md`를 파일로 반환한다.
5. 사용자가 두 파일을 Codex에 첨부한다.
6. DeepSeek가 원본과 산출물을 비교해 구조, 설정 충돌, 미확보 정보 누설을 검수한다.
7. Codex는 검수 결과와 실제 unified diff를 읽고 승인 가능한 변경만 적용한다.
8. Codex는 JSON 파싱, `git diff --check`, 관련 Godot 장면·회귀 테스트를 실행하고 결과를 보고한다.

DeepSeek 또는 Codex 검수에서 문제가 발견되면 패치를 부분 수정하거나 외부 GPT에 수정 지시문을 다시 제공한다. 검수 실패 상태의 패치를 그대로 적용하지 않는다.

## 외부 GPT 산출물 계약

### `dialogue_rewrite.patch`

- UTF-8로 저장한 적용 가능한 unified diff여야 한다.
- 저장소 루트 기준 상대 경로를 사용한다.
- 요청받은 대사 파일만 포함한다.
- 기존 ID, JSON 키, 플래그, 조건, 분기, expression 값과 들여쓰기를 보존한다.
- 전체 파일 사본, 설명문, Markdown 코드 울타리를 포함하지 않는다.

### `dialogue_review.md`

검수 분류 키 `no-new-systems`는 신규 시스템·분기·데이터를 만들지 않았다는 뜻으로 사용한다.

다음 항목을 포함한다.

- 변경 파일과 node/tutorial ID
- 화자별 말투 원칙
- 변경 이유
- 미확보 단서와 정답 누설 점검
- 신규 시스템, 신규 분기, 신규 단서·플래그를 추가하지 않았다는 확인
- 길이, expression, 위험 상황의 농담, 플레이어 선택 평가 점검
- 확신할 수 없어 변경하지 않은 항목

## 외부 GPT 작업 지시문 템플릿

Codex는 대사 수정 요청마다 대괄호 항목을 실제 값으로 채워 사용자에게 제공한다.

```text
당신은 한국어 도시괴담 조사 게임의 시나리오 라이터이자 내러티브 UX 편집자다.

목표:
[수정하려는 장면과 플레이어 경험]

대상 파일:
[저장소 상대 경로 목록]

참고 파일:
[캐릭터 설정, 사건 데이터, 기존 설계 문서]

포함 범위:
[수정할 화자, node/tutorial ID, 상황]

제외 범위:
- 신규 시스템, 신규 분기, 신규 단서, 신규 플래그, 신규 저장 필드
- 기존 ID, 조건, JSON 키와 게임 진행 구조 변경
- 플레이어가 아직 획득하지 않은 정보와 정답 행동 공개
- 다른 작품의 고유 문장과 캐릭터 말투 모방

작성 기준:
[캐릭터별 말투, UI 길이, 장면 분위기, expression 허용값]

먼저 원본에서 캐릭터 설정과 현재 공개 가능한 정보를 확인하라. 불명확한 설정을 임의로 채우지 말고 변경하지 않은 채 dialogue_review.md에 기록하라.

결과는 채팅 본문에 전체 파일을 붙이지 말고 다음 두 UTF-8 파일로 제공하라.

1. dialogue_rewrite.patch
- 저장소 루트에서 적용 가능한 unified diff
- 대사 변경만 포함
- 설명문과 Markdown 코드 울타리 제외

2. dialogue_review.md
- 변경 파일과 ID
- 변경 이유와 화자별 말투 원칙
- 미확보 정보/정답 누설 검수
- 신규 시스템·분기·단서·플래그 미추가 확인
- UI 길이와 expression 검수
- 확인이 필요한 항목
```

## DeepSeek 검수 계약

DeepSeek에는 창작이 아니라 다음 읽기 전용 검수만 맡긴다.

```text
원본 저장소와 dialogue_rewrite.patch, dialogue_review.md를 비교 검수하라. 파일을 수정하거나 새 대사를 작성하지 마라.

확인 항목:
1. 패치가 유효한 unified diff이며 요청된 파일만 수정하는가
2. 기존 ID, 키, 조건, 플래그, 분기, expression 구조를 바꾸지 않는가
3. 존재하지 않거나 현재 시점에 미확보 상태인 정보를 말하지 않는가
4. 정답 행동 직접 지시, 플레이어 선택 평가, 위험 상황 농담이 없는가
5. 신규 시스템·분기·단서·플래그·저장 필드를 만들지 않는가
6. 캐릭터 설정, 장면 맥락, UI 길이와 충돌하지 않는가
7. dialogue_review.md의 자기 검수 내용이 실제 패치와 일치하는가

결과는 PASS 또는 FAIL로 시작하고 파일, ID, 오류 분류, 근거, 최소 수정 방향을 표로 출력하라.
```

DeepSeek 검수는 `research-scout-flash`가 아니라 로컬 읽기 전용 `scout-flash`로 실행한다. 공개 웹 근거가 필요한 별도 벤치마킹만 `research-scout-flash`로 라우팅한다.

## Codex 적용 및 최소 검증

Codex는 다음 순서만 수행한다.

1. 패치 대상 경로가 사용자 승인 범위인지 확인한다.
2. `git apply --check dialogue_rewrite.patch`에 해당하는 비파괴 검사를 수행한다.
3. DeepSeek PASS와 `dialogue_review.md`의 누설 검수 내용을 실제 diff와 대조한다.
4. 승인 가능한 패치만 적용한다.
5. 변경된 JSON을 파싱하고 GDScript 문법, `git diff --check`, 관련 장면과 회귀 테스트를 실행한다.
6. 변경 파일, 적용하지 않은 항목, 검증 결과, 남은 위험을 보고한다.

외부 GPT 결과는 참고 산출물이며 자동 승인 근거가 아니다. DeepSeek 보고서만 믿지 않고 Codex가 실제 diff와 테스트 결과를 최종 확인한다.

## 기본 첨부 파일

로그 대사 작업에는 최소한 다음 파일을 사용한다.

- `scripts/ui/log_tutorial_catalog.gd`
- 해당 `data/episodes/*.json`
- 로그 설계 문서와 관련 캐릭터 설정

요원 또는 피해자 대사 작업에는 대상 에피소드 JSON과 해당 캐릭터 설정만 추가한다. 관련 없는 전체 저장소나 긴 대화 기록은 전달하지 않는다.
