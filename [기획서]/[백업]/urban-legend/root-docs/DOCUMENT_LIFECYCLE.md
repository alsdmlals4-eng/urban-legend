# Document Lifecycle

> 문서 위치: `docs/DOCUMENT_LIFECYCLE.md` | 백업 인덱스: `docs/archive/README.md`

## 목적

현재 작업자가 오래된 설계·완료 기록·중복 설명을 매번 읽지 않도록 **주제별 현행 원본 1개**와 **기본 읽기에서 제외된 백업**을 분리한다.

## 현행 문서 규칙

1. 한 주제에는 현행 원본을 하나만 둔다.
2. 구현 사실과 미구현 계획을 같은 상태로 쓰지 않는다.
3. 현행 문서 첫 부분에는 다음 형식의 위치 표시줄을 한 줄 둔다.

```text
> 문서 위치: `<현행 경로>` | 과거·상세 백업: `<백업 경로>`
```

4. 다른 현행 문서는 같은 내용을 복사하지 않고 원본 링크와 작업별 차이만 적는다.
5. 문서가 변경돼도 과거 버전 비교가 목적이 아니면 새 `v0.x` 파일을 늘리지 않고 같은 현행 파일을 갱신한다.
6. 구현 완료 뒤 상태 원본은 `docs/CURRENT_STATUS.md`, 상세 설계는 `docs/GAME_DESIGN_DOCUMENT.md`, 검증은 `TEST_CHECKLIST.md`를 갱신한다.

## 백업 전환 조건

다음 중 하나에 해당하면 `docs/archive/backup/YYYY-MM-DD/`로 보낸다.

- 구현이 끝나 현행 문서와 내용이 중복됨
- 과거 MVP의 완료 체크·감사·보고서로서 일상 작업에 필요하지 않음
- 공식 명칭·주인공·저장 구조 등 핵심 전제가 바뀜
- 다음 작업이 이미 완료됐거나 다른 계획으로 대체됨
- 외부 전달본·일회성 제안서·중간 검토본으로서 현재 원본이 아님
- 읽는 비용에 비해 현재 작업에서 사용 빈도가 낮음

백업 파일에는 원래 경로, 보관일, 보관 이유, 현행 대체 문서, 전체 원문을 찾는 Git 커밋 또는 경로를 적는다.

## 리디렉션 문서

기존 링크 파손 가능성이 큰 루트 문서는 삭제하지 않고 짧은 리디렉션만 남길 수 있다.

- 제목
- 위치 표시줄 1줄
- 더 이상 기본 읽기 대상이 아니라는 문장
- 현행 원본과 백업 원문 링크

리디렉션 문서는 설계 원본이 아니며 Codex가 내용을 재검토하거나 동기화 대상으로 취급하지 않는다.

## 기본 읽기 제외

다음은 사용자 요청이나 현재 작업 조건이 명시적으로 요구할 때만 읽는다.

- `docs/archive/**`
- 완료된 `docs/qa/**`
- 완료된 `docs/CODEX_GOAL_*`
- `docs/benchmarks/**`
- `docs/superpowers/**`
- 리디렉션 상태인 `DESIGN_INTENT.md`, `PROJECT_BRIEF.md`, `docs/CONTENT_DIRECTION_V09.md`
- Base 동기화 작업이 아닌 경우 `docs/BASE_RULES_VERSION.md`

## 현행 상태 표기

| 표기 | 의미 |
|---|---|
| 구현 확인 | 현재 `main`의 코드·데이터·테스트로 확인됨 |
| 승인 계획 | 사용자 승인과 전달 문서는 있으나 `main` 구현 완료가 아님 |
| 진행 중 | 작업 브랜치·dirty diff·미완료 테스트가 존재함 |
| 백업 | 현재 작업의 기본 읽기 대상이 아님 |

## 감사 주기

- 큰 MVP 완료 시 현재 상태·로드맵·체크리스트를 갱신한다.
- 5개 MVP마다 중복 문서, 오래된 다음 작업, 깨진 링크, 완료 Goal·QA의 기본 읽기 노출을 감사한다.
- 동일 정보가 세 문서 이상에 반복되면 책임 원본 하나를 정하고 나머지는 링크로 줄인다.
- GDD 변경 시 Markdown 원본과 DOCX 생성본의 동기화를 검증한다.
