# 괴이 기록국 게임 기획서

> 문서 위치: `[기획서]/01_게임기획서.md` | 상태 기준: `codex/mvp048-campaign-novel`의 `4cf39a2` (진행 브랜치) | 상세 부록: [`docs/GAME_DESIGN_DOCUMENT.md`](../docs/GAME_DESIGN_DOCUMENT.md)

## 이 문서만 읽는 사람을 위한 요약

**괴이 기록국**은 현대 도시에서 사람의 마음과 기억이 되살린 괴이의 규칙을 조사하는 PC 오컬트 수사 어드벤처다. 플레이어는 괴이를 처치하지 않는다. 관찰과 비교로 규칙을 밝혀 현재 출현을 **안정화**하고, 남은 **잔향**을 회수해 다음 피해를 막을 **괴이 매뉴얼**을 기록한다.

이 문서는 게임 기획팀의 활성 원본이다. 세계관, 플레이어 경험, 사건·UI·경제의 판단 원칙을 정한다. JSON ID, 저장 구조, 테스트 명령은 부록과 프로그래밍/QA 기획서를 따른다.

## 1. 플레이어 약속과 불변 조건

| 약속 | 플레이어가 하는 일 | 구현에서 지킬 것 |
|---|---|---|
| 미확인에서 검증으로 | 기록과 현장을 비교해 가설을 세운다 | 정답을 요원·장비·AI가 대신 고르지 않는다 |
| 실패도 기록이 된다 | 위험 행동의 결과를 보고 다음 후보를 배제한다 | 실패가 진행을 영구 차단하지 않으며 위험 사례를 남긴다 |
| 괴이는 기억의 반복이다 | 사람마다 달라진 기억의 공통 규칙을 찾는다 | 처치·사냥·킬 수치 언어를 쓰지 않는다 |
| 기록은 다음 생존이다 | 사건 뒤 매뉴얼과 DB에 검증된 사실을 남긴다 | 보고서·DB는 획득하지 않은 사실을 단정하지 않는다 |

공식 용어는 **괴이 기록국, 안정화 상태, 위험 사례, 잔향, 괴이 매뉴얼, 기록관 아카**다. 내부 호환 ID `로그`는 화면에 드러내지 않는다.

## 2. 핵심 플레이 루프

```text
긴급 보고/일상 선택
→ 현장 관찰과 기록 수집
→ 실패 사례·성공 사례·현재 관찰 비교
→ 판단 또는 노선 검증
→ 자동 관측 전조에 대응하며 안정화
→ 잔향 회수·보고서·매뉴얼 갱신
→ HQ의 다음 반일 계획
```

### 조사와 추리

- 주요 판단은 약 다섯 후보를 제시한다. 후보는 객관 기록 확인, 괴이 재현, 피해자 진술 신뢰, 현장 표기 신뢰, 공식 기록 대조처럼 서로 다른 정보원을 대표해야 한다.
- 실패 사례는 명백한 오답 2~3개를 배제하는 근거다. 성공 사례는 남은 후보의 공통 대응 원칙을, 현재 관찰은 마지막 적용 조건을 제공한다.
- 분석 지원은 최종 정답을 표시하지 않는다. 가능한 경우 오답 하나만 비활성화하고, 최소 두 후보가 남아 플레이어의 판단을 보장한다.
- 위험 행동은 적색 `위험 사례 기록`과 즉시 결과를 보여 준 뒤 같은 선택 목록으로 복귀한다. 단서·위험 사례·이미 적용된 효과는 되돌리지 않는다.

### 안정화·회수

`전조 자동 관측 → 행동 선택 → 직전 결과와 다음 전조`가 기본 리듬이다. 안정화 기준을 충족하면 회수 실행으로 전환한다. 대응 카드와 계약·장비는 안전을 보조할 뿐 성공을 자동으로 만들지 않는다.

### 저승역의 특례

첫 사건 저승역은 온보딩이다. `조사 → 내부 3×3 조작 학습(결과 미저장) → 4×4 최종 검증(결과 1회 저장) → 안전 노선 검증 기록 → 회수`를 사용한다. 3×3은 완성 미니게임이 아니다.

## 3. 세계관과 캠페인

### 세계관의 질문

괴이는 감정과 기억이 남아 있는 한 다른 장소와 모습으로 재현될 수 있다. 기록국은 그 반복 조건을 조사하고 현재 출현을 안정화한 뒤, 다음 조사자가 쓸 매뉴얼을 남긴다. 첫 사건 전에는 시장, 세력, 연구, 최적 편성을 장황하게 설명하지 않는다.

### 플레이 가능한 사건

| 사건 | 발생 감정·핵심 질문 | 플레이어 동사 | 현재 상태 |
|---|---|---|---|
| 저승역 | 놓친 마지막 기회·귀환 강박 / 왜 같은 방송을 서로 다른 목적지로 기억하는가 | 비교한다, 기다린다, 재구성한다 | 구현 확인 |
| 비 오는 골목의 빨간 우산 | 고립과 돌봄의 왜곡 | 관찰한다, 거리와 반응을 조절한다 | 구현 확인 |
| 폐주파수 방송국 | 통신 규칙과 출처의 혼선 | 출처를 판별하고 규칙을 대조한다 | 구현 확인 |
| 새벽 2시 17분의 도착 알림 | 이미 늦었다는 불안 / 왜 서로 다른 도착지를 기억하는가 | 시간표를 비교하고 발신 경로를 대조하고 기다린다 | 진행 브랜치 구현, 통합·수동 QA 전 |

### 하루와 요원

캠페인은 최대 10일의 오전·오후 반일로 진행한다. 주인공은 권나래이고, 최대 두 명의 서포트를 현장에 데려간다. 일상·관계는 수치형 연애 호감도가 아니라 신뢰·경계·부채·경쟁·공감 같은 선택 기억을 쌓는다. 요원의 역할은 판단의 근거를 보완하거나 위험 손실을 줄이는 것이며, 핵심 정답을 말해 주는 것이 아니다.

## 4. 진행 보상과 경제

| 자원 | 쓰임 | 금지 |
|---|---|---|
| 잔향 파편 | 검증된 시장 구매, 제작, 외부 계약 비용 | 현금·랜덤 가격·반복 노가다 |
| 연구 포인트 | 연구 과제 해금 | 정답 직접 공개 |
| 세력 관계 | 의뢰·연구·계약 접근 조건 | 필수 사건을 막는 과도한 등급 요구 |
| 사건 결과 | 보상 지급의 멱등 근거 | 재진입·재로드 중복 지급 |

외부 계약은 사건당 하나만 예약한다. 예약은 무료이고 사건 시작 때만 비용을 소비하며 취소는 무료다. 계약은 안전·비교 위치·검증 보상을 제한적으로 보조한다. 추리의 답이나 안정화 성공을 대신할 수 없다.

## 5. UI/UX 방향

- 기본 화면은 **현재 지문 → 선택지 → 즉시 결과**가 먼저 보인다. 단서, 위험 사례, 지원, 상세 수치는 매뉴얼·DB·접힌 상세에서 연다.
- 저승역은 예외적으로 장소/현장 기록/페이지형 매뉴얼의 3열 전용 화면을 사용한다. 4×4는 비교 근거/보드/현장 반응의 3열을 사용한다.
- 텍스트 노벨 장면은 배경, 현재 화자 한 명, 반응 인물 최대 한 명, 3~6줄 지문과 2~3개 선택지를 중심으로 한다. 저승역의 전용 조사·보드 UI를 덮어쓰지 않는다.
- 모든 핵심 입력은 1280×720에서 스크롤 없이 첫 선택, 닫기, 다음 진행을 볼 수 있어야 하며 1920×1080에서는 중앙 안전 폭을 유지한다.

시각 언어·최신 참조 이미지·자산의 정본은 [아트 기획서](03_아트기획서.md)와 [이미지 인덱스](이미지_인덱스.md)만 사용한다. 이 문서에는 이미지 사본을 넣지 않는다.

## 6. 현재 상태와 다음 우선순위

| 구분 | 사실 |
|---|---|
| 구현 확인 | 세 사건, 저승역 노선 검증, 5인·아카·일상·관계 기반, 연구·외부 계약 기반 |
| 진행 브랜치 구현 | 네 번째 사건, 텍스트 노벨 표현 셸, 추가 연구·계약 |
| 아직 검증 또는 보완 필요 | 네 번째 사건의 실제 완주/DB·보고서 수동 QA, 1280×720·1920×1080 캡처, 다섯 후보 추리 UI의 사건 간 일관성, 사운드 자산 제작 |

다음 작업은 이 표의 미검증 항목을 작은 end-to-end 단위로 닫는다. 구현 상태의 최종 기준은 항상 실제 코드·데이터·테스트와 [`docs/CURRENT_STATUS.md`](../docs/CURRENT_STATUS.md)다.

## 7. 작업 때 함께 갱신할 것

게임 규칙·서사·UI/UX·밸런스·경제를 바꾸면 이 문서를 갱신한다. 실제 장면이 달라지면 아트 기획서와 이미지 인덱스를, 검증 기준이 달라지면 QA 기획서를 동시에 심사한다. 자세한 강제 절차와 커밋 게이트는 [프로그래밍 기획서의 문서 동기화 계약](02_프로그래밍기획서_로드맵_MVP.md#7-문서-동기화-계약)에서 정한다.
