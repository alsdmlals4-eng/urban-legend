# MVP-038 순차 일정·세력 의뢰·조사 탈출

## 목표

캠페인을 `오전 계획 → 실행 → 결과 확인 → 오후 계획 → 실행 → 일일 마감`으로 진행한다. 조사와 의뢰는 각각 반일 단위 선택이며, 조사 중 HQ에 복귀해도 같은 작전과 반일을 이어간다.

## 구현 기준

- 캠페인 슬롯 단계는 `planning`, `in_progress`, `result`다.
- 현재 반일 일정만 편성하며 미래 반일은 미리 지정하지 않는다.
- 현장 편성 요원 2~3명은 모두 조사에 배치해야 사건을 시작한다.
- 실제 콘텐츠가 없는 일상·연구·정비는 잠금 사유를 표시한다.
- 오전 결과 확인 뒤 오후로, 오후 결과 확인 뒤 다음 날 오전으로 전환한다.
- 사건은 조사 일정이 있을 때만 선택하며 신규 캠페인에 기본 사건 선택은 없다.
- 조사는 반일만 소비한다. HQ 복귀 시 작전을 `suspended`로 저장하며 다른 일정과 사건 선택을 잠근다.
- 조사 화면의 `LOG`, `설정`, `HQ로 복귀`, `Esc`만 실제 조작으로 노출한다. 재생열이 없는 조사에서는 `AUTO/SKIP`을 표시하지 않는다.

## 의뢰 게시판

- 게시판은 최대 3칸이며 서로 다른 템플릿만 동시에 게시한다.
- `dispatch`는 요원 한 명의 반일과 1d100 판정을 사용한다.
- `recovery_action`은 수락 후 회수 화면에서 지정 능력을 처음 수행할 때 판정한다.
- 수락 의뢰는 완료·실패·취소까지 유지한다. 미수락 의뢰는 반일 전환마다 50% 확률로 유지 또는 교체한다.
- 거부는 무료, 수락 후 취소는 해당 세력 관계도 -1이다. 빈 슬롯은 다음 반일 전환에만 보충한다.
- 게시판과 난수 상태를 저장해 불러오기로 결과를 재추첨할 수 없게 한다.

## 공용 판정

`chance = clamp(50 + (score - difficulty) × 10, 5, 95)`이며 1d100 결과는 대성공(성공 범위의 1~5), 성공, 부분 성공(성공률 초과 15 이내), 실패로 나눈다. 조사 방식은 같은 판정을 사용하되 기존 성공·실패 효과를 유지하고, 회수 규칙 판단과 미니게임은 확률화하지 않는다.
